# Документация компонентов: wiki

## Общее описание

Директория `components/wiki` содержит компоненты для системы вики (базы знаний) в HR Helper. Компоненты обеспечивают отображение списка страниц вики, создание и редактирование страниц, просмотр детальной информации, управление тегами и категориями, загрузку файлов и историю изменений.

**Расположение**: `@/components/wiki`

Сводка отличий формы/UX от Next-эталона и правило фиксации расхождений: `docs/MIGRATION_DIVERGENCES.md`.

## Компоненты

### Основные компоненты списка

#### 1. WikiHeader
- **Файл**: `WikiHeader.tsx`
- **Назначение**: Заголовок страницы списка вики
- **Пропсы**: нет
- **Функциональность**: 
  - Отображает заголовок "Список страниц вики"
  - Кнопки быстрых действий:
    - "Быстрый старт" (RocketIcon)
    - "Путеводитель" (PersonIcon)
    - "Создать страницу" (PlusIcon) - переход к `/wiki/new/edit`
- **Используется в**: `/app/wiki/page.tsx`

#### 2. WikiFilters
- **Файл**: `WikiFilters.tsx`
- **Назначение**: Компонент поиска и фильтров для вики
- **Пропсы**:
  - `tags`: массив тегов (Tag[])
  - `selectedTag`: выбранный тег (string | null)
  - `onTagSelect`: обработчик выбора тега ((tagId: string | null) => void)
  - `searchQuery`: текущий поисковый запрос (string)
  - `onSearchChange`: обработчик изменения поискового запроса ((query: string) => void)
- **Функциональность**: 
  - Поле поиска: поиск по названию или содержимому
  - Фильтр по тегам: выбор тега из списка (или "Все теги")
  - Цветовая индикация выбранных тегов
- **Используется в**: `/app/wiki/page.tsx`

#### 3. WikiCategory
- **Файл**: `WikiCategory.tsx`
- **Назначение**: Компонент категории страниц вики
- **Пропсы**:
  - `category`: название категории (string)
  - `pages`: массив страниц в категории (WikiPage[])
- **Функциональность**: 
  - Отображает заголовок категории с иконкой папки
  - Отображает карточки страниц в категории через WikiCard
  - Сетка карточек с адаптивной версткой
- **Используется в**: `/app/wiki/page.tsx`

#### 4. WikiCard
- **Файл**: `WikiCard.tsx`
- **Назначение**: Компонент карточки страницы вики
- **Пропсы**:
  - `page`: данные страницы вики (WikiPage)
- **Функциональность**: 
  - Отображает карточку страницы вики
  - Заголовок, теги, описание, дата
  - Иконка открытия в новой вкладке
  - Выделение первой страницы в категории "Архитектура"
  - Цветовая индикация тегов
- **Используется в**: `WikiCategory.tsx`

### Компоненты детального просмотра

#### 5. WikiDetailHeader
- **Файл**: `WikiDetailHeader.tsx`
- **Назначение**: Заголовок страницы детального просмотра вики
- **Пропсы**:
  - `title`: название страницы (string)
  - `category`: категория страницы (string)
  - `tags`: массив тегов (string[])
- **Функциональность**: 
  - Отображает название страницы, категорию и теги
  - Навигационные хлебные крошки
  - Цветовая индикация тегов
- **Используется в**: `/app/wiki/[id]/page.tsx`

#### 6. WikiDetailContent
- **Файл**: `WikiDetailContent.tsx`
- **Назначение**: Компонент содержимого страницы вики
- **Пропсы**:
  - `content`: структурированное содержимое страницы
    - `heading`: главный заголовок (string)
    - `sections`: массив секций (ContentSection[])
- **Функциональность**: 
  - Отображает структурированное содержимое страницы
  - Главный заголовок, секции, подсекции
  - Списки элементов
  - ID для навигации по секциям
- **Используется в**: `/app/wiki/[id]/page.tsx`

#### 7. WikiDetailSidebar
- **Файл**: `WikiDetailSidebar.tsx`
- **Назначение**: Боковая панель с содержанием страницы вики
- **Пропсы**:
  - `pageId`: ID страницы (string)
  - `sections`: массив секций для навигации (Array<{ title: string }>)
  - `onSectionClick`: опциональный колбэк, вызывается после клика по пункту (для закрытия popover на мобильных)
- **Функциональность**: 
  - Отображает содержание страницы
  - Навигация по секциям при клике (плавная прокрутка к секции)
- **Используется в**: `/app/wiki/[id]/page.tsx` (на десктопе — в сайдбаре; на мобильных — внутри Popover по кнопке «Содержание»)

#### 8. WikiDetailHistory
- **Файл**: `WikiDetailHistory.tsx`
- **Назначение**: Компонент истории изменений страницы вики
- **Пропсы**: данные истории
- **Функциональность**: 
  - Отображает историю изменений страницы
  - Версии, даты, авторы, изменения
  - Просмотр полного текста версии
- **Используется в**: `/app/wiki/[id]/page.tsx`

### Компоненты редактирования

#### 9. WikiEditForm
- **Файл**: `WikiEditForm.tsx`
- **Назначение**: Форма редактирования/создания страницы вики
- **Пропсы**:
  - `initialData`: начальные данные страницы (Partial<WikiPageData>, опционально)
  - `isNew`: флаг создания новой страницы (boolean, по умолчанию false)
- **Функциональность**: 
  - Создание новой страницы вики
  - Редактирование существующей страницы
  - Поля: название, slug, категория, связанное приложение, теги, описание, содержимое, порядок, публикация
  - Автогенерация slug из названия
  - Управление тегами через WikiTagSelector
  - Загрузка файлов через WikiFileUploadModal
  - Удаление страницы через тост с подтверждением; плавающая панель кнопок через FloatingConfirmActions
  - Примечание к изменениям
- **Используется в**: `/app/wiki/new/edit/page.tsx`, `/app/wiki/[id]/edit/page.tsx`

**Структура и UX формы (актуальная):**

- **Шаблоны**: Верхний селект «Шаблон страницы» убран. Выбор шаблона только через кнопку «Добавить шаблон» в тулбаре блока «Содержание» и модальное окно (шаблоны страницы + блоки для вставки).
- **Раскладка**:
  - Верхняя строка: колонка полей (заголовок, название, URL, категория, приложение, описание, теги) + справа блок «Справка» (на десктопе). Форма обёрнута в `form#wiki-edit-form` с классом `formWrapper` (колонка на всю ширину).
  - Ниже — три отдельные карточки на всю ширину контента:
    1. **Карточка «Содержание»** (`mainContentBlock`): заголовок «Содержание», «Загрузить из файла», «Показать исходник», тулбар (форматирование, «Вставить блок…», «Добавить шаблон»), текстовое поле/превью.
    2. **Карточка «Параметры»** (`optionsCard`): порядок сортировки, чекбокс «Опубликовано», примечание к изменению (при редактировании).
    3. **Карточка «Действия»** (`actionsCard`): слева «Удалить» (только при редактировании), справа «Отмена» и «Сохранить изменения» / «Создать страницу».
- **Справка на мобильных** (≤768px): боковой блок «Справка» скрыт (`.helpContainer { display: none }`). В шапке формы справа от заголовка показывается круглая кнопка с иконкой Info (`.helpTriggerMobile`). По нажатию открывается модальное окно с тем же текстом справки (Markdown и рекомендации). Контент справки вынесен в переменную `helpContent` для использования в сайдбаре и в модалке.
- **Плавающие кнопки** (сохранение и отмена):
  - Показываются, когда карточка с кнопками действий не в зоне видимости (`IntersectionObserver` по `actionsCardRef`, порог 0.99).
  - Расположены справа, над футером (`bottom: 56px`). Позиция `right` вычисляется по контейнеру контента `[data-app-layout-content]` через `ResizeObserver`, чтобы при открытии/закрытии бокового меню кнопки смещались вместе с областью контента (без привязки к краю всего экрана).
  - В режиме «только иконка» — круглые кнопки 40×40 px (чуть меньше кнопок в карточке), иконка по центру без лишних отступов; при прокрутке к карточке кнопки визуально расширяются до полноразмерных с подписью (анимация по `--expansion`, 0…1).
  - Порядок в панели и в карточке: «Отмена» левее «Сохранить»/«Создать страницу».
  - Чтобы не было двойного контура, статические кнопки «Отмена» и «Сохранить» в карточке скрыты, пока видна плавающая панель (класс `actionsGroupHidden`: `opacity: 0`, `pointer-events: none`).
  - Фон панели: полупрозрачный за счёт `color-mix(in srgb, var(--color-background) 62%, transparent)` (светлая тема) и `color-mix(in srgb, var(--gray-2) 58%, transparent)` (тёмная); цвет берётся из темы, меняется только прозрачность. Размытие фона `backdrop-filter: blur(14px)`.
- **Прокрутка**: Прокручивается только контентная область (контейнер с `data-app-layout-content`). Body не прокручивается (в `globals.css`: `html, body { height: 100%; overflow: hidden }`, `#root { height: 100%; min-height: 0 }`).

- **Пропсы**:

- **Файл**: `WikiFileUploadModal.tsx`
- **Назначение**: Модальное окно загрузки файлов для страницы вики
- **Пропсы**:
  - `isOpen`: флаг открытости модального окна (boolean)
  - `onClose`: обработчик закрытия (() => void)
  - `onFileUpload`: обработчик загрузки файла ((file: File) => void)
- **Функциональность**: 
  - Загрузка файлов (изображения, документы)

#### 12. WikiDeleteConfirmDialog
- **Файл**: `WikiDeleteConfirmDialog.tsx`
- **Назначение**: Диалог подтверждения удаления страницы вики (Popover)
- **Пропсы**:
  - `isOpen`, `onClose`, `onOpenChange`, `onConfirm`, `trigger`, `message`
- **Используется в**: при необходимости поповера подтверждения (в форме вики удаление идёт через тост)

#### 13. FloatingConfirmActions
- **Файл**: `FloatingConfirmActions.tsx`, `FloatingConfirmActions.module.css`
- **Назначение**: Плавающие кнопки подтверждения («Удалить», «Отмена», «Сохранить») с анимацией расширения при прокрутке к карточке действий. Название отражает суть, без привязки к вики.
- **Пропсы**:
  - `formId`: id формы для `form="..."` у кнопки отправки
  - `onCancel`, `onDelete`: обработчики
  - `isNew`, `isSubmitting`: состояние формы
  - `actionsCardRef`: ref на блок карточки действий — по его **ширине** определяется «мобильный» режим (ширина ≤800px), плюс IntersectionObserver для анимации и видимости панели
  - `onMobileModeChange`: колбэк при смене режима по ширине блока (родитель скрывает карточку при узком блоке)
  - `onVisibilityChange`: колбэк при смене видимости панели (скрытие статических кнопок в карточке)
  - `saveLabel`, `createLabel`, `submittingLabel`: подписи кнопки сохранения
- **Функциональность**:
  - Расчёт отображения (мобильная версия или нет) определяется **шириной блока** карточки действий (ResizeObserver), а не шириной viewport.
  - При ширине блока ≤800px вызывается `onMobileModeChange(true)`; родитель скрывает карточку, панель всегда видна.
  - Фиксированная панель справа над футером, привязка к правому краю контента (ResizeObserver по контейнеру контента).
  - Кнопка «Удалить» (красная, круглая) — только при редактировании; капсула с «Отмена» и «Сохранить», анимация по `--expansion`.
- **Используется в**: `WikiEditForm.tsx`

## Интерфейсы и типы

### WikiPage
```typescript
interface WikiPage {
  id: string                                    // Уникальный идентификатор страницы
  title: string                                 // Название страницы
  description: string                            // Описание страницы
  tags: string[]                                // Массив тегов
  date: string                                  // Дата создания/обновления
  category: string                              // Категория страницы
}
```

### WikiPageData
```typescript
interface WikiPageData {
  id: string                                    // Уникальный идентификатор страницы
  title: string                                 // Название страницы
  slug: string                                  // URL-слаг страницы
  category: string                              // Категория страницы
  relatedApp: string                            // Связанное приложение
  tags: string[]                                // Массив тегов
  description: string                            // Описание страницы
  content: string                                // Содержимое страницы
  order: number                                 // Порядок сортировки
  isPublished: boolean                          // Флаг публикации
}
```

### ContentSection
```typescript
interface ContentSection {
  title: string                                 // Заголовок секции
  content?: string                              // Текст содержимого
  items?: string[]                              // Список элементов
  subsections?: Array<{                          // Подсекции
    title: string
    items: string[]
  }>
}
```

### Tag
```typescript
interface Tag {
  id: string                                    // Уникальный идентификатор тега
  label: string                                 // Название тега
  color: string                                 // Цвет тега (hex)
}
```

## Функции и обработчики

### WikiEditForm

#### handleChange
- **Назначение**: Обработчик изменения полей формы
- **Параметры**: `field: keyof WikiPageData`, `value: any`
- **Логика**: Обновляет значение поля в форме

#### handleSubmit
- **Назначение**: Обработчик отправки формы
- **Логика**: 
  - Валидирует обязательные поля
  - Сохраняет страницу через API
  - Перенаправляет на страницу просмотра

#### Плавающая панель и наблюдатели
- **ResizeObserver** на `[data-app-layout-content]`: при изменении размера контента (меню, ресайз) пересчитывается `floatingBarRight` (позиция панели справа от контента).
- **IntersectionObserver** на `actionsCardRef`: при появлении карточки действий в зоне видимости обновляются `expansionRatio` (0…1) и `actionsCardInView`; при `ratio >= 0.99` плавающая панель скрывается, статические кнопки в карточке показываются.

### WikiDetailSidebar

#### handleSectionClick
- **Назначение**: Обработчик клика на секцию в содержании
- **Параметры**: `sectionTitle: string`
- **Логика**: 
  - Находит элемент секции по ID
  - Прокручивает к секции с плавной анимацией

## Особенности реализации

1. **Структурированное содержимое**: WikiDetailContent поддерживает секции, подсекции и списки
2. **Автогенерация slug**: Slug автоматически генерируется из названия при создании
3. **Цветовая индикация тегов**: Каждый тег имеет свой цвет для визуального различия
4. **Навигация по секциям**: WikiDetailSidebar обеспечивает быструю навигацию по содержимому
5. **История изменений**: Отслеживание всех изменений страницы с версиями
6. **Загрузка файлов**: Поддержка загрузки изображений и документов
7. **Фильтрация по тегам**: Удобная фильтрация страниц по тегам
8. **Поиск**: Поиск по названию и содержимому страниц
9. **Категории**: Организация страниц по категориям
10. **Публикация**: Управление публикацией страниц (опубликовано/не опубликовано)
11. **Форма редактирования (WikiEditForm)**: Три карточки на всю ширину (Содержание, Параметры, Действия); справка на мобильных в модалке по кнопке инфо; плавающие кнопки «Отмена»/«Сохранить» справа над футером с привязкой к контейнеру контента и анимацией расширения при прокрутке; прокрутка только в области контента (body без скролла)
