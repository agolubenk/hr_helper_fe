# Документация компонентов: корневая директория components

## Общее описание

Корневая директория `components` содержит основные компоненты приложения HR Helper, которые обеспечивают базовую структуру, навигацию, управление темой и общие UI элементы. Эти компоненты используются во всем приложении и формируют основу пользовательского интерфейса.

**Расположение**: `@/components`

## Компоненты

### 1. AppLayout
- **Файл**: `AppLayout.tsx`, `AppLayout.module.css`
- **Назначение**: Основной layout компонент приложения
- **Пропсы**:
  - `children`: содержимое страницы (ReactNode)
  - `pageTitle`: заголовок страницы (string, по умолчанию "HR Helper")
  - `userName`: имя пользователя (string, по умолчанию "Голубенко Андрей")
  - `onLogout`: обработчик выхода из системы (() => void, опционально)
- **Функциональность**: 
  - Оборачивает все страницы приложения единым layout
  - Управляет структурой страницы (Header, Sidebar, контент)
  - Управляет состоянием бокового меню (открыто/закрыто)
  - Адаптивная верстка для мобильных и десктопных устройств
  - Сохранение состояния меню в localStorage
  - Условный рендеринг StatusBar для страницы recr-chat
- **Состояние**:
  - `menuOpen`: флаг открытости бокового меню (boolean)
  - `theme`: текущая тема приложения (light/dark)
- **Особенности**: 
  - На десктопе: меню может быть открыто/закрыто, состояние сохраняется в localStorage
  - На мобильных: меню всегда закрыто при загрузке, не сохраняется в localStorage
  - При изменении размера окна: если переключились на мобильное - меню закрывается
  - На странице recr-chat: дополнительно показывается StatusBar
  - Контент страницы отображается с отступом от Header (64px) и StatusBar (если есть, 48px)
  - Плавные переходы при изменении размера и открытии/закрытии меню
- **Константы**:
  - `SIDEBAR_STATE_STORAGE_KEY`: ключ для сохранения состояния меню в localStorage ('sidebarMenuOpen')
  - `DESKTOP_BREAKPOINT`: точка перелома для определения десктопных устройств (768px)
- **Используется в**: Все страницы приложения через layout

### 2. FloatingActions
- **Файл**: `FloatingActions.tsx`
- **Назначение**: Плавающие кнопки быстрых действий
- **Пропсы**:
  - `actions`: массив кастомных действий (FloatingAction[], опционально)
- **Функциональность**: 
  - Быстрый доступ к часто используемым функциям
  - Настраиваемые быстрые кнопки (ссылки, текст, дата/время)
  - Кнопка прокрутки страницы вверх
  - Кнопка настроек быстрых кнопок
  - Закрепление/открепление панели
  - Невидимая зона срабатывания: показ панели при наведении на левый край экрана
  - Сохранение состояния в localStorage (закрепление, видимость кнопок)
  - Синхронизация состояния между вкладками браузера
- **Состояние**:
  - `isPinned`: флаг закрепления панели (boolean)
  - `isVisible`: флаг видимости панели (boolean)
  - `isHovering`: флаг наведения на панель (boolean)
  - `isScrollTopEnabled`: флаг видимости кнопки "Вверх" (boolean)
  - `isSettingsEnabled`: флаг видимости кнопки "Настройки" (boolean)
  - `isQuickButtonsEnabled`: флаг включения быстрых кнопок (boolean)
- **Особенности**: 
  - По умолчанию скрыта, появляется при наведении на левый край экрана
  - При закреплении всегда видна
  - При выключении быстрых кнопок скрывается (если не закреплена)
  - Кнопки можно включать/выключать в настройках профиля
  - Состояние синхронизируется между вкладками браузера через CustomEvent и storage events
  - Плавная прокрутка страницы вверх с easing функцией
  - Адаптивное позиционирование (учитывает StatusBar на странице recr-chat)
- **Константы**:
  - `STORAGE_KEY`: ключ для сохранения состояния закрепления ('floatingActionsPinned')
  - `SCROLL_TOP_BUTTON_STORAGE_KEY`: ключ для видимости кнопки "Вверх" ('floatingActionsScrollTopEnabled')
  - `SETTINGS_BUTTON_STORAGE_KEY`: ключ для видимости кнопки "Настройки" ('floatingActionsSettingsEnabled')
  - `QUICK_BUTTONS_ENABLED_KEY`: ключ для включения быстрых кнопок ('quickButtonsEnabled', экспортируется)
- **Используется в**: `AppLayout.tsx`

### 3. FloatingLabelInput
- **Файл**: `FloatingLabelInput.tsx`
- **Назначение**: Поле ввода с плавающей меткой
- **Пропсы**:
  - `id`: уникальный идентификатор поля (string)
  - `label`: текст метки поля (string)
  - `type`: тип поля ввода (string, по умолчанию "text")
  - `value`: значение поля (string)
  - `onChange`: обработчик изменения значения ((e: React.ChangeEvent<HTMLInputElement>) => void)
  - `placeholder`: текст плейсхолдера (string, опционально)
  - `required`: флаг обязательности поля (boolean, по умолчанию false)
  - `icon`: иконка слева от поля (ReactNode, опционально)
  - `disabled`: флаг отключенного состояния (boolean, по умолчанию false)
- **Функциональность**: 
  - Плавающая метка: метка перемещается вверх при фокусе или наличии значения
  - Иконка: опциональная иконка слева от поля ввода
  - Обязательное поле: отображение звездочки для обязательных полей
  - Отключенное состояние: визуальная индикация отключенного поля
  - Адаптивные стили: изменение стилей при фокусе и заполнении
- **Состояние**:
  - `isFocused`: флаг фокуса на поле ввода (boolean)
- **Особенности**: 
  - При фокусе: метка перемещается вверх, показывается placeholder (если есть)
  - При вводе текста: метка остается вверху
  - При потере фокуса без значения: метка возвращается в центр
  - При потере фокуса со значением: метка остается вверху
  - При фокусе: граница и тень меняют цвет на акцентный
  - Плавные переходы при изменении позиции метки и иконки
  - Фон метки для перекрытия границы поля когда "всплыла"
- **Используется в**: Формах по всему приложению

### 4. Header
- **Файл**: `Header.tsx`, `Header.module.css`
- **Назначение**: Верхняя панель приложения
- **Пропсы**:
  - `pageTitle`: заголовок текущей страницы (string)
  - `userName`: имя пользователя (string, опционально, по умолчанию "Голубенко Андрей")
  - `onMenuToggle`: обработчик открытия/закрытия меню (() => void, опционально)
  - `onThemeToggle`: обработчик переключения темы (() => void)
  - `currentTheme`: текущая тема ('light' | 'dark')
  - `menuOpen`: флаг открытости меню (boolean, опционально, по умолчанию false)
  - `onLogout`: обработчик выхода из системы (() => void)
- **Функциональность**: 
  - Отображение заголовка текущей страницы
  - Глобальный поиск по приложению (GlobalSearch)
  - Управление темой (светлая/темная)
  - Уведомления (в разработке)
  - Информация о пользователе и выход из системы
  - Кнопка открытия/закрытия бокового меню
  - Горячие клавиши Cmd+S (Mac) / Ctrl+S (Windows/Linux) для фокуса на поиск
- **Состояние**:
  - `userHover`: флаг наведения на область пользователя (boolean)
  - `logoutHover`: флаг наведения на кнопку выхода (boolean)
  - `shortcutKey`: текст горячей клавиши для поиска ('⌘S' | 'Ctrl+S')
- **Особенности**: 
  - Фиксированная позиция вверху страницы (position: fixed, top: 0)
  - Высота 64px
  - z-index 1000 (поверх контента)
  - Адаптивный фон в зависимости от темы
  - Адаптивная верстка для мобильных и десктопных устройств
  - На мобильных скрывается полное имя пользователя и текст "Выход"
  - При клике на имя пользователя - переход на страницу профиля
  - При клике на кнопку выхода - вызов onLogout
- **Используется в**: `AppLayout.tsx`

### 5. QuickButtonsContext
- **Файл**: `QuickButtonsContext.tsx`
- **Назначение**: Контекст для управления быстрыми кнопками
- **Пропсы**:
  - `children`: дочерние компоненты (ReactNode)
  - `buttons`: массив быстрых кнопок (QuickButton[])
- **Функциональность**: 
  - Предоставляет контекст для доступа к быстрым кнопкам
  - Используется для синхронизации быстрых кнопок между компонентами
- **Интерфейсы**:
  - `QuickButton`: интерфейс быстрой кнопки (id, name, icon, color, type, value, order)
  - `QuickButtonsContextType`: интерфейс контекста (buttons)
- **Хуки**:
  - `useQuickButtons`: хук для доступа к контексту быстрых кнопок
- **Используется в**: Компонентах, которые работают с быстрыми кнопками

### 6. Sidebar
- **Файл**: `Sidebar.tsx`, `Sidebar.module.css`
- **Назначение**: Боковое меню навигации приложения
- **Пропсы**:
  - `isOpen`: флаг открытости меню (boolean)
  - `onClose`: обработчик закрытия меню (() => void)
- **Функциональность**: 
  - Навигация по разделам приложения
  - Иерархическая структура меню с вложенными пунктами
  - Подсветка активного пункта меню
  - Автоматическое раскрытие активных разделов
  - Закрытие меню на мобильных устройствах при навигации
  - Обработка пунктов "в разработке" (показ toast вместо навигации)
  - Синхронизация активной вкладки профиля через localStorage и CustomEvent
- **Особенности**: 
  - Иерархическое меню с поддержкой вложенных пунктов
  - Автоматическое определение активного пункта по текущему пути
  - Раскрытие/сворачивание разделов меню
  - Навигация по внутренним и внешним ссылкам
  - Адаптивное позиционирование (учитывает StatusBar на странице recr-chat)
  - На мобильных закрывается при навигации
  - Плавная анимация появления/исчезновения через transform
  - Вертикальная прокрутка при переполнении
- **Константы**:
  - `IN_DEVELOPMENT_IDS`: множество ID пунктов меню, которые находятся в разработке
- **Используется в**: `AppLayout.tsx`

### 7. StatusBar (AtsStatusBar)
- **Файлы**: `StatusBar.tsx` (реэкспорт), `layout/StatusBar/AtsStatusBar.tsx`, `layout/StatusBar/AtsStatusBar.module.css` — как в приложении `frontend`.
- **Назначение**: Статусная панель для страницы `/ats`
- **Пропсы**:
  - `selectedStatus`: выбранный фильтр по статусу (string | null, опционально; иначе внутреннее состояние)
  - `onStatusChange`: смена фильтра ((status: string | null) => void, опционально; иначе событие `ATS_STATUS_CHANGE_EVENT`)
  - `statusCounts`: счётчики по статусам (Record<string, number>, опционально; иначе внутреннее состояние / событие `ATS_STATUS_COUNTS_EVENT`)
- **Функциональность**: 
  - Выбор вакансии (Мои / Все / конкретная), пиллы этапов с числами, группа «N этапов без кандидатов», раскрытие группы
- **Состояние**:
  - `selectedVacancy`, `viewMode`, `expandedGroups`, локальный выбранный статус и счётчики (или из пропсов)
- **Особенности**: 
  - Показывается только на странице `/ats` (см. `AppLayout`)
  - Фиксированная позиция под Header (top: 64px)
  - Высота 48px
  - Горизонтальный скролл со статусами
  - Группировка неактивных статусов (count === undefined || count === 0)
  - Раскрытие групп для просмотра всех статусов
  - Статус "Все" показывает всех кандидатов независимо от статуса
- **Моковые данные**:
  - `DEFAULT_VACANCIES`, демо-счётчики `DEMO_STATUS_COUNTS` (до API)
- **Используется в**: `AppLayout.tsx` (только на странице `/ats`)

### 8. ThemeProvider
- **Файл**: `ThemeProvider.tsx`
- **Назначение**: Провайдер темы приложения
- **Пропсы**:
  - `children`: дочерние компоненты (ReactNode)
- **Функциональность**: 
  - Управление темой приложения (светлая/темная)
  - Управление акцентными цветами для светлой и темной темы
  - Сохранение настроек темы в localStorage
  - Определение системной темы при первой загрузке
  - Применение темы к HTML и body элементам
  - Предотвращение hydration mismatch (mounted флаг)
- **Состояние**:
  - `theme`: текущая тема (light/dark)
  - `mounted`: флаг монтирования компонента (boolean)
  - `lightThemeAccentColor`: акцентный цвет для светлой темы (AccentColorValue)
  - `darkThemeAccentColor`: акцентный цвет для темной темы (AccentColorValue)
- **Особенности**: 
  - При первой загрузке: проверяет localStorage, если нет - использует системную тему
  - При переключении темы: сохраняет в localStorage и применяет к DOM
  - При изменении акцентного цвета: сохраняет в localStorage
  - Применяет тему к HTML и body для глобального применения
  - Использует Radix UI Theme компонент для применения темы к компонентам
  - Предотвращает hydration mismatch через mounted флаг
- **Хуки**:
  - `useTheme`: хук для доступа к контексту темы
- **Используется в**: `/app/layout.tsx` (оборачивает все страницы)

## Функции и обработчики

### AppLayout

#### isDesktop
- **Назначение**: Проверка, является ли устройство десктопом
- **Возвращает**: `boolean` - true если устройство десктоп
- **Логика**: Проверяет ширину окна браузера (>= 768px)

#### handleLogout
- **Назначение**: Обработчик выхода из системы
- **Логика**: Вызывает переданный обработчик onLogout, если он есть

#### handleMenuToggle
- **Назначение**: Обработчик переключения состояния меню
- **Логика**: Инвертирует текущее состояние menuOpen

### FloatingActions

#### handleSettingsClick
- **Назначение**: Обработчик клика на кнопку "Настройки"
- **Логика**: 
  - Сохраняет активную вкладку 'quick-buttons' в localStorage
  - Отправляет CustomEvent для синхронизации
  - Переходит на страницу профиля

#### handlePinToggle
- **Назначение**: Обработчик переключения закрепления панели
- **Логика**: Инвертирует состояние isPinned, при закреплении сразу показывает панель

#### handleMouseEnter
- **Назначение**: Обработчик наведения на панель
- **Логика**: Показывает панель при наведении (если быстрые кнопки включены), отменяет таймер скрытия

#### handleMouseLeave
- **Назначение**: Обработчик ухода курсора с панели
- **Логика**: Скрывает панель через 300мс после ухода курсора (если не закреплена)

#### smoothScrollTo
- **Назначение**: Плавная прокрутка с easing функцией
- **Параметры**: `element: HTMLElement | Window`, `target: number`, `duration: number = 800`
- **Возвращает**: `Promise<void>`
- **Логика**: Плавно прокручивает элемент к целевой позиции с easing функцией ease-in-out-cubic

#### scrollToTop
- **Назначение**: Прокрутка страницы в самый верх
- **Логика**: 
  - Проверяет текущую позицию прокрутки
  - Если уже вверху - не выполняет прокрутку
  - Прокручивает все элементы (window, documentElement, body) для совместимости
  - Использует easing функцию для плавной анимации
  - Выполняет финальную корректировку позиций после анимации

#### renderIcon
- **Назначение**: Рендеринг иконки на основе имени
- **Параметры**: `iconName: string`, `size: number = 20`
- **Возвращает**: React компонент иконки или null
- **Логика**: Преобразует строковое имя иконки в React компонент

### Sidebar

#### isItemOrChildrenActive
- **Назначение**: Проверка, является ли пункт меню или его дочерние элементы активными
- **Параметры**: `item: MenuItem`, `pathname: string | null | undefined`
- **Возвращает**: `boolean`
- **Логика**: 
  - Проверяет, соответствует ли текущий путь пункту меню или его дочерним элементам
  - Обрабатывает специальные случаи (home -> /workflow, wiki -> /wiki/* и т.д.)
  - Рекурсивно проверяет дочерние элементы

#### handleClick (MenuItemComponent)
- **Назначение**: Обработчик клика на пункт меню
- **Логика**: 
  - Для пунктов "в разработке" показывает toast
  - Для пунктов с дочерними элементами - раскрывает/сворачивает
  - Для пунктов без дочерних элементов - выполняет навигацию
  - Синхронизирует активную вкладку профиля через localStorage и CustomEvent

### StatusBar

#### toggleGroup
- **Назначение**: Переключение раскрытости группы статусов
- **Параметры**: `groupId: string`
- **Логика**: Добавляет/удаляет groupId из множества раскрытых групп

#### handleAddVacancy
- **Назначение**: Обработчик добавления новой вакансии
- **Логика**: Открывает модальное окно добавления вакансии

### ThemeProvider

#### toggleTheme
- **Назначение**: Переключение темы
- **Логика**: 
  - Инвертирует текущую тему (light <-> dark)
  - Сохраняет новую тему в localStorage
  - Применяет тему к DOM элементам

#### setLightThemeAccentColor
- **Назначение**: Установка акцентного цвета для светлой темы
- **Параметры**: `color: AccentColorValue`
- **Логика**: Устанавливает цвет и сохраняет в localStorage

#### setDarkThemeAccentColor
- **Назначение**: Установка акцентного цвета для темной темы
- **Параметры**: `color: AccentColorValue`
- **Логика**: Устанавливает цвет и сохраняет в localStorage

## Хуки и эффекты

### AppLayout

#### useEffect: сохранение состояния меню в localStorage
- **Назначение**: Сохраняет состояние меню в localStorage при изменении
- **Зависимости**: `menuOpen`
- **Логика**: 
  - На десктопе: сохраняет состояние в localStorage
  - На мобильных: удаляет сохраненное состояние

#### useEffect: обработчик изменения размера окна
- **Назначение**: Отслеживает изменение размера окна браузера
- **Зависимости**: нет
- **Логика**: 
  - При уменьшении окна до мобильного размера (< 768px) закрывает меню
  - Удаляет сохраненное состояние из localStorage при переключении на мобильное

### FloatingActions

#### useEffect: синхронизация состояния быстрых кнопок между вкладками
- **Назначение**: Слушает CustomEvent 'localStorageChange' для синхронизации
- **Зависимости**: нет
- **Логика**: Обновляет isQuickButtonsEnabled при изменении в другой вкладке

#### useEffect: синхронизация состояния кнопок "Вверх" и "Настройки" между вкладками
- **Назначение**: Слушает события storage и localStorageChange для синхронизации
- **Зависимости**: нет
- **Логика**: 
  - Слушает события storage (для синхронизации между вкладками)
  - Слушает кастомное событие localStorageChange (для синхронизации в той же вкладке)
  - Периодически проверяет состояние (каждые 500мс)
  - Проверяет состояние при возвращении фокуса на окно

#### useEffect: сохранение состояния закрепления в localStorage
- **Назначение**: Сохраняет состояние isPinned в localStorage при изменении
- **Зависимости**: `isPinned`
- **Логика**: Сохраняет состояние для восстановления при следующей загрузке

#### useEffect: скрытие панели при выключении быстрых кнопок
- **Назначение**: Скрывает панель, если быстрые кнопки выключены и панель не закреплена
- **Зависимости**: `isQuickButtonsEnabled`, `isPinned`
- **Логика**: Скрывает панель только если быстрые кнопки выключены И панель не закреплена

### Header

#### useEffect: определение горячей клавиши по операционной системе
- **Назначение**: Определяет, является ли устройство Mac
- **Зависимости**: нет
- **Логика**: Устанавливает '⌘S' для Mac, 'Ctrl+S' для других ОС

#### useEffect: обработчик горячих клавиш для поиска
- **Назначение**: Обрабатывает комбинацию Cmd+S / Ctrl+S
- **Зависимости**: нет
- **Логика**: 
  - Предотвращает стандартное поведение браузера
  - Фокусирует input поиска и выделяет текст

### Sidebar

#### useEffect: обновление состояния раскрытости при изменении pathname
- **Назначение**: Обновляет isExpanded при изменении shouldBeExpanded
- **Зависимости**: `shouldBeExpanded`
- **Логика**: Обеспечивает автоматическое раскрытие разделов при навигации

### ThemeProvider

#### useEffect: инициализация темы при монтировании компонента
- **Назначение**: Загружает сохраненную тему из localStorage или определяет системную
- **Зависимости**: нет
- **Логика**: 
  - Загружает сохраненную тему из localStorage
  - Если тема не сохранена - определяет системную тему (prefers-color-scheme)
  - Загружает сохраненные акцентные цвета
  - Применяет тему к DOM элементам
  - Устанавливает mounted в true

#### useEffect: применение темы к DOM при изменении
- **Назначение**: Применяет текущую тему к DOM элементам при изменении
- **Зависимости**: `theme`, `mounted`
- **Логика**: 
  - Применяет тему только после монтирования (mounted === true)
  - Обновляет атрибуты, стили и классы для темы

## Вычисляемые значения

### AppLayout

#### isRecrChatPage
- **Назначение**: Проверка, является ли текущая страница recr-chat
- **Вычисление**: `pathname?.startsWith('/recr-chat')`
- **Используется для**: Условного рендеринга StatusBar

### FloatingActions

#### topOffset
- **Назначение**: Отступ сверху для панели
- **Вычисление**: 
  - На странице recr-chat: '112px' (64px Header + 48px StatusBar)
  - На остальных страницах: '64px' (только Header)
- **Используется для**: Корректного позиционирования панели

#### isFloating (FloatingLabelInput)
- **Назначение**: Флаг "всплытия" метки
- **Вычисление**: `isFocused || value.length > 0`
- **Используется для**: Определения позиции метки

#### firstName (Header)
- **Назначение**: Извлечение имени из полного имени пользователя
- **Вычисление**: `userName.split(' ')[1] || userName`
- **Используется для**: Отображения сокращенного имени на мобильных

### StatusBar

#### myVacancies
- **Назначение**: Список "моих" вакансий
- **Вычисление**: 
  - Если передан myVacancyIds - фильтрует вакансии по этим ID
  - Иначе - берет первые 2 вакансии из списка
- **Используется для**: Отображения в разделе "Мои"

#### triggerLabel
- **Назначение**: Текст для кнопки выбора вакансии
- **Вычисление**: 
  - '__all__' -> 'Все'
  - '__my__' -> 'Мои'
  - ID вакансии -> название вакансии
  - Иначе -> 'Выберите вакансию'
- **Используется для**: Отображения выбранной вакансии

#### groupedStatuses
- **Назначение**: Сгруппированные статусы (активные отдельно, неактивные группами)
- **Вычисление**: 
  - Группирует неактивные статусы (count === undefined || count === 0) в группы
  - Активные статусы (count > 0) остаются отдельными элементами
- **useMemo**: Пересчитывается только при изменении statuses
- **Используется для**: Отображения статусов в горизонтальном скролле

### ThemeProvider

#### currentAccentColor
- **Назначение**: Текущий акцентный цвет на основе активной темы
- **Вычисление**: 
  - Если тема светлая - возвращает lightThemeAccentColor
  - Если тема темная - возвращает darkThemeAccentColor
- **Используется для**: Применения акцентного цвета к Radix UI Theme компоненту

## Особенности реализации

1. **Адаптивность**: Все компоненты адаптированы для мобильных и десктопных устройств
2. **localStorage**: Сохранение состояния меню, темы, быстрых кнопок в localStorage
3. **Синхронизация между вкладками**: CustomEvent и storage events для синхронизации состояния
4. **SSR поддержка**: Проверка `typeof window !== 'undefined'` для предотвращения ошибок SSR
5. **Hydration mismatch**: Использование mounted флага для предотвращения несоответствий
6. **Плавные анимации**: Плавные переходы при открытии/закрытии меню, появлении панелей
7. **Горячие клавиши**: Поддержка горячих клавиш для поиска (Cmd+S / Ctrl+S)
8. **Условный рендеринг**: StatusBar показывается только на странице recr-chat
9. **Группировка статусов**: Умная группировка неактивных статусов в StatusBar
10. **Плавающая метка**: Улучшенный UX в FloatingLabelInput с плавными переходами
11. **Закрепление панели**: Возможность закрепления FloatingActions для постоянной видимости
12. **Автоматическое раскрытие**: Автоматическое раскрытие активных разделов в Sidebar
13. **Системная тема**: Определение системной темы при первой загрузке
14. **Акцентные цвета**: Отдельные акцентные цвета для светлой и темной темы
15. **Плавная прокрутка**: Easing функция для плавной прокрутки страницы вверх
