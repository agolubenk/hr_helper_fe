# Документация компонента: IntegrationsPage

## Общее описание

Компонент страницы интеграций и API. Отображает статус всех доступных интеграций и позволяет настраивать API ключи и токены.

**Расположение**: `components/profile/IntegrationsPage.tsx`  
**Использование**: Вкладка "Интеграции и API" страницы профиля (`/account/profile?tab=integrations`)

## Функциональность

### Поддерживаемые интеграции

1. **Gemini AI** - AI сервис от Google
2. **Huntflow** - ATS система (с поддержкой песочницы и продакшн)
3. **ClickUp** - система управления проектами
4. **Notion** - система документов и баз знаний
5. **Telegram** - мессенджер
6. **Google** - OAuth и API сервисы Google
7. **hh.ru / rabota.by** - платформы для поиска работы
8. **OpenAI** - AI сервис
9. **Cloud AI** - облачный AI сервис
10. **n8n** - автоматизация и интеграции

### Отображение статуса

**Общий статус**:
- Иконка галочки (зеленая) - интеграция настроена
- Кнопка настроек (шестеренка) - открывает модальное окно

**Детальный статус (Huntflow)**:
- Источник ключей: мои / компании / выключено
- Активная система: prod / sandbox
- Статус токенов: Access Token, Refresh Token
- Статус API Песочницы (если активна система sandbox)

## Состояние компонента

### State переменные

- `selectedIntegration`: название выбранной интеграции для настройки (null если ничего не выбрано)
- `isModalOpen`: флаг открытости модального окна настроек
- `huntflowUserSettings`: локальные настройки пользователя для Huntflow (из localStorage)

## Функции и обработчики

### useEffect - загрузка настроек Huntflow

**Назначение**: Загрузка локальных настроек Huntflow при монтировании компонента

**Логика**: Читает настройки из localStorage через `getHuntflowUserSettings()`

### handleConfigure(integrationName: string)

**Назначение**: Открытие модального окна настроек интеграции

**Поведение**: Устанавливает `selectedIntegration` и открывает модальное окно

### handleCloseModal()

**Назначение**: Закрытие модального окна настроек

**Поведение**: Закрывает модальное окно и сбрасывает выбранную интеграцию

### handleSave(data: any)

**Назначение**: Сохранение настроек интеграции

**Логика**:
- Для Huntflow: перечитывает локальные настройки из localStorage
- Закрывает модальное окно

## Компоненты

## Специальная обработка Huntflow

### Локальные настройки пользователя

**Источник**: localStorage через `getHuntflowUserSettings()`

**Структура**:
- `credentialSource`: источник ключей ('mine' | 'company' | 'disabled')
- `activeSystem`: активная система ('prod' | 'sandbox')

### Отображение статуса

**Если `credentialSource === 'disabled'`**:
- Показывает "Выключено (подтверждено)"

**Если `credentialSource === 'company'`**:
- Показывает "Ключи компании"

**Если `credentialSource === 'mine'`**:
- Показывает активную систему (prod/sandbox)
- Показывает статус Access Token и Refresh Token
- Если активна система sandbox - показывает статус API Песочницы
