# Документация компонента: QuickButtonsPage

## Общее описание

Компонент управления быстрыми кнопками. Позволяет создавать, редактировать и удалять быстрые кнопки для быстрого доступа к важной информации.

**Расположение**: `components/profile/QuickButtonsPage.tsx`  
**Использование**: Вкладка "Быстрые кнопки" страницы профиля (`/account/profile?tab=quick-buttons`)

## Функциональность

### Управление кнопками

- **Создание**: создание новой быстрой кнопки через модальное окно
- **Редактирование**: редактирование существующей кнопки
- **Удаление**: удаление кнопки с подтверждением через toast
- **Максимальное количество**: 15 кнопок

### Типы быстрых кнопок

1. **'link'** - Ссылка
   - Открывается в новой вкладке при клике
   - Пример: `https://huntflow.ru`, `mailto:email@example.com`

2. **'text'** - Текст
   - Вставляется в активное поле ввода при клике
   - Пример: "11:00 - 18:30"

3. **'datetime'** - Дата/время
   - Вставляется текущая дата-время при клике
   - Пример: "2026-01-15T14:00"

### Настройки видимости

1. **Быстрые кнопки** (`isQuickButtonsEnabled`)
   - Включение/выключение всех быстрых кнопок
   - Ключ localStorage: `QUICK_BUTTONS_ENABLED_KEY`

2. **Кнопка "Вверх"** (`isScrollTopEnabled`)
   - Видимость кнопки прокрутки в начало страницы
   - Ключ localStorage: `SCROLL_TOP_BUTTON_STORAGE_KEY`

3. **Кнопка "Настройки"** (`isSettingsEnabled`)
   - Видимость кнопки настроек
   - Ключ localStorage: `SETTINGS_BUTTON_STORAGE_KEY`

## Состояние компонента

### State переменные

- `buttons`: массив быстрых кнопок пользователя
- `isModalOpen`: флаг открытости модального окна создания/редактирования
- `editingButton`: кнопка для редактирования (null если создание новой)
- `isQuickButtonsEnabled`: включение/выключение всех быстрых кнопок
- `isScrollTopEnabled`: видимость кнопки "Вверх"
- `isSettingsEnabled`: видимость кнопки "Настройки"

## Функции и обработчики

### useEffect - сохранение настроек в localStorage

**Назначение**: Сохранение настроек видимости кнопок в localStorage и синхронизация

**Механизмы синхронизации**:
1. Сохранение в localStorage
2. Отправка кастомного события 'localStorageChange' для синхронизации в той же вкладке
3. Слушатель изменений localStorage для синхронизации между вкладками браузера

### handleCreate()

**Назначение**: Создание новой быстрой кнопки

**Логика**:
- Проверяет максимальное количество кнопок (15)
- Открывает модальное окно для создания новой кнопки

### handleEdit(button: QuickButton)

**Назначение**: Редактирование существующей быстрой кнопки

**Логика**:
- Устанавливает кнопку для редактирования
- Открывает модальное окно с данными выбранной кнопки

### handleDelete(id: string)

**Назначение**: Удаление быстрой кнопки

**Логика**:
- Показывает подтверждение удаления через toast
- Удаляет кнопку из массива после подтверждения

### handleSave(data)

**Назначение**: Сохранение быстрой кнопки (создание или редактирование)

**Логика**:
- Если `editingButton` не null - обновляет существующую кнопку
- Если `editingButton` null - создает новую кнопку с временным ID
- Сортирует кнопки по полю `order` после добавления новой

### getTypeDisplay(type: string)

**Назначение**: Преобразование типа кнопки в читаемое название

**Маппинг**:
- 'link' → 'Ссылка'
- 'text' → 'Текст'
- 'datetime' → 'Дата/время'

### renderIcon(iconName: string, size: number)

**Назначение**: Рендеринг иконки по имени

**Логика**:
- Получает компонент иконки из `iconComponents` по имени
- Использует fallback (эмодзи ⚡) если иконка не найдена

**Поддерживает**: Более 60 различных иконок из Radix UI

## Интерфейсы

### QuickButton

```typescript
interface QuickButton {
  id: string
  name: string
  icon: string
  color: string
  type: 'link' | 'text' | 'datetime'
  value: string
  order: number
}
```

## Константы

### Ключи localStorage

- `QUICK_BUTTONS_ENABLED_KEY`: включение/выключение всех быстрых кнопок
- `SCROLL_TOP_BUTTON_STORAGE_KEY`: видимость кнопки "Вверх"
- `SETTINGS_BUTTON_STORAGE_KEY`: видимость кнопки "Настройки"

### iconComponents

Маппинг имен иконок на компоненты из Radix UI (более 60 иконок)

## Синхронизация с FloatingActions

Компонент синхронизируется с `FloatingActions` через общие ключи localStorage:
- При изменении настроек в QuickButtonsPage обновляется FloatingActions
- При изменении настроек в FloatingActions обновляется QuickButtonsPage
- Синхронизация работает между вкладками браузера через storage events
