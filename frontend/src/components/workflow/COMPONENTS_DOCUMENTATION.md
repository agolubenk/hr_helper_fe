# Документация компонентов: workflow

## Общее описание

Директория `components/workflow` содержит компоненты для управления процессом скрининга и интервью кандидатов в HR Helper. Компоненты обеспечивают чат-интерфейс для обработки команд, управление расписанием интервью, отображение свободных слотов, настройку параметров интервью и отображение статистики.

**Расположение**: `@/components/workflow`

## Компоненты

### Основные компоненты

#### 1. WorkflowHeader
- **Файл**: `WorkflowHeader.tsx`
- **Назначение**: Заголовок страницы workflow (скрининг/интервью)
- **Пропсы**:
  - `onSlotsClick`: обработчик клика на кнопку "слоты" (() => void)
  - `slotsOpen`: флаг открытости панели слотов (boolean)
- **Функциональность**: 
  - Управление процессом скрининга и интервью
  - Выбор типа процесса (динамические кнопки на основе этапов найма)
  - Выбор вакансии
  - Настройка параметров встреч (формат, интервьюеры, офис)
  - Быстрые действия (ссылки на коммуникацию, календарь, вакансия, слоты)
  - Быстрые кнопки: ссылки на коммуникацию (Telegram, WhatsApp, Viber, LinkedIn, Email)
  - Тогглер типа процесса: динамические кнопки на основе этапов найма с меткой "встреча"
    - Кнопка "Скрининг" - всегда доступна (30 мин)
    - Динамические кнопки этапов-встреч - формируются из этапов найма с `isMeeting = true`
    - Количество кнопок: 0 и более (зависит от количества этапов с `isMeeting = true`)
    - Названия кнопок: берутся из названий этапов найма, отмеченных чекбоксом "встреча"
    - Если этапов-встреч нет, тогглер может быть пустым или содержать только "Скрининг"
  - Выбор вакансии: выпадающий список с вакансиями
  - Кнопки управления: Календарь, Вакансия, Свободные слоты, Обновить
  - Настройки встречи (для этапов с `isMeeting = true`):
    - Формат встречи: Онлайн / Офис (отображается если `showOffices = true`)
    - Выбор интервьюеров: чекбоксы для выбора нескольких интервьюеров (отображается если `showInterviewers = true`)
    - Выбор офиса: появляется после выбора формата "Офис", если `showOffices = true`
- **Состояние**:
  - `selectedWorkflow`: выбранный тип процесса ('screening' | string) - 'screening' для скрининга, или ID этапа найма с `isMeeting = true`
  - `selectedVacancy`: выбранная вакансия (string)
  - `interviewFormat`: формат интервью ('online' | 'office')
  - `selectedInterviewers`: массив ID выбранных интервьюеров (string[])
  - `selectedOffice`: выбранный офис ('minsk' | 'warsaw' | 'gomel')
- **Настройки этапов-встреч**:
  - `isMeeting`: метка "встреча" для этапа
  - `showOffices`: отображать ли выбор офисов (да/нет)
  - `showInterviewers`: отображать ли выбор интервьюеров (да/нет)
  - Эти настройки задаются на странице `/company-settings/recruiting/stages` в модальном окне редактирования этапа
- **Используется в**: `/app/workflow/page.tsx`

#### 2. WorkflowChat
- **Файл**: `WorkflowChat.tsx`
- **Назначение**: Компонент чата workflow (скрининг/интервью)
- **Пропсы**: нет
- **Функциональность**: 
  - Чат-интерфейс для управления процессом скрининга и интервью
  - Обработка команд через теги (#add, #hr_screening, #tech_screening, #interview, #delete)
  - Управление кандидатами и инвайтами
  - Отображение истории действий и ответов системы
  - Отображение сообщений чата (пользователь, система, инвайты)
  - Управление файлами: загрузка резюме
  - Управление инвайтами: создание, копирование, пересоздание scorecard
  - Управление отклонениями: выбор причины отказа
  - Скрытие сообщений
  - Автопрокрутка к новым сообщениям
- **Состояние**:
  - `message`: текст сообщения в поле ввода (string)
  - `hiddenMessages`: множество ID скрытых сообщений (Set<string>)
  - `attachedFiles`: массив прикрепленных файлов (AttachedFile[])
  - `currentCommand`: текущая обрабатываемая команда (string | null)
  - `rejectedCandidates`: множество ID отклоненных кандидатов (Set<string>)
  - `rejectionReasons`: Map причин отказа (Map<string, string>)
- **Команды**:
  - `#add`: добавление кандидата из файла резюме
  - `#hr_screening`: HR скрининг (ссылка Huntflow + текст ответов)
  - `#tech_screening`: технический скрининг (ссылка Huntflow + дата-время)
  - `#interview`: создание интервью (ссылка Huntflow + дата-время)
  - `#delete`: отмена последнего действия
- **Используется в**: `/app/workflow/page.tsx`

#### 3. WorkflowSidebar
- **Файл**: `WorkflowSidebar.tsx`
- **Назначение**: Боковая панель страницы workflow
- **Пропсы**: нет
- **Функциональность**: 
  - Отображение отчетов по последним неделям
  - Быстрый доступ к вики
  - Быстрые действия для коммуникации (Telegram, WhatsApp, Viber, LinkedIn, Email)
  - Кнопка "Вики": открытие вики в новой вкладке
  - Отчеты последних недель: раскрывающиеся секции с таблицами этапов
    - Текущая неделя: статистика по этапам за текущую неделю
    - Предыдущая неделя: статистика по этапам за предыдущую неделю
  - Быстрые действия: карточки для быстрого доступа к коммуникации
    - Telegram: несколько аккаунтов
    - WhatsApp: несколько номеров
    - Viber: несколько номеров
    - LinkedIn: несколько профилей
    - Email: несколько адресов
- **Состояние**:
  - `reportsExpanded`: флаг раскрытости секции "Отчеты последних недель" (boolean)
  - `currentWeekExpanded`: флаг раскрытости секции "Текущая неделя" (boolean)
  - `previousWeekExpanded`: флаг раскрытости секции "Предыдущая неделя" (boolean)
- **Используется в**: `/app/workflow/page.tsx`

#### 4. SlotsPanel
- **Файл**: `SlotsPanel.tsx`
- **Назначение**: Панель свободных слотов для интервью
- **Пропсы**: нет
- **Функциональность**: 
  - Отображение свободных слотов для интервью по неделям
  - Управление расписанием интервьюеров
  - Планирование интервью на свободные слоты
  - Текущая неделя: отображение свободных слотов на текущей неделе
  - Следующая неделя: отображение свободных слотов на следующей неделе
  - Раскрытие/сворачивание секций недель
  - Карточки слотов: дата, день недели, список временных слотов, количество слотов
- **Состояние**:
  - `currentWeekExpanded`: флаг раскрытости секции "Текущая неделя" (boolean)
  - `nextWeekExpanded`: флаг раскрытости секции "Следующая неделя" (boolean)
- **Используется в**: `/app/workflow/page.tsx`

## Интерфейсы и типы

### WorkflowType
```typescript
type WorkflowType = 'screening' | string
```
- Типы процессов:
  - `screening`: скрининг (30 минут) - всегда доступен
  - `string`: ID этапа найма с `isMeeting = true` - динамические кнопки на основе этапов найма
    - Количество кнопок: 0 и более (зависит от количества этапов с `isMeeting = true`)
    - Названия кнопок: берутся из названий этапов найма, отмеченных чекбоксом "встреча"

### InterviewFormat
```typescript
type InterviewFormat = 'online' | 'office'
```
- Форматы интервью:
  - `online`: онлайн интервью
  - `office`: офисное интервью

### Office
```typescript
type Office = 'minsk' | 'warsaw' | 'gomel'
```
- Офисы:
  - `minsk`: Минск
  - `warsaw`: Варшава
  - `gomel`: Гомель

### Interviewer
```typescript
interface Interviewer {
  id: string                                    // Уникальный идентификатор интервьюера
  name: string                                 // Имя интервьюера
}
```

### WeekSlot
```typescript
interface WeekSlot {
  date: string                                  // Дата слота (DD.MM.YYYY)
  dayOfWeek: string                            // День недели (сокращенный)
  slots: string[]                              // Массив временных интервалов
  count: number                                // Количество свободных слотов
}
```

### WeekSection
```typescript
interface WeekSection {
  title: string                                // Название секции
  slots: WeekSlot[]                            // Массив слотов на неделю
}
```

### AttachedFile
```typescript
interface AttachedFile {
  id: string                                   // Уникальный идентификатор файла
  name: string                                 // Имя файла
  size: number                                 // Размер файла
  type: string                                 // Тип файла
}
```

## Функции и обработчики

### WorkflowHeader

#### handleInterviewerToggle
- **Назначение**: Обработчик переключения выбора интервьюера
- **Параметры**: `interviewerId: string`
- **Логика**: 
  - Если интервьюер уже выбран - удаляет его из списка
  - Если интервьюер не выбран - добавляет его в список

### WorkflowChat

#### handleSendMessage
- **Назначение**: Обработчик отправки сообщения
- **Логика**: 
  - Обрабатывает команды через теги
  - Отправляет сообщение в чат
  - Показывает ответ системы

#### handleFileUpload
- **Назначение**: Обработчик загрузки файла
- **Логика**: 
  - Обрабатывает файл резюме
  - Добавляет кандидата из резюме
  - Показывает результат обработки

#### handleCommand
- **Назначение**: Обработчик команды
- **Параметры**: `command: string`, `args: string[]`
- **Логика**: 
  - Обрабатывает команды (#add, #hr_screening, #tech_screening, #interview, #delete)
  - Выполняет соответствующие действия
  - Показывает результат выполнения

### WorkflowSidebar

#### handleWikiClick
- **Назначение**: Обработчик клика на кнопку "Вики"
- **Логика**: Открывает страницу вики в новой вкладке

#### handleCurrentWeekClick
- **Назначение**: Обработчик клика на секцию "Текущая неделя"
- **Логика**: 
  - Переключает раскрытость секции
  - Закрывает секцию "Предыдущая неделя" при открытии текущей

#### handlePreviousWeekClick
- **Назначение**: Обработчик клика на секцию "Предыдущая неделя"
- **Логика**: 
  - Переключает раскрытость секции
  - Закрывает секцию "Текущая неделя" при открытии предыдущей

### SlotsPanel

#### handleWeekToggle
- **Назначение**: Обработчик переключения раскрытости недели
- **Параметры**: `week: 'current' | 'next'`
- **Логика**: Переключает раскрытость соответствующей секции

## Особенности реализации

1. **Чат-интерфейс**: WorkflowChat обеспечивает интерактивный чат для управления процессом
2. **Обработка команд**: Команды обрабатываются через теги (#add, #hr_screening и т.д.)
3. **Управление файлами**: Загрузка и обработка резюме кандидатов
4. **Управление инвайтами**: Создание, копирование и пересоздание инвайтов на интервью
5. **Синхронизация с календарем**: SlotsPanel синхронизируется с Google Calendar
6. **Отчеты**: WorkflowSidebar отображает статистику по этапам найма
7. **Быстрые действия**: Быстрый доступ к коммуникации и внешним сервисам
8. **Адаптивность**: Компоненты адаптированы для разных размеров экрана
9. **Автопрокрутка**: Автоматическая прокрутка к новым сообщениям в чате
10. **Скрытие сообщений**: Возможность скрытия сообщений в чате
