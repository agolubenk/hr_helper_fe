# Документация компонентов: Toast

## Общее описание

Директория `components/Toast` содержит систему уведомлений (toast notifications) для приложения. Система обеспечивает централизованное управление уведомлениями различных типов (информационные, предупреждения, ошибки, успех, системные) с поддержкой действий (кнопок) и автоматическим закрытием.

**Расположение**: `@/components/Toast`

## Компоненты

### 1. Toast
- **Файл**: `Toast.tsx`
- **Назначение**: Компонент отдельного уведомления
- **Пропсы**:
  - `id`: уникальный идентификатор уведомления (string)
  - `type`: тип уведомления (`'info' | 'warning' | 'error' | 'success' | 'system'`)
  - `title`: заголовок уведомления (string)
  - `message`: текст сообщения (string)
  - `duration`: продолжительность отображения в миллисекундах (number, опционально, по умолчанию 3000)
  - `onClose`: обработчик закрытия уведомления ((id: string) => void)
  - `actions`: массив действий (кнопок) в уведомлении (опционально)
- **Функциональность**: 
  - Отображает уведомление с иконкой, заголовком, сообщением и кнопками действий
  - Автоматически закрывается через указанное время (duration)
  - Показывает прогресс-бар с анимацией
  - Поддерживает ручное закрытие через кнопку
  - Анимация появления и исчезновения
  - Увеличенная длительность для тостов с действиями (12 секунд)
- **Состояние**:
  - `isExiting`: флаг выхода уведомления (boolean)
- **Особенности**: 
  - Использует разные иконки для разных типов уведомлений
  - Цветовая индикация типа уведомления (левая граница)
  - Прогресс-бар показывает оставшееся время
  - Анимация slideIn/slideOut
  - Hover эффект с поднятием и увеличением тени
  - Кнопки действий закрывают уведомление после клика
  - Предотвращение всплытия событий при клике на кнопки действий
- **Используется в**: 
  - `ToastContext.tsx`: рендерится для каждого активного уведомления

### 2. ToastContext (ToastProvider)
- **Файл**: `ToastContext.tsx`
- **Назначение**: Контекст для управления уведомлениями в приложении
- **Пропсы**:
  - `children`: дочерние компоненты (ReactNode)
- **Функциональность**: 
  - Централизованное управление уведомлениями
  - Отображение различных типов уведомлений
  - Управление жизненным циклом уведомлений (показ, удаление)
  - Поддержка действий в уведомлениях (кнопки)
  - Рендеринг уведомлений через React Portal в body
  - Применение темы к уведомлениям
- **Состояние**:
  - `toasts`: массив всех активных уведомлений (ToastData[])
  - `isMounted`: флаг монтирования компонента (для SSR) (boolean)
- **Особенности**: 
  - Использует React Context для предоставления функций управления уведомлениями
  - Рендерит уведомления через `createPortal` в `document.body` для отображения поверх модальных окон
  - Применяет тему через `Theme` компонент Radix UI
  - Использует акцентный цвет из `ThemeProvider`
  - Генерирует уникальные ID для каждого уведомления
  - Поддерживает SSR (проверка `isMounted` и `document`)
- **Используется в**: 
  - `/app/layout.tsx`: оборачивает все страницы приложения

## Интерфейсы и типы

### ToastType
```typescript
type ToastType = 'info' | 'warning' | 'error' | 'success' | 'system'
```
- Типы уведомлений:
  - `info`: информационное уведомление (синий цвет)
  - `warning`: предупреждающее уведомление (желтый цвет)
  - `error`: уведомление об ошибке (красный цвет)
  - `success`: уведомление об успехе (зеленый цвет)
  - `system`: системное уведомление (серый цвет)

### ToastProps
```typescript
interface ToastProps {
  id: string                                    // Уникальный идентификатор уведомления
  type: ToastType                                // Тип уведомления
  title: string                                  // Заголовок уведомления
  message: string                                // Текст сообщения
  duration?: number                              // Продолжительность отображения в миллисекундах
  onClose: (id: string) => void                // Обработчик закрытия уведомления
  actions?: Array<{                              // Массив действий (кнопок)
    label: string                                // Текст кнопки
    onClick: () => void                          // Обработчик клика
    variant?: 'solid' | 'soft' | 'outline' | 'ghost'  // Вариант кнопки
    color?: 'blue' | 'green' | 'red' | 'yellow' | 'gray'  // Цвет кнопки
  }>
}
```

### ToastAction
```typescript
interface ToastAction {
  label: string                                  // Текст кнопки действия
  onClick: () => void                            // Обработчик клика на кнопку
  variant?: 'solid' | 'soft' | 'outline' | 'ghost'  // Вариант кнопки
  color?: 'blue' | 'green' | 'red' | 'yellow' | 'gray'  // Цвет кнопки
}
```

### ToastData
```typescript
interface ToastData {
  id: string                                    // Уникальный идентификатор уведомления
  type: ToastType                                // Тип уведомления
  title: string                                  // Заголовок уведомления
  message: string                                // Текст сообщения
  duration?: number                              // Продолжительность отображения в миллисекундах
  actions?: ToastAction[]                        // Массив действий (кнопок)
}
```

### ToastContextType
```typescript
interface ToastContextType {
  showToast: (toast: Omit<ToastData, 'id'>) => void  // Базовая функция для показа уведомления
  showInfo: (title: string, message: string, options?: { duration?: number; actions?: ToastAction[] }) => void
  showWarning: (title: string, message: string, options?: { duration?: number; actions?: ToastAction[] }) => void
  showError: (title: string, message: string, options?: { duration?: number; actions?: ToastAction[] }) => void
  showSuccess: (title: string, message: string, options?: { duration?: number; actions?: ToastAction[] }) => void
  showSystem: (title: string, message: string, options?: { duration?: number; actions?: ToastAction[] }) => void
  removeToast: (id: string) => void              // Функция удаления уведомления
}
```

## Функции и обработчики

### Toast компонент

#### handleClose
- **Назначение**: Закрытие уведомления
- **Логика**: 
  1. Устанавливает `isExiting` в true для запуска анимации выхода
  2. Через 300ms вызывает `onClose(id)` для удаления уведомления из массива
- **Особенности**: 
  - Ждет завершения анимации перед удалением
  - Используется при автоматическом закрытии и ручном закрытии

#### getIcon
- **Назначение**: Возвращает иконку в зависимости от типа уведомления
- **Возвращает**: React элемент с иконкой
- **Логика**: 
  - `info`: `InfoCircledIcon`
  - `warning`: `ExclamationTriangleIcon`
  - `error`: `CrossCircledIcon`
  - `success`: `CheckCircledIcon`
  - `system`: `GearIcon`
  - По умолчанию: `InfoCircledIcon`
- **Особенности**: 
  - Размер иконки: 20x20px

#### getToastClassName
- **Назначение**: Возвращает CSS классы для уведомления
- **Возвращает**: строка с классами
- **Логика**: 
  - Базовый класс: `styles.toast`
  - Класс типа: `styles.toast{Type}` (например, `toastInfo`)
  - Класс выхода: `styles.toastExiting` (если `isExiting`)
- **Особенности**: 
  - Динамическое формирование имени класса типа

#### getIconClassName
- **Назначение**: Возвращает CSS класс для иконки
- **Возвращает**: строка с классом
- **Логика**: 
  - Класс типа: `styles.toastIcon{Type}` (например, `toastIconInfo`)

#### getProgressBarClassName
- **Назначение**: Возвращает CSS класс для прогресс-бара
- **Возвращает**: строка с классом
- **Логика**: 
  - Класс типа: `styles.toastProgressBar{Type}` (например, `toastProgressBarInfo`)

### ToastContext компонент

#### removeToast
- **Назначение**: Удаление уведомления из массива
- **Параметры**: `id: string` - ID уведомления для удаления
- **Логика**: 
  - Фильтрует массив `toasts`, удаляя уведомление с указанным ID
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Особенности**: 
  - Вызывается при закрытии уведомления (вручную или автоматически)

#### showToast
- **Назначение**: Базовая функция для показа уведомления
- **Параметры**: `toast: Omit<ToastData, 'id'>` - данные уведомления (без ID)
- **Логика**: 
  1. Генерирует уникальный ID на основе времени и случайной строки
  2. Создает новый объект `ToastData` с ID
  3. Добавляет уведомление в конец массива `toasts`
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Особенности**: 
  - ID формат: `toast-{timestamp}-{randomString}`

#### showInfo
- **Назначение**: Показ информационного уведомления
- **Параметры**: 
  - `title: string` - заголовок уведомления
  - `message: string` - текст сообщения
  - `options?: { duration?: number; actions?: ToastAction[] }` - опциональные параметры
- **Логика**: Вызывает `showToast` с типом `'info'`
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `showToast`

#### showWarning

- **Логика**: Вызывает `showToast` с типом `'warning'`
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `showToast`

#### showError

### useToast
- **Назначение**: Хук для доступа к контексту уведомлений
- **Возвращает**: `ToastContextType` - контекст уведомлений
- **Логика**: 
  1. Получает контекст через `useContext(ToastContext)`
  2. Если контекст не найден - выбрасывает ошибку
- **Особенности**: 
  - Должен использоваться внутри `ToastProvider`
  - Выбрасывает ошибку, если используется вне провайдера

## Хуки и эффекты

### Toast компонент

#### useEffect: автоматическое закрытие
- **Назначение**: Автоматически закрывает уведомление через указанное время
- **Логика**: 
  1. Устанавливает таймер на `effectiveDuration`
  2. По истечении времени вызывает `handleClose`
  3. Очищает таймер при размонтировании
- **Зависимости**: `effectiveDuration`
- **Особенности**: 
  - `effectiveDuration` зависит от наличия действий (12 секунд для тостов с кнопками, иначе `duration`)

### ToastContext компонент

#### useEffect: проверка монтирования
- **Назначение**: Проверяет, что компонент смонтирован (для SSR)
- **Логика**: 
  1. Устанавливает `isMounted` в true после монтирования
  2. Используется для проверки перед рендерингом портала
- **Зависимости**: нет
- **Особенности**: 
  - Предотвращает ошибки SSR при использовании `document`

## Вычисляемые значения

### effectiveDuration (Toast)
- **Назначение**: Эффективная длительность отображения уведомления
- **Вычисление**: 
  - Если есть действия (`actions && actions.length > 0`): `MAX_DURATION_FOR_ACTION_TOASTS_MS` (12000ms)
  - Иначе: `duration ?? 3000`
- **Особенности**: 
  - Тосты с кнопками действий показываются дольше (12 секунд)

### currentAccentColor (ToastContext)
- **Назначение**: Акцентный цвет в зависимости от темы
- **Вычисление**: 
  - Если тема `'dark'`: `darkThemeAccentColor`
  - Иначе: `lightThemeAccentColor`
- **Особенности**: 
  - Используется для применения к `Theme` компоненту Radix UI

## Константы

### MAX_DURATION_FOR_ACTION_TOASTS_MS
- **Значение**: `12000` (12 секунд)
- **Назначение**: Максимальная длительность для тостов с кнопками действий
- **Использование**: Применяется к тостам, которые содержат действия (кнопки)

## Использование

### Пример использования ToastProvider в layout:
```tsx
import { ToastProvider } from '@/components/Toast/ToastContext'

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <ThemeProvider>
          <ToastProvider>
            {children}
          </ToastProvider>
        </ThemeProvider>
      </body>
    </html>
  )
}
```

### Пример использования useToast в компоненте:
```tsx
import { useToast } from '@/components/Toast/ToastContext'

function MyComponent() {
  const { showSuccess, showError, showInfo } = useToast()
  
  const handleSave = async () => {
    try {
      await saveData()
      showSuccess('Успешно', 'Данные сохранены')
    } catch (error) {
      showError('Ошибка', 'Не удалось сохранить данные')
    }
  }
  
  const handleInfo = () => {
    showInfo('Информация', 'Это информационное сообщение', {
      duration: 5000,
      actions: [
        {
          label: 'Подробнее',
          onClick: () => console.log('Подробнее'),
          variant: 'solid',
          color: 'blue'
        }
      ]
    })
  }
  
  return (
    <div>
      <button onClick={handleSave}>Сохранить</button>
      <button onClick={handleInfo}>Показать информацию</button>
    </div>
  )
}
```

### Пример с действиями:
```tsx
const { showWarning } = useToast()

showWarning('Подтверждение', 'Вы уверены, что хотите удалить?', {
  duration: 10000,
  actions: [
    {
      label: 'Отмена',
      onClick: () => console.log('Отменено'),
      variant: 'soft',
      color: 'gray'
    },
    {
      label: 'Удалить',
      onClick: () => {
        deleteItem()
        showSuccess('Удалено', 'Элемент успешно удален')
      },
      variant: 'solid',
      color: 'red'
    }
  ]
})
```

## Связи с другими компонентами

- **ToastProvider** используется в `/app/layout.tsx` - оборачивает все страницы приложения
- **useToast** используется во всех компонентах приложения для показа уведомлений
- Использует `Box`, `Text`, `Button`, `Theme` из `@radix-ui/themes`
- Использует иконки из `@radix-ui/react-icons` (InfoCircledIcon, ExclamationTriangleIcon, CrossCircledIcon, CheckCircledIcon, GearIcon, Cross2Icon)
- Использует `useTheme` из `@/components/ThemeProvider` для получения темы и акцентных цветов
- Использует `createPortal` из React для рендеринга в body
- Связан с React Context API для предоставления функций управления уведомлениями

## Особенности реализации

1. **React Portal**: Уведомления рендерятся через портал в `document.body` для отображения поверх модальных окон
2. **SSR поддержка**: Проверка `isMounted` и `document` для корректной работы на сервере
3. **Тема**: Применение темы через `Theme` компонент Radix UI с акцентными цветами
4. **Уникальные ID**: Генерация уникальных ID на основе времени и случайной строки
5. **Автоматическое закрытие**: Таймер для автоматического закрытия с учетом наличия действий
6. **Анимации**: Плавные анимации появления и исчезновения (slideIn/slideOut)
7. **Прогресс-бар**: Визуальная индикация оставшегося времени
8. **Действия**: Поддержка кнопок действий с различными вариантами и цветами
9. **Предотвращение всплытия**: `stopPropagation` при клике на кнопки действий
10. **Адаптивность**: Мобильные стили для корректного отображения на маленьких экранах
11. **Z-index**: Высокий z-index (10000) для отображения поверх всех элементов
12. **Backdrop filter**: Размытие фона для современного вида
13. **Hover эффекты**: Интерактивность при наведении мыши
14. **Цветовая индикация**: Левая граница и иконка соответствуют типу уведомления
15. **Мемоизация**: Использование `useCallback` для оптимизации производительности
