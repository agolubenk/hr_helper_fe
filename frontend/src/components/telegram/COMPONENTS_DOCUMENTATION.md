# Документация компонентов: telegram

## Общее описание

Директория `components/telegram` содержит компонент для ввода текста с визуальным отображением форматирования. Компонент обеспечивает rich text ввод с поддержкой markdown-форматирования, горячих клавиш и вставки ссылок.

**Расположение**: `@/components/telegram`

## Компоненты

### 1. RichTextInput
- **Файл**: `RichTextInput.tsx`
- **Назначение**: Компонент для ввода текста с визуальным отображением форматирования
- **Пропсы**:
  - `value`: текст в markdown формате (string)
  - `onChange`: обработчик изменения текста ((value: string) => void)
  - `onKeyDown`: обработчик нажатий клавиш ((e: React.KeyboardEvent<HTMLDivElement>) => void, опционально)
  - `placeholder`: текст плейсхолдера (string, по умолчанию 'Введите сообщение...')
  - `style`: дополнительные стили (React.CSSProperties, опционально)
  - `rows`: количество строк (number, по умолчанию 1)
- **Функциональность**: 
  - Отображает markdown-форматирование визуально (жирный, курсив, подчеркнутый, зачеркнутый, ссылки, цитаты)
  - Поддерживает горячие клавиши для форматирования (Ctrl+B, Ctrl+I, Ctrl+U, Ctrl+Shift+X, Ctrl+Shift+., Cmd+K)
  - Хранит текст в markdown-формате для отправки
  - Поддерживает многострочный ввод (Shift+Enter для новой строки)
  - Вставка ссылок через тултип с умным добавлением текста из контекста
  - Поддержка IME ввода (composition events)
  - Сохранение и восстановление позиции курсора при обновлении
- **Поддерживаемые форматы**:
  - `**текст**` → жирный (`<strong>`)
  - `*текст*` → курсив (`<em>`)
  - `***текст***` → жирный+курсив (`<strong><em>`)
  - `<u>текст</u>` → подчеркнутый (`<u>`)
  - `~~текст~~` → зачеркнутый (`<del>`)
  - `[текст](url)` → ссылка (`<a>`)
  - `> текст` → цитата (`<blockquote>`)
- **Горячие клавиши**:
  - `Ctrl+B` (Mac: `Cmd+B`): жирный текст (переключение)
  - `Ctrl+I` (Mac: `Cmd+I`): курсив (переключение)
  - `Ctrl+U` (Mac: `Cmd+U`): подчеркнутый (переключение)
  - `Ctrl+Shift+X` (Mac: `Cmd+Shift+X`): зачеркнутый (переключение)
  - `Ctrl+Shift+.` (Mac: `Cmd+Shift+.`): цитата (переключение)
  - `Ctrl+K` (Mac: `Cmd+K`): вставка ссылки
- **Состояние**:
  - `isComposing`: флаг активной композиции IME (boolean)
  - `isLinkTooltipOpen`: флаг открытости тултипа ссылки (boolean)
  - `linkUrl`: URL ссылки (string)
  - `linkText`: текст ссылки (string)
  - `tooltipPosition`: позиция тултипа ({ top: number, left: number })
- **Особенности**: 
  - Использует `contentEditable` для редактирования
  - Двусторонняя конвертация между markdown и HTML
  - Умное сохранение и восстановление позиции курсора
  - Обработка композиции IME для корректной работы с восточными языками
  - Тултип для вставки ссылок с кнопками добавления текста из контекста
  - Автоматическое удаление `https://` и `https://www.` из URL при вставке
- **Используется в**: 
  - `/app/telegram/chats/page.tsx`: используется в чате Telegram для ввода сообщений

## Интерфейсы и типы

### RichTextInputProps
```typescript
interface RichTextInputProps {
  value: string                                    // Текст в markdown формате
  onChange: (value: string) => void              // Обработчик изменения текста
  onKeyDown?: (e: React.KeyboardEvent<HTMLDivElement>) => void  // Обработчик нажатий клавиш
  placeholder?: string                            // Текст плейсхолдера
  style?: React.CSSProperties                     // Дополнительные стили
  rows?: number                                    // Количество строк
}
```

### linkContextRef (внутренний тип)
```typescript
{
  leftText: string                                 // Текст слева от выделения
  rightText: string                                 // Текст справа от выделения
  leftIndex: number                                // Индекс начала текста ссылки слева
  rightIndex: number                               // Индекс начала текста ссылки справа
  originalSelectedText: string                     // Исходный выделенный текст
  originalLeftText: string                         // Исходный текст слева
  originalRightText: string                        // Исходный текст справа
}
```

## Функции и обработчики

### convertMarkdownToHTML
- **Назначение**: Конвертирует markdown в HTML для визуального отображения
- **Параметры**: `text: string` - текст в markdown формате
- **Возвращает**: HTML строка
- **Логика**: 
  1. Разбивает текст на строки для обработки цитат
  2. Обрабатывает цитаты (строки, начинающиеся с `> `)
  3. Обрабатывает форматирование внутри цитат
  4. Обрабатывает обычные строки:
     - Экранирует HTML символы (`&`, `<`, `>`)
     - Обрабатывает зачеркнутый (`~~текст~~`)
     - Обрабатывает жирный (`**текст**`)
     - Обрабатывает жирный+курсив (`***текст***`)
     - Обрабатывает курсив (`*текст*`, но не `**текст**` или `***текст***`)
     - Обрабатывает подчеркнутый (`<u>текст</u>`)
     - Обрабатывает ссылки (`[текст](url)`)
  5. Объединяет строки, сохраняя переносы через `<br>`
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Особенности**: 
  - Использует регулярные выражения для парсинга markdown
  - Обрабатывает вложенные форматы (жирный+курсив)
  - Сохраняет переносы строк через `<br>`
  - Правильно обрабатывает пустые строки

### convertHTMLToMarkdown
- **Назначение**: Конвертирует HTML обратно в markdown
- **Параметры**: `html: string` - HTML строка
- **Возвращает**: текст в markdown формате
- **Логика**: 
  1. Создает временный DOM элемент для парсинга HTML
  2. Рекурсивно обрабатывает узлы через `nodeToMarkdown`:
     - Текстовые узлы: возвращает текст
     - Элементы: обрабатывает теги и их содержимое
  3. Обрабатывает теги:
     - `strong`, `b`: `**текст**`
     - `em`, `i`: `*текст*` (проверяет, не внутри ли `strong`)
     - `u`: `<u>текст</u>`
     - `del`, `s`: `~~текст~~`
     - `a`: `[текст](url)` (убирает протокол из URL)
     - `blockquote`: `> текст` (разбивает на строки)
     - `br`: `\n`
     - `div`, `p`: добавляет перенос строки
  4. Обрабатывает переносы строк между узлами
  5. Очищает лишние переносы (но сохраняет одиночные)
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Особенности**: 
  - Рекурсивная обработка вложенных тегов
  - Правильная обработка комбинаций форматов (жирный+курсив)
  - Сохранение переносов строк
  - Очистка от множественных переносов

### handleInput
- **Назначение**: Обработка ввода текста
- **Параметры**: `e: React.FormEvent<HTMLDivElement>`
- **Логика**: 
  1. Устанавливает флаг активного ввода (`isUserTypingRef.current = true`)
  2. Использует `requestAnimationFrame` для отложенной обработки
  3. Сохраняет текущую позицию курсора
  4. Проверяет, пустой ли редактор (только `<br>` или пусто)
  5. Если пустой - устанавливает `<br>` для правильной работы плейсхолдера
  6. Конвертирует HTML в markdown
  7. Обновляет значение только если markdown изменился
  8. Сбрасывает флаг активного ввода через 200ms
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `onChange`, `convertHTMLToMarkdown`, `value`, `isComposing`
- **Особенности**: 
  - Не обновляет, если идет композиция IME или обновление извне
  - Сохраняет позицию курсора при обновлении

### saveSelection
- **Назначение**: Сохраняет текущее выделение текста
- **Логика**: 
  - Получает выделение из `window.getSelection()`
  - Клонирует Range и сохраняет в `savedSelectionRef.current`
- **useCallback**: Мемоизирует функцию для оптимизации производительности

### restoreSelection
- **Назначение**: Восстанавливает сохраненное выделение
- **Логика**: 
  - Проверяет наличие сохраненного выделения
  - Восстанавливает выделение через `selection.addRange()`
- **useCallback**: Мемоизирует функцию для оптимизации производительности

### applyQuote
- **Назначение**: Применяет или снимает цитату к выделенному тексту
- **Логика**: 
  - Если есть выделенный текст:
    - Проверяет, все ли строки уже в цитате (начинаются с `> `)
    - Если все в цитате - снимает цитату (убирает `> `)
    - Если не все в цитате - применяет цитату (добавляет `> `)
  - Если нет выделения:
    - Проверяет текущую строку
    - Если начинается с `> ` - убирает цитату
    - Если не начинается - добавляет цитату
  - Обновляет markdown через `requestAnimationFrame`
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `onChange`, `convertHTMLToMarkdown`

### openLinkTooltip
- **Назначение**: Открывает тултип для вставки ссылки
- **Логика**: 
  1. Сохраняет текущее выделение
  2. Получает весь текст редактора
  3. Находит позицию выделенного текста в общем тексте через TreeWalker
  4. Находит границы строки (до переноса строки)
  5. Извлекает текст строки и разделяет на левую и правую части
  6. Сохраняет контекст в `linkContextRef.current`
  7. Устанавливает текст ссылки из выделенного текста
  8. Вычисляет позицию тултипа (центр выделенного текста)
  9. Открывает тултип
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `saveSelection`

### insertLink
- **Назначение**: Вставляет ссылку в markdown формате
- **Логика**: 
  1. Восстанавливает выделение
  2. Обрабатывает URL: убирает `https://` или `https://www.` если они есть
  3. Формирует markdown ссылку: `[текст](https://url)`
  4. Вычисляет, какой текст был добавлен через кнопки +/-
  5. Если был добавлен текст - расширяет выделение и удаляет добавленный текст
  6. Вставляет ссылку как текстовый узел (markdown формат)
  7. Перемещает курсор после вставленной ссылки
  8. Обновляет markdown через `requestAnimationFrame`
  9. Закрывает тултип и очищает состояние
  10. Возвращает фокус в редактор
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `linkUrl`, `linkText`, `restoreSelection`, `onChange`, `convertHTMLToMarkdown`, `value`

### addCharLeft
- **Назначение**: Добавляет текст слева от текста ссылки из контекста строки
- **Логика**: 
  1. Проверяет наличие контекста и текста слева
  2. Находит следующее слово/символ слева:
     - Пропускает пробелы
     - Если знак препинания - берет его
     - Иначе ищет начало слова
  3. Добавляет найденный текст к `linkText`
  4. Обновляет `leftIndex` в контексте
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Особенности**: 
  - Умное добавление текста по словам и знакам препинания
  - Кнопка отключается, если нет больше текста слева

### addCharRight
- **Назначение**: Добавляет текст справа от текста ссылки из контекста строки
- **Логика**: 
  1. Проверяет наличие контекста и текста справа
  2. Находит следующее слово/символ справа:
     - Пропускает пробелы
     - Если знак препинания - берет его
     - Иначе ищет конец слова
  3. Добавляет найденный текст к `linkText`
  4. Обновляет `rightIndex` в контексте
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Особенности**: 
  - Умное добавление текста по словам и знакам препинания
  - Кнопка отключается, если нет больше текста справа

### isFormatApplied
- **Назначение**: Проверяет, применено ли форматирование к выделенному тексту
- **Параметры**: `formatType: 'bold' | 'italic' | 'underline' | 'strikethrough'`
- **Возвращает**: `boolean` - true если форматирование применено
- **Логика**: 
  1. Фокусирует editor для корректной работы `queryCommandState`
  2. Использует встроенные команды браузера (`document.queryCommandState`)
  3. Если не поддерживается - использует fallback проверку через DOM:
     - Проверяет тег элемента или наличие родителя с нужным тегом
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Особенности**: 
  - Использует встроенные возможности браузера для более точной проверки
  - Fallback на проверку через DOM если не поддерживается

### handleKeyDown
- **Назначение**: Обработка горячих клавиш для форматирования
- **Параметры**: `e: React.KeyboardEvent<HTMLDivElement>`
- **Логика**: 
  1. Определяет платформу (Mac или Windows/Linux)
  2. Определяет модификатор (Cmd для Mac, Ctrl для Windows/Linux)
  3. Обрабатывает горячие клавиши:
     - `Ctrl+B` / `Cmd+B`: жирный (переключение через `isFormatApplied`)
     - `Ctrl+I` / `Cmd+I`: курсив (переключение)
     - `Ctrl+U` / `Cmd+U`: подчеркнутый (переключение)
     - `Ctrl+Shift+X` / `Cmd+Shift+X`: зачеркнутый (переключение)
     - `Ctrl+Shift+.` / `Cmd+Shift+.`: цитата (через `applyQuote`)
     - `Ctrl+K` / `Cmd+K`: вставка ссылки (через `openLinkTooltip`)
  4. Использует `e.code` вместо `e.key` для работы независимо от раскладки клавиатуры
  5. Вызывает внешний обработчик `onKeyDown` для других клавиш
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `onChange`, `convertHTMLToMarkdown`, `openLinkTooltip`, `applyQuote`, `isFormatApplied`
- **Особенности**: 
  - Использует `document.execCommand` для применения форматирования
  - Переключение: повторное нажатие снимает форматирование
  - Обновляет markdown через `requestAnimationFrame` после применения форматирования

### handleCompositionStart
- **Назначение**: Обработка начала композиции IME
- **Логика**: Устанавливает `isComposing` в true
- **useCallback**: Мемоизирует функцию для оптимизации производительности

### handleCompositionEnd
- **Назначение**: Обработка окончания композиции IME
- **Параметры**: `e: React.CompositionEvent<HTMLDivElement>`
- **Логика**: 
  1. Устанавливает `isComposing` в false
  2. Вызывает `handleInput` для обработки введенного текста
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `handleInput`

## Хуки и эффекты

### useEffect: синхронизация содержимого editor с value
- **Назначение**: Синхронизирует содержимое contentEditable с внешним value
- **Логика**: 
  1. Не синхронизирует, если пользователь активно вводит текст или идет композиция
  2. Сравнивает текущий HTML с ожидаемым HTML из value
  3. Если отличается:
     - Сохраняет позицию курсора через TreeWalker
     - Обновляет HTML
     - Если пустой - добавляет `<br>` для правильной работы плейсхолдера
     - Восстанавливает курсор на основе сохраненной позиции через `requestAnimationFrame`
- **Зависимости**: `value`, `convertMarkdownToHTML`, `isComposing`
- **Особенности**: 
  - Использует `isUpdatingRef` для предотвращения циклов обновления
  - Использует `isUserTypingRef` для предотвращения обновления во время ввода
  - Точное сохранение и восстановление позиции курсора

### useEffect: обработка клика вне тултипа
- **Назначение**: Закрывает тултип ссылки при клике вне его
- **Логика**: 
  1. Выполняется только если тултип открыт
  2. Проверяет, был ли клик вне тултипа и редактора
  3. Если да - закрывает тултип и очищает состояние
- **Зависимости**: `isLinkTooltipOpen`

## Вычисляемые значения

### tooltipPosition
- **Назначение**: Позиция тултипа для вставки ссылки
- **Вычисление**: Центр выделенного текста относительно редактора
- **Логика**: 
  - `top`: `rect.top - editorRect.top + rect.height / 2 + scrollTop`
  - `left`: `rect.left - editorRect.left + rect.width / 2 + scrollLeft`

## Стили

### RichTextInput.module.css

#### .richTextInput
- Основной класс для contentEditable элемента
- `word-wrap: break-word` - перенос длинных слов
- `white-space: pre-wrap` - сохранение пробелов и переносов

#### .richTextInput[data-placeholder]:empty:before
- Плейсхолдер для пустого редактора
- Использует `data-placeholder` атрибут

#### .richTextInput[data-placeholder]:not(:focus):has(> br:only-child):before
- Плейсхолдер когда есть только `<br>`
- Показывается только когда редактор не в фокусе

#### .richTextInput:focus
- Стили для фокуса
- Изменяет цвет границы и добавляет тень

#### .richTextInput strong
- Стили для жирного текста
- `font-weight: 600`

#### .richTextInput em
- Стили для курсива
- `font-style: italic`

#### .richTextInput u
- Стили для подчеркнутого текста
- `text-decoration: underline`

#### .richTextInput del
- Стили для зачеркнутого текста
- `text-decoration: line-through`

#### .richTextInput a
- Стили для ссылок
- Цвет акцента, подчеркивание, курсор pointer
- При наведении меняет цвет

#### .richTextInput blockquote
- Стили для цитат
- Отступ слева, фон, скругление углов
- Курсивный шрифт

## Использование

### Пример использования RichTextInput:
```tsx
import RichTextInput from '@/components/telegram/RichTextInput'

function TelegramChat() {
  const [message, setMessage] = useState('')
  
  const handleSend = () => {
    // Отправка сообщения в markdown формате
    sendMessage(message)
    setMessage('')
  }
  
  return (
    <RichTextInput 
      value={message}
      onChange={setMessage}
      placeholder="Введите сообщение..."
      onKeyDown={(e) => {
        if (e.key === 'Enter' && !e.shiftKey) {
          e.preventDefault()
          handleSend()
        }
      }}
      rows={3}
    />
  )
}
```

## Особенности реализации

1. **Двусторонняя конвертация**: Markdown ↔ HTML для визуального отображения
2. **Сохранение курсора**: Точное сохранение и восстановление позиции курсора при обновлении
3. **IME поддержка**: Обработка composition events для корректной работы с восточными языками
4. **Умная вставка ссылок**: Кнопки +/- для добавления текста из контекста строки
5. **Независимость от раскладки**: Использование `e.code` вместо `e.key` для горячих клавиш
6. **Переключение форматирования**: Повторное нажатие горячей клавиши снимает форматирование
7. **Обработка цитат**: Умное применение/снятие цитат к строкам
8. **Автоматическая очистка URL**: Удаление `https://` и `https://www.` при вставке ссылок
9. **Плейсхолдер**: Поддержка плейсхолдера для пустого редактора и редактора только с `<br>`
10. **Обработка пустых строк**: Правильная обработка пустых строк и переносов
