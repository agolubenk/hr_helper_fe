# Документирование страниц и компонентов (fe_migration)

**Связь с планом:** **`Детальный план остаточной миграции hr_helper_fe.md`** (корень репо) — **фаза 15**; журнал задач — **`MIGRATION_PLAN_UPD.md`** (корень репо) §5; статус фаз — **`DETAILED_PLAN_STATUS.md`**. Источник данных (моки до API, фаза 13) — **`MOCK_SERVICES.md`**.

## Зачем

- Быстрый онбординг: маршрут → точка входа → зависимости (моки, API, стор).
- Связь «экран ↔ файлы» без обхода всего `src/`.
- Единый формат рядом с уже существующими `COMPONENTS_DOCUMENTATION.md` в модулях.

## Где что лежит

| Тип | Расположение |
|-----|----------------|
| Страницы (роуты) | `fe_migration/src/app/pages/*.tsx` и вложенные по доменам |
| Фичи (крупные) | `fe_migration/src/features/<name>/` |
| Общие компоненты | `fe_migration/src/components/<area>/` |
| Примеры подробных описаний модулей | `**/COMPONENTS_DOCUMENTATION.md` (wiki, vacancies, workflow, …) |

## Формат записи по странице / фиче

Для **ключевых** маршрутов из `MIGRATION_CHECK_URLS.md` и разделов из smoke-приоритета (workflow, wiki, company-settings, ATS, профиль) завести файл **`docs/pages/<route-segment>.md`** или один накопительный **`docs/pages/INDEX.md`** с подразделами.

**Минимальный чеклист в карточке страницы:**

1. **Маршрут** (path) и **имя в меню** (`sidebar.md`).
2. **Точка входа:** `*Page.tsx` (путь от `src/`).
3. **Layout / оболочка:** `AppLayout`, вложенные layout при наличии.
4. **Ключевые компоненты:** 3–10 импортов, не дублировать полный список.
5. **Данные:** моки / `import.meta.env` / API (`src/lib/api.ts` и т.д.).
6. **Особенности:** скролл, модалки, права (когда появится фаза 12).
7. **Ссылка** на модульный `COMPONENTS_DOCUMENTATION.md`, если есть.

## Приоритет заполнения

1. Workflow, ATS, company-settings (крупные формы).
2. Профиль (вкладки).
3. Вакансии (модалки — см. `VACANCIES_MODALS_AUDIT.md`).
4. Остальное по мере доработки.
