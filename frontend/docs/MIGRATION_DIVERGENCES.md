# Расхождения fe_migration ↔ эталон (Next / `frontend old`)

**Правило команды:** намеренные изменения из этой ветки **не откатывать**. Если при сравнении с **3001** (или с `frontend old`) что-то **не совпадает** — **не «чинить под эталон» автоматически**: занести сюда строку (раздел, кратко «было / стало / файл»). Исправление под эталон — только по явному запросу в задаче.

**Связанные документы:** мастер-план — **`Детальный план остаточной миграции hr_helper_fe.md`** (корень репо); статус фаз — **`DETAILED_PLAN_STATUS.md`**; очередь smoke — **`MIGRATION_PLAN_UPD.md`** (корень репо). Обзор UI-профиля и вики — `docs/UI_MIGRATION_PLAN.md`; детали вики — `src/components/wiki/COMPONENTS_DOCUMENTATION.md`; чек-URL — `docs/MIGRATION_CHECK_URLS.md`.

**Глобальные исключения (не трогать в рамках текущего плана):** `Sidebar`, `Header`, `Footer`, `FloatingActions`. Для `FloatingActions` зафиксирован `left: 12px`.

## 7. Профиль, футер, тема, плавающая панель

Сводка из **`docs/UI_MIGRATION_PLAN.md`** (не дублировать полностью — при расхождении с 3001 дополнять §9):

- Профиль: ФИО одной строкой, мок телефона, блок «Активная среда» убран, email вне блока соцсетей.
- **ProfileEditForm (вкладка «Редактирование»):** в `frontend old` в одной форме были **расписание** (общее/по дням) и **интервал между встречами**. В `fe_migration` этот блок **намеренно вынесен** на вкладку «Расписание» (`ScheduleSettingsPage`); в форме редактирования остаются контакты и соцсети. **Не дублировать** расписание в `ProfileEditForm` без отдельной продуктовой задачи (детальный план §3.2 vs §3.8).
- UserCard: убрана строка Telegram в карточке; аватар/заглушка с рамкой и тенью.
- Соцсети: лимит **3 аккаунта на платформу**, тултипы при ≥2 иконках, копирование в углу иконки.
- Тема: Авто / Свет / Тёмная + акценты, `ThemeProvider`.
- Футер: фиксированный, трей задач, связка с `QuickTasksSettings`.
- **FloatingActions**: выравнивание с основным frontend (позиции, стекло, порядок кнопок, `quickButtonsStorage`), текущая позиция `left: 12px`.

## 12. Настройки компании (фаза 8) — структурная сверка (март 2026)

### 12.1 `components/company-settings/`

Каталоги `frontend old/components/company-settings` и `fe_migration/src/components/company-settings` совпадают по **набору из 26 файлов**. По `diff -rq` расходятся только четыре `.tsx` — **ожидаемые** отличия миграции (`Link` / `useRouter` из `@/router-adapter`, сигнатура `toast.showError` в `RatingScalesSettings.tsx`). Подробная таблица — **`COMPANY_SETTINGS_PHASE8.md` §1**.

### 12.2 Экран `/company-settings/integrations`

Карточки, фильтр по группам, **`IntegrationScopeModal`**, **`GoogleServicesModal`** и localStorage вынесены в **`fe_migration/src/components/company-settings/integrations/`** (паритет с `frontend old copy/app/company-settings/integrations/page.tsx`). **Нужна более глубокая доработка:** сценарии с реальным API, согласование ключей/состояний с бэкендом, финальный UX-полиш и smoke — очередь в **`COMPANY_SETTINGS_FOLLOWUP_2026-03-24.md`** (отдельно от мастер-плана и без учёта прочих стилевых задач).

### 12.3 Экран `/company-settings/user-groups`

Перенос с эталона **`frontend old copy/app/company-settings/user-groups/page.tsx`**: **`UserGroupsSettings`** + **`UserGroupsSettings.module.css`** — загрузка/таблица, поиск, форма добавления, inline-редактирование, **`GroupAccessModal`**, toasts, мок-данные. Дальнейшая работа — подключение API по контракту (фаза 13).
