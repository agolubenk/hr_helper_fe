## Реализация (текущая)

- Роли и матрица: `src/app/admin/roles.ts`.
- Хук доступа: `src/app/admin/useRoleAccess.ts` (берёт роль из API профиля при `VITE_USE_PROFILE_API=true`, fallback — `localStorage['adminRole']`).
- Guard route: `AdminLayoutShell` редиректит на `/errors/403` при запрете.
- Sidebar и `AdminPage`: скрывают недоступные пункты.
- `AdminUsersPage` / `AdminGroupsPage`: показывают текущую роль и права действий (create/edit/remove).
