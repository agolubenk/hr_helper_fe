# Документация страницы: Пользователи (company-settings/users/page.tsx)

## Общее описание

Страница управления пользователями компании. Позволяет добавлять, редактировать, удалять пользователей, управлять их правами доступа и группами.

## Компоненты страницы

### 1. UserAccessModal
- **Расположение**: `@/components/company-settings/UserAccessModal`
- **Назначение**: Модальное окно управления правами доступа пользователя
- **Пропсы**:
  - `isOpen`: флаг открытости
  - `onClose`: обработчик закрытия
  - `userId`: ID пользователя
  - `userName`: имя пользователя
  - `currentAccess`: текущие права доступа
  - `onSave`: обработчик сохранения прав
- **Логика**: 
  - Отображение текущих прав доступа
  - Редактирование прав доступа
  - Сохранение через API
- **Связи**: Открывается при клике на кнопку "Права доступа"

### 2. Таблица пользователей
- **Компонент**: `Table` из Radix UI
- **Назначение**: Отображение списка всех пользователей
- **Колонки**:
  - Пользователь (аватар, имя, email, должность)
  - Группы (Badge для каждой группы)
  - Статус (активен/неактивен)
  - Последний вход (дата или "Никогда")
  - Действия (редактирование, удаление, права доступа)
- **Логика**: 
  - Фильтрация по `searchTerm`
  - Пагинация (10 элементов на страницу)
  - Обработка пустого состояния

### 3. Форма добавления пользователя
- **Компонент**: Форма с полями
- **Назначение**: Создание нового пользователя
- **Поля**:
  - Email (обязательное)
  - Имя (обязательное)
  - Фамилия (обязательное)
  - Должность
  - Группы (множественный выбор)
  - Статус (активен/неактивен)
- **Логика**: 
  - Валидация обязательных полей
  - Отправка данных на сервер
  - Обновление списка после создания

### 4. Форма редактирования пользователя
- **Компонент**: Форма с полями (аналогична форме добавления)
- **Назначение**: Редактирование существующего пользователя
- **Логика**: 
  - Заполнение полей текущими данными пользователя
  - Валидация обязательных полей

## Состояние компонента

### State переменные:
- `users`: массив всех пользователей
- `loading`: флаг загрузки данных
- `searchTerm`: поисковый запрос
- `editingUser`: пользователь для редактирования (null если не редактируется)
- `showAddForm`: флаг отображения формы добавления
- `saving`: флаг сохранения (для индикатора)
- `availableGroups`: массив доступных групп
- `accessModalOpen`: флаг открытия модального окна прав доступа
- `selectedUserForAccess`: пользователь для управления правами

  - Обновляет состояние
- **Вызывается**: 
  - При монтировании компонента (useEffect)
  - После создания/редактирования/удаления пользователя

### handleDeleteUser(id)
- **Назначение**: Удаление пользователя
- **Логика**: 
  1. Показывает подтверждение через toast
  2. Вызывает `performDeleteUser(id)`
- **Поведение**: Вызывается при клике на кнопку удаления

### performDeleteUser(id)
- **Назначение**: Выполнение удаления пользователя
- **Логика**: 
  1. Отправляет запрос на удаление через API
  2. При успехе обновляет список пользователей
  3. При ошибке показывает toast
- **TODO**: Реализовать реальный API вызов

### filteredUsers
- **Назначение**: Отфильтрованный список пользователей
- **Логика**: 
  - Фильтрует по `searchTerm` (email, имя, фамилия, должность)
  - Без учета регистра
- **Используется**: Для отображения в таблице

### formatDate(dateString)
- **Назначение**: Форматирование даты для отображения
- **Логика**: Преобразует ISO строку в формат DD.MM.YYYY HH:MM
- **Используется**: Для отображения дат создания и последнего входа

### getInitials(firstName, lastName)
- **Назначение**: Получение инициалов для аватара
- **Логика**: Берет первые буквы имени и фамилии
- **Используется**: Для отображения аватара, если нет фото

## Фильтрация и поиск

### Поиск (searchTerm)
- **Поле**: TextField с иконкой поиска
- **Логика**: 
  - Ищет по email, имени, фамилии, должности
  - Без учета регистра
  - Фильтрация происходит в реальном времени
- **Поведение**: При вводе обновляется список пользователей

## Пагинация

- **Размер страницы**: 10 элементов
- **Кнопки**: Предыдущая / Следующая
- **Отображение**: "Страница X из Y"
- **Логика**: 
  - `currentPage` и `totalPages` вычисляются на основе `filteredUsers`
  - При переключении страницы обновляется отображаемый список
