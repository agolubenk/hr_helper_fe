# Документация страницы: Команды workflow (company-settings/recruiting/commands/page.tsx)

## Общее описание

Страница управления командами для workflow чата. Позволяет создавать, редактировать и удалять команды, связывать их с этапами найма и типами действий.

**Путь**: `/company-settings/recruiting/commands`

## Компоненты страницы

### 1. RecruitingCommandsSettings
- **Расположение**: `@/components/company-settings/RecruitingCommandsSettings`
- **Назначение**: Компонент управления командами
- **Функциональность**:
  - Список команд с возможностью редактирования и удаления
  - Форма создания/редактирования команды
  - Выбор типа действия для команды
  - Выбор этапа найма (опционально)
  - Настройка цвета команды
  - Валидация уникальности команд

## Структура команды

### Интерфейс Command
```typescript
interface Command {
  id: string                    // Уникальный идентификатор
  command: string               // Текст команды (например, "/s", "/t", "/in")
  actionType: string            // Тип действия (hrscreening, tech_screening, final_interview, invite)
  stageId?: string              // ID этапа найма (опционально)
  color: string                 // Цвет для отображения в чате (hex)
  description?: string          // Описание команды
  order: number                 // Порядок отображения
}
```

### Типы действий (ActionType)
- `hrscreening` - HR скрининг кандидата
- `tech_screening` - Технический скрининг
- `final_interview` - Финальное интервью
- `invite` - Приглашение на интервью

## Системные команды

### /add
- **Тип**: Системная команда (не настраивается)
- **Назначение**: Добавление кандидата из файла
- **Использование**: Автоматически добавляется при прикреплении файла
- **Action Type**: `add_candidate`
- **Цвет**: Синий (rgba(59, 130, 246, 0.75))

### /del
- **Тип**: Системная команда (не настраивается)
- **Назначение**: Удаление последнего действия
- **Использование**: Только команда (без сообщения)
- **Action Type**: `delete_last`
- **Цвет**: Красный (rgba(239, 68, 68, 0.75))

## Функциональность

### Создание команды
1. Нажать кнопку "Создать команду"
2. Заполнить форму:
   - Команда (должна начинаться с "/")
   - Тип действия (обязательно)
   - Этап найма (опционально)
   - Цвет (для отображения в чате)
   - Описание (опционально)
3. Нажать "Создать"

### Редактирование команды
1. Нажать кнопку редактирования (иконка карандаша) в строке команды
2. Изменить нужные поля в форме
3. Нажать "Сохранить"

### Удаление команды
1. Нажать кнопку удаления (иконка корзины) в строке команды
2. Подтвердить удаление (через toast)

### Валидация
- Команда должна начинаться с "/"
- Команда должна быть уникальной
- Тип действия обязателен
- Цвет обязателен

## Связь команд с этапами и действиями

### Логика связи
1. **Тип действия (actionType)** - определяет, какое действие будет создано в workflow:
   - `hrscreening` → создается HR скрининг
   - `tech_screening` → создается технический скрининг
   - `final_interview` → создается финальное интервью
   - `invite` → создается приглашение

2. **Этап найма (stageId)** - опционально, определяет, на какой этап воронки перевести кандидата:
   - Если указан, кандидат автоматически переводится на этот этап
   - Если не указан, этап определяется автоматически на основе типа действия

### Примеры связей
- Команда `/s` → `actionType: 'hrscreening'`, `stageId: 'hr-screening'`
  - При использовании создается HR скрининг и кандидат переводится на этап "HR Screening"
  
- Команда `/t` → `actionType: 'tech_screening'`, `stageId: 'tech-screening'`
  - При использовании создается технический скрининг и кандидат переводится на этап "Tech Screening"

- Команда `/in` → `actionType: 'final_interview'`, `stageId: 'final-interview'`
  - При использовании создается финальное интервью и кандидат переводится на этап "Final Interview"

## Состояние компонента

### State переменные:
- `commands`: список всех команд
- `editingCommand`: команда для редактирования (null если создается новая)
- `showDialog`: флаг отображения диалога создания/редактирования
- `formData`: данные формы для создания/редактирования команды

## Функции и обработчики

### handleCreate()
- **Назначение**: Открытие диалога создания новой команды
- **Логика**: Сбрасывает форму и открывает диалог

### handleEdit(command)
- **Назначение**: Открытие диалога редактирования команды
- **Параметры**: `command` - команда для редактирования
- **Логика**: Заполняет форму данными команды и открывает диалог

### handleDelete(commandId)
- **Назначение**: Удаление команды
- **Параметры**: `commandId` - ID команды для удаления
- **Логика**: Удаляет команду из списка и показывает toast

### handleSave()
- **Назначение**: Сохранение команды (создание или обновление)
- **Логика**: 
  1. Валидирует данные формы
  2. Проверяет уникальность команды
  3. Создает новую команду или обновляет существующую
  4. Закрывает диалог
  5. Показывает toast с результатом

### handleCancel()
- **Назначение**: Отмена редактирования
- **Логика**: Закрывает диалог и сбрасывает форму

## Состояние компонента (детальное описание)

### State переменные в RecruitingCommandsSettings:

#### commands: Command[]
- **Тип**: `Command[]`
- **Инициализация**: `[]` (пустой массив)
- **Обновление**: 
  - При загрузке из API (useEffect при монтировании)
  - При создании новой команды (handleSave)
  - При обновлении команды (handleSave)
  - При удалении команды (handleDelete)
- **Использование**: 
  - Отображение списка команд в таблице
  - Проверка уникальности при создании/редактировании
  - Определение порядка новых команд

#### stages: Stage[]
- **Тип**: `Stage[]`
- **Инициализация**: `[]` (пустой массив)
- **Обновление**: При загрузке из API (useEffect при монтировании)
- **Использование**: 
  - Заполнение выпадающего списка этапов в форме
  - Отображение названия этапа в таблице команд

#### loading: boolean
- **Тип**: `boolean`
- **Инициализация**: `true` (показывается индикатор загрузки)
- **Обновление**: 
  - `true` - в начале загрузки команд
  - `false` - после завершения загрузки (успешной или с ошибкой)
- **Использование**: 
  - Условный рендеринг индикатора загрузки в таблице
  - Блокировка действий во время загрузки

#### editingCommand: Command | null
- **Тип**: `Command | null`
- **Инициализация**: `null` (режим создания)
- **Обновление**: 
  - `null` - при создании новой команды (handleCreate)
  - `Command` - при редактировании существующей (handleEdit)
  - `null` - при отмене редактирования (handleCancel)
- **Использование**: 
  - Определение режима работы диалога (создание/редактирование)
  - Заполнение формы данными команды при редактировании
  - Проверка уникальности (исключение текущей команды)

#### showDialog: boolean
- **Тип**: `boolean`
- **Инициализация**: `false` (диалог закрыт)
- **Обновление**: 
  - `true` - при открытии диалога (handleCreate, handleEdit)
  - `false` - при закрытии диалога (handleSave, handleCancel)
- **Использование**: 
  - Управление видимостью диалога создания/редактирования
  - Контроль состояния формы

#### formData: Partial<Command>
- **Тип**: `Partial<Command>`
- **Инициализация**: Значения по умолчанию
  ```typescript
  {
    command: '',
    actionType: 'hrscreening',
    stageId: undefined,
    color: '#3B82F6',
    description: '',
  }
  ```
- **Обновление**: 
  - При изменении полей формы (onChange обработчики)
  - При открытии диалога редактирования (handleEdit)
  - При сбросе формы (handleCreate, handleCancel)
- **Использование**: 
  - Связывание полей формы с состоянием
  - Валидация перед сохранением
  - Отправка данных на сервер

## Использование команд в workflow чате

### Обработка команд
Команды используются в workflow чате (`/workflow` или `/recr-chat`) для создания действий:

1. Пользователь вводит команду в чат (например, `/s`)
2. Система находит команду в настройках компании
3. Система определяет тип действия из команды
4. Система создает соответствующее действие (HR скрининг, технический скрининг и т.д.)
5. Если указан этап найма, кандидат переводится на этот этап

### Пример обработки
```
Пользователь: /s https://huntflow.ru/applicants/12345
↓
Система находит команду "/s" → actionType: 'hrscreening', stageId: 'hr-screening'
↓
Создается HR скрининг для кандидата
↓
Кандидат переводится на этап "HR Screening"
```

- `/recr-chat` - чат рекрутинга (использует настроенные команды)

## Особенности

- Команды /add и /del не отображаются в списке (системные)
- Команды должны начинаться с "/"
- Каждая команда связана с типом действия
- Опционально команда может быть связана с этапом найма
- Цвет команды используется для визуального отображения в чате
- Команды уникальны в рамках компании
