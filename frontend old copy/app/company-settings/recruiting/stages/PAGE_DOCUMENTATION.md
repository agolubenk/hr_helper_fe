# Документация страницы: Этапы найма (company-settings/recruiting/stages/page.tsx)

## Общее описание

Страница управления этапами найма и причинами отказа. Позволяет создавать, редактировать и удалять этапы найма, настраивать их порядок, управлять причинами отказа для каждого этапа. Этапы задают структуру pipeline в ATS и используются в отчётах. Важно настроить до начала активного подбора.

**Путь**: `/company-settings/recruiting/stages`

## Компоненты страницы

### 1. RecruitingStagesSettingsPage (основной компонент)
- **Расположение**: `@/app/company-settings/recruiting/stages/page.tsx`
- **Назначение**: Главный компонент страницы настроек этапов найма
- **Функциональность**: 
  - Отображает заголовок "Этапы найма и причины отказа"
  - Рендерит компонент RecruitingStagesSettings с формой управления этапами
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - RecruitingStagesSettings: компонент с формой управления этапами
  - Sidebar: содержит ссылку на эту страницу в разделе "Настройки рекрутинга"
  - Приветственный тур: используется data-tour="recruiting-settings-page"

### 2. RecruitingStagesSettings
- **Расположение**: `@/components/company-settings/RecruitingStagesSettings`
- **Назначение**: Компонент управления этапами найма
- **Пропсы**: Нет (использует внутреннее состояние)
- **Функциональность**: 
  - Список этапов найма с возможностью редактирования
  - Форма редактирования этапа (модальное окно)
  - Управление причинами отказа для каждого этапа
  - Порядок этапов в воронке подбора
  - **Настройка этапов-встреч**:
    - Метка "встреча" (`isMeeting`): чекбокс в модальном окне редактирования этапа
    - Если чекбокс "встреча" активен, появляется дополнительная карточка "Настройки встречи"
    - В карточке "Настройки встречи":
      - "Отображать офисы?" (да/нет) - настройка `showOffices`
        - Если "Да", то в панели настроек встречи будет выбор формата (Онлайн/Офис) и выбор офиса при выборе "Офис"
        - Если "Нет", то по определению используется только онлайн
      - "Отображать интервьюеров?" (да/нет) - настройка `showInterviewers`
        - Если "Да", то в панели настроек встречи будет выбор интервьюеров
- **Особенности**: 
  - Кнопка "Добавить этап" убрана (создание этапов происходит на другой странице)
  - Кнопки "Удалить" убраны из таблицы этапов найма (удаление происходит на другой странице)
  - Этапы с `isMeeting = true` используются в тогглере на странице `/workflow` и `/recr-chat`
  - Названия этапов-встреч становятся названиями кнопок в тогглере
  - Настройки `showOffices` и `showInterviewers` определяют содержимое панели настроек встречи
- **Связи**: 
  - Используется только на этой странице
  - Связан с настройками рекрутинга
  - `/workflow`: этапы-встречи используются для формирования тогглера типа процесса
  - `/recr-chat`: этапы-встречи используются для формирования тогглера типа процесса
  - WorkflowHeader: использует этапы-встречи для динамического формирования кнопок
  - VacancyEditModal: использует настройки этапов для управления форматом встречи

## Интерфейсы и типы

### Stage (интерфейс этапа найма)
```typescript
interface HiringStage {
  id: string                                    // Уникальный идентификатор этапа
  name: string                                  // Название этапа (Заявка, Скрининг, Интервью, Оффер, Принят, Отказ)
  order: number                                 // Порядок следования в воронке
  color: string                                 // Цвет этапа (hex)
  description?: string                          // Описание этапа (опционально)
  autoTransition?: boolean                       // Автоматический переход на следующий этап (опционально)
  rejectionReasonIds?: string[]                 // Массив ID причин отказа для этого этапа (опционально)
  isMeeting?: boolean                           // Метка "встреча" - этап используется в тогглере на странице /workflow и /recr-chat
  showOffices?: boolean                          // Отображать офисы для этапа-встречи (да/нет)
  showInterviewers?: boolean                    // Отображать интервьюеров для этапа-встречи (да/нет)
}
```

**Особенности этапов-встреч**:
- Этапы с `isMeeting = true` используются в тогглере на странице `/workflow` и `/recr-chat`
- Названия этапов-встреч становятся названиями кнопок в тогглере
- Количество кнопок в тогглере: 0 и более (зависит от количества этапов с `isMeeting = true`)
- Настройки `showOffices` и `showInterviewers` определяют содержимое панели настроек встречи

### RejectionReason (интерфейс причины отказа)
```typescript
interface RejectionReason {
  id: string                                    // Уникальный идентификатор причины
  name: string                                  // Название причины отказа
  description?: string                          // Описание причины (опционально)
}
```

## Состояние компонента

### State переменные (в RecruitingStagesSettings):
- `stages`: массив всех этапов найма
  - Инициализация: пустой массив []
  - Обновление: при загрузке из API или при создании/редактировании/удалении
  - Использование: для отображения списка этапов

- `editingStage`: этап для редактирования
  - Инициализация: null
  - Обновление: при клике на кнопку редактирования
  - Использование: для заполнения формы редактирования

- `showAddForm`: флаг отображения формы добавления
  - Инициализация: false
  - Обновление: при клике на кнопку "Добавить этап"
  - Использование: для условного рендеринга формы добавления
