# Документация страницы: SLA (company-settings/sla/page.tsx)

## Общее описание

Страница управления SLA (Service Level Agreement) для заявок на подбор. Позволяет создавать, редактировать и удалять SLA, настраивать сроки и параметры.

## Компоненты страницы

### 1. SLASettings
- **Расположение**: `@/components/company-settings/SLASettings`
- **Назначение**: Компонент настроек SLA
- **Пропсы**: Нет (использует внутреннее состояние)
- **Логика**: 
  - Отображение списка всех SLA
  - Создание новых SLA
  - Редактирование существующих SLA
  - Удаление SLA
- **Связи**: Используется для управления SLA

### 2. SLAEditModal
- **Расположение**: `@/components/company-settings/SLAEditModal`
- **Назначение**: Модальное окно создания/редактирования SLA
- **Пропсы**: 
  - `isOpen`: флаг открытости
  - `onClose`: обработчик закрытия
  - `sla`: данные SLA для редактирования (опционально)
  - `onSave`: обработчик сохранения
- **Логика**: 
  - Форма с полями SLA
  - Валидация обязательных полей
  - Сохранение через API
- **Связи**: Используется для создания и редактирования SLA

## Состояние компонента

### State переменные:
- `slas`: массив всех SLA
- `editingSla`: SLA для редактирования
- `isEditModalOpen`: флаг открытия модального окна

## Функции и обработчики

### CRUD операции:
- **Создание**: Добавление нового SLA через `SLAEditModal`
- **Чтение**: Загрузка списка SLA
- **Обновление**: Редактирование существующего SLA
- **Удаление**: Удаление SLA с подтверждением

## Моковые данные

### mockSLAs
Массив объектов SLA с полями:
- `id`: уникальный идентификатор
- `name`: название SLA
- `days`: количество дней для выполнения
- `description`: описание SLA
- `isActive`: флаг активности
