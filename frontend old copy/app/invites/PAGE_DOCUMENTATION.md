# Документация страницы: Инвайты (invites/page.tsx)

## Общее описание

Страница управления инвайтами на интервью. Позволяет создавать, просматривать, редактировать и удалять инвайты для кандидатов. Инвайт — это приглашение кандидата на интервью с указанием даты, времени, формата (онлайн/офис) и других деталей.

**Путь**: `/invites`

## Компоненты страницы

### 1. InvitesPage (основной компонент)
- **Расположение**: `@/app/invites/page.tsx`
- **Назначение**: Главный компонент страницы управления инвайтами
- **Функциональность**: 
  - Отображение статистики по инвайтам
  - Поиск и фильтрация инвайтов
  - Отображение списка инвайтов в таблице
  - Создание новых инвайтов через модальное окно
  - Навигация к детальному просмотру и редактированию
  - Удаление инвайтов
  - Копирование текста инвайта
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - useRouter: для навигации к детальной странице или редактированию
  - useToast: для отображения уведомлений
  - invitesApi: API для работы с инвайтами
  - InvitesStats: компонент статистики
  - CreateInviteModal: модальное окно создания инвайта

### 2. InvitesStats (компонент статистики)
- **Расположение**: `@/components/invites/InvitesStats.tsx`
- **Назначение**: Отображение статистики по инвайтам
- **Пропсы**:
  - `total`: number - общее количество инвайтов
  - `pending`: number - количество инвайтов со статусом "pending" (ожидают)
  - `sent`: number - количество инвайтов со статусом "sent" (отправлены)
  - `completed`: number - количество инвайтов со статусом "completed" (завершены)
- **Функциональность**: 
  - Отображает 4 карточки со статистикой
  - Каждая карточка имеет цветовую индикацию (границы сверху и снизу)
  - Иконки для визуального различия статусов
- **Карточки**:
  - **Всего инвайтов**: CalendarIcon, синяя граница сверху, зеленая снизу
  - **Ожидают**: ClockIcon, оранжевые границы
  - **Отправлены**: PaperPlaneIcon, синие границы
  - **Завершены**: CheckCircledIcon, зеленые границы
- **Связи**: Получает данные из MOCK_STATS (TODO: заменить на API)

### 3. CreateInviteModal (модальное окно создания)
- **Расположение**: `@/components/invites/CreateInviteModal.tsx`
- **Назначение**: Модальное окно создания нового инвайта
- **Пропсы**:
  - `isOpen`: boolean - флаг открытости модального окна
  - `onClose`: () => void - обработчик закрытия
  - `onSave?`: (data: { combined_data: string }) => void - обработчик успешного сохранения (опционально)
- **Функциональность**: 
  - Поле ввода для комбинированных данных (ссылка на кандидата + дата-время)
  - Поддержка различных форматов даты-времени
  - Поддержка кастомной длительности интервью (в скобках)
  - Информационная секция с примерами форматов
  - Валидация введенных данных
  - Отправка данных на сервер для создания инвайта
- **Форматы данных**:
  - **Ссылка**: https://huntflow.ru/my/org#/vacancy/[id]/filter/[id]/id/[id] или sandbox.huntflow.dev
  - **Дата-время**: YYYY-MM-DD HH:MM, DD.MM.YYYY HH:MM, DD/MM/YYYY HH:MM
  - **Длительность**: (1 час), (30 минут), (полчаса), (2 ч), (45 м)
- **Связи**: 
  - API: `/google-oauth/invites/create/combined/` (TODO: раскомментировать)
  - После создания вызывает `loadInvites()` для обновления списка

### 4. Поиск и фильтры
- **Компонент**: Card с TextField и Select
- **Назначение**: Поиск и фильтрация инвайтов
- **Функциональность**: 
  - **Поиск**: TextField с иконкой MagnifyingGlassIcon
    - Поиск по имени кандидата или названию вакансии
    - Debounce 500мс для уменьшения количества запросов
  - **Фильтр по статусу**: Select с опциями из STATUS_OPTIONS
    - Опции: "Все статусы", "Ожидает", "Отправлен", "Завершен", "Отменен"
  - **Кнопка "Сбросить"**: сбрасывает поиск, фильтр и страницу
- **Поведение**: 
  - При вводе в поле поиска сбрасывает страницу на первую (если не на первой)
  - При изменении фильтра статуса загружает инвайты с новым фильтром

### 5. Таблица инвайтов
- **Компонент**: Table.Root с колонками
- **Назначение**: Отображение списка всех инвайтов
- **Колонки**:
  - **Кандидат**: 
    - Имя кандидата (кликабельно, цвет accent-9)
    - При клике: переход на детальную страницу `/invites/${id}`
  - **Вакансия**: 
    - Название вакансии или "Не указано"
  - **Дата интервью**: 
    - Иконка CalendarIcon
    - Отформатированная дата и время интервью
  - **Статус**: 
    - Badge с цветовой индикацией (STATUS_COLORS)
    - Отображает status_display или status
  - **Создан**: 
    - Дата создания инвайта (отформатированная через formatDate)
  - **Действия**: 
    - Кнопка "Просмотр" (EyeOpenIcon): переход на детальную страницу
    - Кнопка "Редактировать" (Pencil1Icon): переход на страницу редактирования
    - Кнопка "Открыть Scorecard" (FileTextIcon): открывает Google Drive файл (если есть google_drive_file_url)
    - Кнопка "Копировать текст инвайта" (FileTextIcon): копирует текст инвайта в буфер обмена
    - Кнопка "Удалить" (TrashIcon, красная): удаляет инвайт с подтверждением
- **Состояния**: 
  - **Загрузка**: отображает "Загрузка..."
  - **Пусто**: отображает сообщение "Инвайты не найдены" с кнопкой создания
- **Поведение**: 
  - Отображает только инвайты текущей страницы (пагинация)
  - При клике на имя кандидата или кнопку просмотра открывает детальную страницу

### 6. Пагинация
- **Компонент**: Flex с кнопками навигации
- **Назначение**: Навигация по страницам списка инвайтов
- **Элементы**:

  - Информация о странице: "Страница X из Y"
  - Кнопка "Следующая": переход на следующую страницу
- **Поведение**: 
  - Отображается только если hasNext || hasPrevious
  - Кнопки отключены при loading или отсутствии страниц
  - Размер страницы: 20 элементов

### 7. Заголовок и действия
- **Компонент**: Flex с заголовком и кнопками
- **Назначение**: Заголовок страницы и быстрые действия
- **Элементы**: 
  - Заголовок: "Список инвайтов"

## Интерфейсы и типы

### Invite
```typescript
interface Invite {
  id: number                                    // Уникальный идентификатор инвайта
  candidate_name: string                        // Имя кандидата
  candidate_email?: string                      // Email кандидата (опционально)
  candidate_url?: string                        // Ссылка на кандидата в Huntflow (опционально)
  candidate_id?: string                         // ID кандидата (опционально)
  candidate_grade?: string                      // Грейд кандидата (опционально)
  vacancy_id?: string                           // ID вакансии (опционально)
  vacancy_title?: string                        // Название вакансии (опционально)
  interview_datetime: string                     // Дата и время интервью (ISO строка)
  interview_datetime_formatted?: string         // Отформатированная дата и время (опционально)
  custom_duration_minutes?: number              // Кастомная длительность интервью в минутах (опционально)
  status: 'pending' | 'sent' | 'completed' | 'cancelled' // Статус инвайта
  status_display?: string                      // Отображаемое название статуса (опционально)
  interview_format?: 'online' | 'office'        // Формат интервью (опционально)
  google_drive_file_id?: string                 // ID файла в Google Drive (опционально)
  google_drive_file_url?: string                // Ссылка на файл в Google Drive (опционально)
  calendar_event_id?: string                    // ID события в календаре (опционально)
  calendar_event_url?: string                  // Ссылка на событие в календаре (опционально)
  google_meet_url?: string                      // Ссылка на Google Meet (опционально)
  interviewer?: number                          // ID интервьюера (опционально)
  interviewer_name?: string                     // Имя интервьюера (опционально)
  created_at: string                            // Дата создания (ISO строка)
  updated_at: string                             // Дата обновления (ISO строка)
}
```

## Состояние компонента

### State переменные:
- `invites` (Invite[]): массив всех инвайтов
  - Инициализация: MOCK_INVITES
  - Обновление: при загрузке инвайтов через loadInvites()
  - Использование: отображение в таблице
- `loading` (boolean): флаг загрузки данных
  - Инициализация: false
  - Обновление: устанавливается в true при начале загрузки, false при завершении
  - Использование: блокировка кнопок и отображение индикатора загрузки
- `searchQuery` (string): поисковый запрос для фильтрации инвайтов
  - Инициализация: пустая строка
  - Обновление: при вводе в поле поиска
  - Использование: фильтрация инвайтов по имени кандидата или вакансии
- `statusFilter` (string): выбранный статус для фильтрации
  - Инициализация: 'all'
  - Обновление: при выборе статуса в Select
  - Использование: фильтрация инвайтов по статусу
- `page` (number): текущая страница пагинации
  - Инициализация: 1
  - Обновление: при клике на кнопки пагинации
  - Использование: вычисление startIndex и endIndex для пагинации
- `totalCount` (number): общее количество инвайтов
  - Инициализация: 0
  - Обновление: при загрузке инвайтов
  - Использование: отображение количества и вычисление totalPages
- `hasNext` (boolean): флаг наличия следующей страницы
  - Инициализация: false
  - Обновление: при загрузке инвайтов из API
  - Использование: блокировка кнопки "Следующая"
- `hasPrevious` (boolean): флаг наличия предыдущей страницы
  - Инициализация: false
  - Обновление: при загрузке инвайтов из API
  - Использование: блокировка кнопки "Предыдущая"
- `isCreateModalOpen` (boolean): флаг открытия модального окна создания инвайта
  - Инициализация: false
  - Обновление: при открытии/закрытии модального окна
  - Использование: управление видимостью CreateInviteModal

### Константы:
- `STATUS_OPTIONS`: опции статусов для фильтрации
  - Значения: 'all', 'pending', 'sent', 'completed', 'cancelled'
- `STATUS_COLORS`: маппинг статусов на цвета для Badge
  - pending: 'yellow'
  - sent: 'blue'
  - completed: 'green'
  - cancelled: 'gray'
- `MOCK_STATS`: моковые данные статистики
  - total: 563
  - pending: 13
  - sent: 550
  - completed: 0
- `MOCK_INVITES`: моковые данные инвайтов (массив из 3 инвайтов)

### Hooks:
- `useRouter()`: программная навигация
  - Используется для: перехода на детальную страницу и редактирование
- `useToast()`: отображение уведомлений
  - Используется для: показа успешных операций и ошибок

## Функции и обработчики

### loadInvites()
- **Назначение**: Загрузка списка инвайтов с учетом фильтров
- **Параметры**: Нет
- **Логика**: 
  1. Устанавливает loading = true
  2. Имитирует задержку API (300мс)
  3. Фильтрует моковые данные:
     - По searchQuery: проверяет candidate_name и vacancy_title
     - По statusFilter: если не 'all', фильтрует по статусу
  4. Обновляет состояние: invites, totalCount, hasNext, hasPrevious
  5. Устанавливает loading = false
- **Вызывается**: 
  - При изменении page или statusFilter (useEffect)
  - При debounce поиска (через 500мс после ввода)
  - После удаления инвайта
  - После создания инвайта
  - При клике на кнопку "Обновить"
- **TODO**: Раскомментировать реальный API вызов invitesApi.getAll()

### handleDelete(id)
- **Назначение**: Удаление инвайта
- **Параметры**: 
  - `id`: number - ID инвайта для удаления
- **Логика**: 
  1. Показывает confirm диалог для подтверждения
  2. Если подтверждено, вызывает invitesApi.delete(id)
  3. При успехе показывает toast "Успешно" и обновляет список (loadInvites)
  4. При ошибке показывает toast "Ошибка удаления"
- **Используется для**: 
  - Удаления инвайта из таблицы (кнопка удаления)

### handleCopyInvitation(id)
- **Назначение**: Копирование текста инвайта в буфер обмена
- **Параметры**: 
  - `id`: number - ID инвайта для копирования текста
- **Логика**: 
  1. Вызывает invitesApi.getInvitationText(id)
  2. Если успешно, копирует invitation_text в буфер обмена через navigator.clipboard.writeText()
  3. Показывает toast "Успешно" или "Ошибка"
- **Используется для**: 
  - Копирования текста инвайта для отправки кандидату

### formatDate(dateString)
- **Назначение**: Форматирование даты для отображения
- **Параметры**: 
  - `dateString`: string - дата в формате ISO строки
- **Возвращает**: string - отформатированная дата в формате DD.MM.YYYY HH:MM или исходная строка при ошибке
- **Логика**: 
  1. Парсит ISO строку в Date объект
  2. Форматирует через toLocaleString('ru-RU')
  3. При ошибке возвращает исходную строку
- **Используется для**: 
  - Отображения дат создания и времени интервью в таблице

### useEffect для загрузки инвайтов
- **Назначение**: Загрузка инвайтов при изменении страницы или фильтра статуса
- **Зависимости**: [page, statusFilter]
- **Логика**: 
  - Вызывает loadInvites() при изменении page или statusFilter
- **Поведение**: 
  - Выполняется при переключении страниц пагинации
  - Выполняется при изменении фильтра по статусу

### useEffect для debounce поиска
- **Назначение**: Задержка загрузки инвайтов на 500мс после ввода поискового запроса
- **Зависимости**: [searchQuery]
- **Логика**: 
  1. Устанавливает таймер на 500мс
  2. Если page === 1, вызывает loadInvites()
  3. Если page !== 1, сбрасывает page на 1 (что вызовет загрузку через другой useEffect)
  4. Очищает таймер при размонтировании или изменении searchQuery
- **Поведение**: 
  - Предотвращает избыточные запросы при быстром вводе текста
  - Улучшает производительность

## Фильтрация и поиск

### Поиск (searchQuery)
- **Поле**: TextField с иконкой MagnifyingGlassIcon
- **Логика**: 
  - Ищет по имени кандидата (candidate_name) или названию вакансии (vacancy_title)
  - Без учета регистра (toLowerCase)
  - Debounce 500мс для уменьшения количества запросов
- **Поведение**: 
  - При вводе устанавливает таймер на 500мс
  - Если текущая страница не первая, сбрасывает на первую
  - После debounce загружает инвайты с новым поисковым запросом

### Фильтр по статусу (statusFilter)
- **Поле**: Select с опциями из STATUS_OPTIONS
- **Опции**: 
  - "Все статусы" (value: 'all')
  - "Ожидает" (value: 'pending')
  - "Отправлен" (value: 'sent')
  - "Завершен" (value: 'completed')
  - "Отменен" (value: 'cancelled')
- **Логика**: 
  - Если выбран 'all', показывает все инвайты
  - Если выбран другой статус, фильтрует инвайты по статусу
- **Поведение**: 
  - При изменении фильтра загружает инвайты с новым фильтром
  - Вызывается loadInvites() через useEffect

### Кнопка "Сбросить"
- **Назначение**: Сброс всех фильтров и поиска
- **Логика**: 
  - Сбрасывает searchQuery в пустую строку
  - Сбрасывает statusFilter в 'all'
  - Сбрасывает page в 1
- **Поведение**: 
  - После сброса загружаются все инвайты без фильтров

- **Логика**:

  - При переключении страницы вызывается loadInvites() через useEffect
  - totalPages вычисляется: Math.ceil(totalCount / 20)
- **Поведение**: 
  - Отображается только если hasNext || hasPrevious
  - Кнопки отключены при loading или отсутствии страниц
  - При клике обновляет page, что вызывает загрузку инвайтов

## Навигация

- **Детальный просмотр**: 
  - `router.push(/invites/${id})` - переход на страницу `/invites/[id]/page.tsx`
  - Вызывается при клике на имя кандидата или кнопку "Просмотр"
- **Редактирование**: 
  - `router.push(/invites/${id}/edit)` - переход на страницу `/invites/[id]/edit/page.tsx`
  - Вызывается при клике на кнопку "Редактировать"
- **Открытие Scorecard**: 
  - `window.open(google_drive_file_url, '_blank')` - открывает Google Drive файл в новой вкладке
  - Вызывается при клике на кнопку "Открыть Scorecard" (если есть google_drive_file_url)
- **Создание инвайта (внешняя ссылка)**: 
  - `window.open('/google-oauth/invites/create/', '_blank')` - открывает внешнюю страницу создания
  - Вызывается при клике на кнопку "Создать инвайт" в пустом состоянии
