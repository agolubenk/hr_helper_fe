# Документация страницы: Детальный просмотр инвайта (invites/[id]/page.tsx)

## Общее описание

Страница детального просмотра инвайта на интервью. Отображает полную информацию об инвайте, позволяет управлять им (копирование текста, пересоздание scorecard, редактирование, удаление).

## Компоненты страницы

### 1. Заголовок страницы
- **Компонент**: `Flex` с кнопкой "Назад"
- **Назначение**: Навигация и заголовок страницы
- **Логика**: 
  - Кнопка "Назад" возвращает на страницу списка инвайтов
  - Отображает название страницы "Детали инвайта"
- **Связи**: Навигация на `/invites`

### 2. Карточка информации об инвайте
- **Компонент**: `Card` с информацией
- **Назначение**: Отображение основной информации об инвайте
- **Секции**:
  - Статус инвайта (Badge с цветовой индикацией)
  - Информация о кандидате (имя, email, ссылка в Huntflow)
  - Информация о вакансии (название)
  - Время интервью (дата и время)
  - Формат интервью (Онлайн/Офис)
  - Ссылки (Google Meet, Google Calendar, Scorecard)
- **Логика**: Отображает все данные инвайта в структурированном виде

### 3. Кнопки действий
- **Компонент**: `Button` компоненты
- **Назначение**: Управление инвайтом
- **Действия**:
  - Копировать текст инвайта
  - Пересоздать Scorecard
  - Редактировать инвайт
  - Удалить инвайт
- **Логика**: 
  - Каждая кнопка вызывает соответствующую функцию
  - Показывает toast уведомления об успехе/ошибке

## Состояние компонента

### State переменные:
- `invite`: данные инвайта для отображения (тип: `Invite | null`)
- `loading`: флаг загрузки данных

- **Логика**: 
  1. Проверяет наличие inviteId, если нет - выходит
  2. Устанавливает `loading = true`
  3. Имитирует задержку API (300мс)
  4. Загружает моковые данные инвайта (разные данные в зависимости от ID)
  5. Устанавливает invite в состояние

  7. Устанавливает `loading = false` в finally
- **Вызывается**: 
  - При монтировании компонента (useEffect)
  - При изменении inviteId из URL

### handleCopyInvitation()
- **Назначение**: Копирование текста инвайта в буфер обмена
- **Параметры**: Нет (использует inviteId из состояния)
- **Логика**: 
  1. Проверяет наличие inviteId, если нет - выходит
  2. Получает текст инвайта через `invitesApi.getInvitationText(inviteId)`
  3. Проверяет успешность ответа и наличие invitation_text
  4. Копирует текст в буфер обмена через `navigator.clipboard.writeText()`
  5. Показывает toast об успехе или ошибке
- **Поведение**: 
  - Вызывается при клике на кнопку "Копировать приглашение"
  - Обрабатывает ошибки получения или копирования текста
- **Связи**: 
  - invitesApi.getInvitationText: API для получения текста инвайта
  - navigator.clipboard: API браузера для работы с буфером обмена

### handleDelete()
- **Назначение**: Удаление инвайта
- **Параметры**: Нет (использует inviteId из состояния)
- **Логика**: 
  1. Проверяет наличие inviteId, если нет - выходит
  2. Показывает confirm диалог для подтверждения удаления
  3. При подтверждении вызывает `invitesApi.delete(inviteId)`
  4. При успехе показывает toast "Успешно" и редиректит на список инвайтов
  5. При ошибке показывает toast "Ошибка удаления"
- **Поведение**: 
  - Вызывается при клике на кнопку "Удалить инвайт"
  - Обрабатывает ошибки удаления
- **Связи**: 
  - invitesApi.delete: API для удаления инвайта
  - router.push: навигация на список инвайтов после успешного удаления

### formatDate(dateString)
- **Назначение**: Форматирование даты для отображения
- **Параметры**: 
  - `dateString`: string - дата в формате ISO строки
- **Возвращает**: string - отформатированная дата в формате DD.MM.YYYY HH:MM или исходная строка при ошибке
- **Логика**: 
  1. Парсит ISO строку в Date объект
  2. Форматирует через toLocaleString('ru-RU') с опциями:
     - day: '2-digit'
     - month: '2-digit'
     - year: 'numeric'
     - hour: '2-digit'
     - minute: '2-digit'
  3. При ошибке парсинга возвращает исходную строку
- **Используется**: 
  - Для отображения дат создания и обновления инвайта
  - Для отображения времени интервью в метаданных

## URL параметры

### [id]
- **Тип**: Динамический параметр маршрута
- **Назначение**: ID инвайта для просмотра
- **Пример**: `/invites/123` отображает инвайт с ID 123
- **Обработка**: 
  - Читается через `useParams()`
  - Используется для загрузки данных инвайта
