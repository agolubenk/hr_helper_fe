# Документация страницы: Профиль (account/profile/page.tsx)

## Общее описание

Страница профиля пользователя. Позволяет управлять личными данными, настраивать интеграции, управлять быстрыми кнопками и настраивать акцентный цвет темы. Поддерживает несколько вкладок с различным содержимым.

**Путь**: `/account/profile` или `/account/profile?tab=integrations`

## Состояние компонента

### State переменные:
- `activeTab` (TabType): текущая активная вкладка профиля
  - Тип: `'profile' | 'edit' | 'integrations' | 'quick-buttons'`
  - Инициализация: `useState<TabType>(getInitialTab)`
  - Обновление: через `setActiveTab` при клике на вкладку или синхронизации
  - Использование: определение, какой компонент отображать в правой колонке

### Hooks:
- `useTheme()`: получение и установка акцентных цветов темы
  - `lightThemeAccentColor`: акцентный цвет для светлой темы
  - `darkThemeAccentColor`: акцентный цвет для темной темы
  - `setLightThemeAccentColor`: функция установки цвета светлой темы
  - `setDarkThemeAccentColor`: функция установки цвета темной темы
- `useState`, `useEffect`: управление состоянием и жизненным циклом

### Моковые данные:
- `userData`: объект с данными пользователя
  - Структура: `{ firstName, lastName, email, telegram, linkedin, registrationDate, lastLoginDate, workSchedule, workStartTime, workEndTime, meetingInterval, activeEnvironment }`

## Функции и обработчики

### getInitialTab(): TabType
- **Назначение**: Определение начальной активной вкладки
- **Параметры**: Нет
- **Возвращает**: `TabType` - начальная активная вкладка
- **Логика определения**:
  1. Проверяет URL параметр `?tab=` (высший приоритет)
  2. Если параметр 'integrations' - возвращает 'integrations'
  3. Проверяет localStorage на наличие сохраненной вкладки
  4. Если сохраненная вкладка валидна - возвращает её
  5. По умолчанию возвращает 'profile'
- **Используется для**: 
  - Восстановления активной вкладки при перезагрузке страницы
  - Открытия конкретной вкладки через URL параметры
  - Синхронизации с Sidebar (при клике на "Интеграции и API")
- **Валидация**: Проверяет, что сохраненное значение является валидным TabType

### useEffect - сохранение активной вкладки в localStorage
- **Назначение**: Сохранение текущей активной вкладки в localStorage
- **Зависимости**: `[activeTab]`
- **Логика**: 
  - Сохраняет текущую активную вкладку в localStorage с ключом 'profileActiveTab'
  - Позволяет восстановить вкладку при следующем посещении страницы
- **Поведение**: 
  - Выполняется при каждом изменении activeTab
  - Сохраняет значение в localStorage
  - Используется для синхронизации между вкладками браузера
- **Связи**: 
  - Синхронизируется с Sidebar: при клике на "Интеграции и API" Sidebar сохраняет 'integrations'
  - Восстанавливается при следующем открытии страницы профиля

### useEffect - синхронизация активной вкладки между вкладками браузера
- **Назначение**: Синхронизация активной вкладки между вкладками браузера
- **Зависимости**: `[]` (выполняется только при монтировании)
- **Механизмы синхронизации**:
  1. **storage event**: срабатывает при изменении localStorage в другой вкладке
     - Обработчик: `handleStorageChange`
     - Обновляет activeTab, если изменился 'profileActiveTab'
  2. **localStorageChange**: кастомное событие для синхронизации в той же вкладке
     - Обработчик: `handleCustomStorageChange`
     - Используется для синхронизации в той же вкладке (storage event не срабатывает)
  3. **focus event**: проверка при возврате фокуса на окно
     - Обработчик: `handleFocus`
     - Проверяет актуальное значение в localStorage при возврате фокуса
  4. **setInterval**: периодическая проверка каждые 200мс
     - Обеспечивает надежную синхронизацию даже если события не сработали
     - Обновляет только если значение действительно изменилось
- **Поведение**: 
  - При изменении activeTab в другой вкладке - обновляет текущую вкладку
  - При клике на "Интеграции и API" в Sidebar - открывает вкладку integrations
  - При возврате фокуса на окно - проверяет актуальное значение
  - Периодическая проверка обеспечивает надежную синхронизацию
- **Связи**: 
  - Sidebar: при клике на "Интеграции и API" сохраняет 'integrations' в localStorage
  - Header: при клике на имя пользователя может устанавливать активную вкладку
- **Очистка**: 
  - Удаляет все обработчики событий при размонтировании компонента
  - Очищает интервал периодической проверки

### handleSave(data)
- **Назначение**: Обработчик сохранения данных пользователя
- **Параметры**: 
  - `data`: объект с данными пользователя для сохранения
    - `firstName`, `lastName`, `email`: обязательные поля
    - `telegram`, `linkedin`: опциональные поля
    - `workStartTime`, `workEndTime`, `meetingInterval`: опциональные поля
- **Логика**: 
  - В текущей реализации только логирует данные
  - Должен отправлять данные на сервер через API
  - После успешного сохранения можно вернуться на вкладку профиля

  - Обновление данных в компоненте после успешного сохранения
  - Редирект на вкладку профиля после сохранения

### handleCancel()
- **Назначение**: Обработчик отмены редактирования
- **Параметры**: Нет
- **Логика**: 
  - Отменяет редактирование и возвращает на вкладку просмотра профиля
  - Выполняет `setActiveTab('profile')`
- **Поведение**: 
  - Вызывается из ProfileEditForm при нажатии кнопки "Отмена"
  - Переключает активную вкладку на 'profile'
- **Связи**: 
  - ProfileEditForm: вызывает эту функцию при отмене

### renderContent()
- **Назначение**: Функция рендеринга содержимого в зависимости от активной вкладки
- **Параметры**: Нет
- **Возвращает**: JSX компонент для отображения в правой колонке
- **Логика**: 
  - Использует `switch` для определения компонента по `activeTab`
  - Возвращает соответствующий компонент с необходимыми пропсами
- **Вкладки**:
  - `'profile'`: ProfileInfo - просмотр информации о пользователе
  - `'edit'`: ProfileEditForm + AccentColorSettings - редактирование данных и настройка цвета
  - `'integrations'`: IntegrationsPage - настройка интеграций и API
  - `'quick-buttons'`: QuickButtonsPage - управление быстрыми кнопками
- **Поведение**: 
  - Возвращает соответствующий компонент в зависимости от activeTab
  - На вкладке 'edit' отображает форму редактирования и настройки цвета
- **Связи**: 
  - Используется в правой колонке для отображения содержимого активной вкладки

## URL параметры

### ?tab=integrations
- **Назначение**: Открытие вкладки "Интеграции и API" при загрузке страницы
- **Логика**: 
  - Читается в `getInitialTab()` через `new URLSearchParams(window.location.search).get('tab')`
  - Если параметр 'integrations' - возвращает 'integrations'
  - Используется для перехода из Sidebar на "Интеграции и API"
- **Пример**: `/account/profile?tab=integrations` открывает вкладку integrations
- **Другие значения**: 
  - `?tab=profile` - вкладка профиля
  - `?tab=edit` - вкладка редактирования
  - `?tab=quick-buttons` - вкладка быстрых кнопок

## Синхронизация между вкладками браузера

### localStorage
- **Ключ**: `'profileActiveTab'`
- **Значение**: текущая активная вкладка ('profile' | 'edit' | 'integrations' | 'quick-buttons')
- **Логика**: 
  - Сохраняется при изменении `activeTab` через `useEffect`
  - Читается при инициализации в `getInitialTab()`
  - Синхронизируется через события 'storage' между вкладками браузера

### События storage
- **Назначение**: Синхронизация активной вкладки между вкладками браузера
- **Логика**: 
  - Слушает события 'storage' в других вкладках
  - При изменении `profileActiveTab` в другой вкладке обновляет `activeTab`
  - Позволяет синхронизировать состояние между несколькими открытыми вкладками
- **Обработчик**: `handleStorageChange`

### Кастомное событие localStorageChange
- **Назначение**: Синхронизация в той же вкладке (storage event не срабатывает)
- **Логика**: 
  - Слушает кастомное событие 'localStorageChange'
  - Обрабатывает события с `detail.key === 'profileActiveTab'`
  - Валидирует и обновляет activeTab
- **Обработчик**: `handleCustomStorageChange`
- **Связи**: 
  - Sidebar отправляет это событие при клике на "Интеграции и API"

### Событие focus
- **Назначение**: Проверка актуального значения при возврате фокуса
- **Логика**: 
  - Срабатывает при возврате фокуса на окно браузера
  - Проверяет актуальное значение в localStorage
  - Синхронизирует activeTab с localStorage
- **Обработчик**: `handleFocus`

### Периодическая проверка (setInterval)
- **Назначение**: Дополнительная гарантия синхронизации
- **Логика**: 
  - Проверяет значение в localStorage каждые 200мс
  - Обновляет activeTab, если значение изменилось
  - Обеспечивает надежную синхронизацию даже если события не сработали
- **Интервал**: 200мс
- **Очистка**: `clearInterval` при размонтировании компонента
