# Документация страницы: Вход (account/login/page.tsx)

## Общее описание

Страница входа в систему. Поддерживает два метода авторизации: email/пароль и Google OAuth, позволяет переключать тему и переходить на восстановление пароля.

**Путь**: `/account/login`

## Компоненты страницы

### 1. FloatingLabelInput
- **Расположение**: `@/components/FloatingLabelInput`
- **Назначение**: Поле ввода с плавающей меткой
- **Пропсы**:
  - `id`: уникальный идентификатор поля
  - `label`: текст метки
  - `type`: тип поля ('text' | 'password' | 'email')
  - `value`: значение поля (контролируемый компонент)
  - `onChange`: обработчик изменения значения
  - `placeholder`: подсказка в пустом поле
  - `required`: обязательное поле
  - `icon`: иконка слева от поля (опционально)
- **Логика**: 
  - Метка перемещается вверх при фокусе или заполнении поля
  - Анимация перемещения метки
  - Валидация через HTML5 атрибуты
- **Используется**: Для полей email и password

### 2. Кнопка переключения темы
- **Компонент**: `Button` с иконкой `SunIcon`/`MoonIcon`
- **Назначение**: Переключение между светлой и темной темой
- **Логика**: 
  - Использует `useTheme().toggleTheme()` для переключения темы
  - Отображает `MoonIcon` для светлой темы (предложение переключить на темную)
  - Отображает `SunIcon` для темной темы (предложение переключить на светлую)
  - Сохраняет выбор темы в localStorage через ThemeProvider
- **Расположение**: 
  - Десктоп: справа от формы (position: absolute, right: -56px)
  - Мобильные: в правом верхнем углу формы (position: absolute, top: 16px, right: 16px)
- **Стили**: 
  - Круглая кнопка (borderRadius: '50%')
  - Размер: 40x40px
  - CSS классы: `styles.themeButtonDesktop` и `styles.themeButtonMobile` для адаптивности

### 3. Форма авторизации
- **Компонент**: HTML `<form>` с обработчиком `onSubmit`
- **Назначение**: Вход через email и пароль
- **Поля**:
  - Email (обязательное, type="email")
  - Пароль (обязательное, type="password")
- **Логика**: 
  - Валидация через HTML5 атрибуты (required, type="email")
  - При submit вызывает `handleEmailLogin`
  - Показывает индикатор загрузки на кнопке во время отправки

### 4. Разделитель "или"
- **Компонент**: `Separator` с текстом
- **Назначение**: Визуальное разделение между формой email и кнопкой Google
- **Логика**: 
  - Две линии (Separator) с текстом "или" между ними
  - flex: 1 для каждой линии - равномерное распределение пространства

### 5. Кнопка авторизации через Google
- **Компонент**: `Button` с иконкой Google
- **Назначение**: Вход через Google OAuth
- **Логика**: 
  - При клике вызывает `handleGoogleLogin`
  - Выполняет редирект на OAuth endpoint
- **Визуальное оформление**: 
  - Градиентная иконка Google (синий → зеленый → желтый → красный)
  - Буква "G" белого цвета на градиентном фоне
  - variant="outline" для визуального отличия от основной кнопки

### 6. Ссылка "Забыли пароль?"
- **Компонент**: `Button` с variant="ghost"
- **Назначение**: Переход на страницу восстановления пароля
- **Логика**: 
  - При клике выполняет `router.push('/account/forgot-password')`
  - Использует программную навигацию Next.js

## Состояние компонента

### State переменные:
- `email` (string): значение поля email
  - Инициализация: `useState('')`
  - Обновление: через `onChange` обработчик поля ввода
  - Использование: отправка в API при авторизации
- `password` (string): значение поля пароля
  - Инициализация: `useState('')`
  - Обновление: через `onChange` обработчик поля ввода
  - Использование: отправка в API при авторизации
- `isLoading` (boolean): флаг загрузки
  - Инициализация: `useState(false)`
  - Обновление: `true` при отправке формы, `false` после завершения
  - Использование: блокировка кнопки и изменение текста на "Вход..."

### Hooks:
- `useTheme()`: получение темы и функции переключения из ThemeProvider
  - `theme`: текущая тема ('light' | 'dark')
  - `toggleTheme`: функция переключения темы
- `useRouter()`: хук Next.js для программной навигации
  - Используется для перехода на `/account/forgot-password`

  2. `setIsLoading(true)` - устанавливает состояние загрузки (показывает индикатор)

  5. При ошибке: показывает сообщение об ошибке
  6. `setIsLoading(false)` - снимает состояние загрузки

### handleGoogleLogin()
- **Назначение**: Обработчик входа через Google OAuth
- **Параметры**: Нет
- **Логика**: 
  1. Выполняет полный редирект на Google OAuth endpoint
  2. После авторизации Google возвращает пользователя на callback URL
  3. Callback обрабатывает токен и редиректит на главную страницу

## Валидация формы

### Обязательные поля:
- **Email**: 
  - Должен быть валидным email адресом (валидация через `type="email"`)
  - HTML5 валидация проверяет формат email
- **Password**: 
  - Должен быть не пустым (валидация через `required`)
  - HTML5 валидация проверяет наличие значения

- **Клиентская валидация**:

## Структура JSX

### Основной контейнер
- **Flex** (width="100vw" height="100vh"): занимает весь экран
  - Центрирование содержимого (align="center" justify="center")
  - Фон адаптируется к теме (backgroundColor: 'var(--color-background)')
  - Отступы для мобильных устройств (padding: '20px')

### Контейнер формы
- **Box** (maxWidth: '400px'): ограничивает ширину формы
  - position: 'relative' для позиционирования кнопки темы

### Карточка формы
- **Box**: основной контейнер формы
  - padding: '32px' - внутренние отступы
  - backgroundColor: 'var(--color-panel)' - фон карточки
  - border, borderRadius, boxShadow - визуальное оформление

### Содержимое формы
- **Flex** (direction="column" gap="6"): вертикальное расположение элементов
  - Заголовок: название приложения и описание
  - Форма авторизации: поля email и password, кнопка входа
  - Разделитель "или"
  - Кнопка Google OAuth
  - Ссылка "Забыли пароль?"
