# Документация страницы: Восстановление пароля (account/forgot-password/page.tsx)

## Общее описание

Страница восстановления пароля. Позволяет запросить восстановление пароля через email. После отправки запроса показывает сообщение об успехе с инструкциями.

**Путь**: `/account/forgot-password`

## Компоненты страницы

### 1. Форма восстановления пароля
- **Компонент**: Форма с полем email
- **Назначение**: Запрос восстановления пароля
- **Поля**:
  - Email (обязательное, type="email")
- **Логика**: 
  - Валидация email через HTML5 (type="email")
  - Отправка запроса на восстановление на сервер
  - Показ сообщения об успехе после отправки
- **TODO**: Реализовать реальную отправку запроса

### 2. FloatingLabelInput
- **Расположение**: `@/components/FloatingLabelInput`
- **Назначение**: Поле ввода с плавающей меткой
- **Пропсы**: 
  - `id`: уникальный идентификатор поля
  - `label`: текст метки ("Email")
  - `type`: тип поля ('email')
  - `value`: значение поля (контролируемый компонент)
  - `onChange`: обработчик изменения значения
  - `placeholder`: подсказка в пустом поле
  - `required`: обязательное поле
  - `icon`: иконка конверта слева от поля
- **Логика**: Аналогична странице входа (метка перемещается при фокусе/заполнении)

### 3. Кнопка переключения темы
- **Компонент**: `Button` с иконкой `SunIcon`/`MoonIcon`
- **Назначение**: Переключение между светлой и темной темой
- **Логика**: Аналогична странице входа
- **Расположение**: 
  - Десктоп: справа от формы
  - Мобильные: в правом верхнем углу формы

### 4. Сообщение об успехе
- **Компонент**: `Box` с зеленым фоном
- **Назначение**: Информирование об успешной отправке инструкций
- **Логика**: 
  - Отображается только при `isSuccess === true`
  - Показывает email, на который отправлены инструкции
  - Предлагает проверить папку "Спам"
  - Кнопка возврата на страницу входа
- **Стили**: 
  - Зеленый фон (backgroundColor: 'var(--green-a3)')
  - Зеленая граница (border: '1px solid var(--green-a6)')

## Состояние компонента

### State переменные:
- `email` (string): значение поля email
  - Инициализация: `useState('')`
  - Обновление: через `onChange` обработчик поля ввода
  - Использование: отправка в API для восстановления пароля
- `isLoading` (boolean): флаг загрузки
  - Инициализация: `useState(false)`
  - Обновление: `true` при отправке запроса, `false` после завершения
  - Использование: блокировка кнопки и изменение текста на "Отправка..."
- `isSuccess` (boolean): флаг успешной отправки
  - Инициализация: `useState(false)`
  - Обновление: `true` после успешной отправки запроса
  - Использование: переключение между формой и сообщением об успехе

### Hooks:
- `useTheme()`: получение темы и функции переключения из ThemeProvider
- `useRouter()`: хук Next.js для программной навигации
  - Используется для перехода на `/account/login`

## Функции и обработчики

### handleSubmit()
- **Назначение**: Обработчик отправки запроса на восстановление пароля
- **Параметры**: Нет
- **Логика**: 
  1. Проверяет наличие email (`if (!email) return`)
  2. `setIsLoading(true)` - устанавливает состояние загрузки
  3. Отправляет запрос на сервер для отправки инструкций (TODO: реализовать)
  4. При успехе: `setIsLoading(false)` и `setIsSuccess(true)`
  5. При ошибке: показывает сообщение об ошибке

  - Отправка email с инструкциями и токеном сброса пароля

### handleBackToLogin()
- **Назначение**: Обработчик возврата на страницу входа
- **Параметры**: Нет
- **Логика**: 
  - Выполняет `router.push('/account/login')` для перехода на страницу входа
- **Использование**: 
  - Кнопка "Вернуться к входу" в сообщении об успехе
  - Кнопка "Вернуться к входу" в форме (если пользователь передумал)

### Условный рендеринг

  - Сообщение об успешной отправке

  - Форма ввода email

  - Кнопка возврата на страницу входа
