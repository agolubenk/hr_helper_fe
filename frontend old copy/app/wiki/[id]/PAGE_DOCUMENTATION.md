# Документация страницы: Детальный просмотр статьи вики (wiki/[id]/page.tsx)

## Общее описание

Страница детального просмотра статьи вики. Отображает полное содержимое статьи, историю изменений, навигацию по разделам и позволяет редактировать статью.

## Компоненты страницы

### 1. WikiDetailHeader
- **Расположение**: `@/components/wiki/WikiDetailHeader`
- **Назначение**: Заголовок статьи с метаданными
- **Пропсы**:
  - `title`: название статьи
  - `category`: категория статьи
  - `tags`: массив тегов
  - `author`: автор статьи
  - `lastEditor`: последний редактор
  - `created`: дата создания
  - `lastEdited`: дата последнего редактирования
  - `onEdit`: обработчик редактирования
- **Логика**: 
  - Отображает название, категорию, теги
  - Показывает информацию об авторе и редакторе
  - Кнопка редактирования статьи
- **Связи**: При клике на "Редактировать" переходит на `/wiki/[id]/edit`

### 2. WikiDetailContent
- **Расположение**: `@/components/wiki/WikiDetailContent`
- **Назначение**: Основное содержимое статьи
- **Пропсы**:
  - `content`: объект с heading и sections
- **Логика**: 
  - Отображает заголовок статьи
  - Отображает разделы статьи с подзаголовками
  - Форматирование текста (markdown или HTML)
- **Связи**: Использует данные из `wikiPage.content`

### 3. WikiDetailSidebar
- **Расположение**: `@/components/wiki/WikiDetailSidebar`
- **Назначение**: Боковая панель с навигацией по разделам
- **Пропсы**:
  - `sections`: массив разделов статьи
- **Логика**: 
  - Отображает список разделов статьи
  - Позволяет быстро перейти к нужному разделу
  - Прокрутка к разделу при клике
- **Связи**: Использует данные из `wikiPage.content.sections`

### 4. WikiDetailHistory
- **Расположение**: `@/components/wiki/WikiDetailHistory`
- **Назначение**: История изменений статьи
- **Пропсы**:
  - `history`: массив записей истории
- **Логика**: 
  - Отображает историю изменений статьи
  - Показывает автора, дату и описание изменений
  - Возможность просмотра предыдущих версий

## Функции и обработчики

### Загрузка данных статьи
- **Логика**: 
  - Читает ID статьи из URL параметра `[id]`
  - Загружает данные статьи по ID
  - Отображает содержимое, историю и метаданные
- **TODO**: Реализовать загрузку из API

## URL параметры

### [id]
- **Тип**: Динамический параметр маршрута
- **Назначение**: ID статьи вики для просмотра
- **Пример**: `/wiki/1` отображает статью с ID 1
- **Обработка**: 
  - Читается через `useParams()`
  - Используется для загрузки данных статьи

## Структура данных статьи

### wikiPage
Объект статьи с полями:
- `id`: уникальный идентификатор
- `title`: заголовок статьи
- `category`: категория статьи
- `tags`: массив тегов
- `author`: автор статьи
- `lastEditor`: последний редактор
- `created`: дата создания
- `lastEdited`: дата последнего редактирования
- `content`: объект с:
  - `heading`: заголовок статьи
  - `sections`: массив разделов с `title` и `items`
