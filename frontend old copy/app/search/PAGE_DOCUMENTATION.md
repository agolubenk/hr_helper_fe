# Документация страницы: Поиск (search/page.tsx)

## Общее описание

Страница результатов глобального поиска по приложению. Отображает результаты поиска с возможностью фильтрации по областям (вакансии, кандидаты, компании). Управляет областью поиска через URL параметры для поддержки глубоких ссылок и возможности делиться результатами поиска.

**Путь**: `/search`

## Компоненты страницы

### 1. SearchPage (обертка с Suspense)
- **Расположение**: `@/app/search/page.tsx`
- **Назначение**: Обертка для SearchPageContent с Suspense
- **Функциональность**: 
  - Оборачивает SearchPageContent в Suspense для обработки асинхронной загрузки
  - Показывает fallback (загрузку) пока компонент загружается
  - Необходимо для использования useSearchParams() внутри компонента
- **Поведение**: 
  - При загрузке показывает индикатор загрузки "Загрузка…"
  - После загрузки отображает SearchPageContent
- **Причина использования Suspense**: 
  - useSearchParams() требует обертки в Suspense для корректной работы с серверным рендерингом в Next.js

### 2. SearchPageContent (основной компонент)
- **Расположение**: `@/app/search/page.tsx`
- **Назначение**: Основной компонент содержимого страницы поиска
- **Функциональность**: 
  - Читает параметры поиска из URL
  - Отображает текущий запрос и область поиска
  - Предоставляет кнопки переключения области поиска
  - Обновляет URL при изменении области поиска
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - useSearchParams: получение параметров из URL
  - useRouter: для обновления URL при изменении области поиска

### 3. Отображение текущего запроса и области
- **Компонент**: Text с отображением запроса
- **Назначение**: Отображение текущего поискового запроса и области поиска
- **Функциональность**: 
  - Показывает запрос из URL параметра `?q=` (или '—' если пустой)
  - Показывает понятное название области поиска из SCOPE_LABELS
  - Формат: "Поиск: «запрос» — Область: название области"
- **Поведение**: 
  - Обновляется при изменении параметров URL
  - Отображает актуальное состояние поиска

### 4. Кнопки переключения области поиска
- **Компонент**: Flex с Button компонентами
- **Назначение**: Переключение области поиска
- **Кнопки**:
  - **"Показать везде"** (scope='all'): поиск во всех данных
  - **"Только вакансии"** (scope='vacancies'): поиск только в вакансиях
  - **"Только кандидаты"** (scope='candidates'): поиск только в кандидатах
  - **"Только компании"** (scope='companies'): поиск только в компаниях
- **Функциональность**: 
  - При клике обновляет URL параметр `?scope=`
  - Сохраняет поисковый запрос `?q=` при переключении области
  - Подсвечивает активную область (variant='solid' для активной, 'soft' для неактивной)
- **Поведение**: 
  - Активная кнопка имеет variant='solid'
  - Неактивные кнопки имеют variant='soft'
  - При клике вызывается setScope с соответствующей областью

### 5. Информационное сообщение
- **Компонент**: Text с информацией
- **Назначение**: Информационное сообщение о функциональности страницы
- **Функциональность**: 
  - Показывает сообщение о том, что здесь будут отображаться результаты поиска
  - Временное сообщение (TODO: заменить на реальные результаты)

## Интерфейсы и типы

### SCOPE_LABELS
```typescript
const SCOPE_LABELS: Record<string, string> = {
  all: 'Во всех данных',        // Поиск во всех данных
  vacancies: 'Вакансии',        // Поиск только в вакансиях
  candidates: 'Кандидаты',      // Поиск только в кандидатах
  companies: 'Компании',        // Поиск только в компаниях
}
```

## Состояние компонента

### State переменные:
- Нет state переменных (все данные получаются из URL параметров)

### Hooks:
- `useSearchParams()`: получение параметров из URL
  - Используется для: чтения параметров `?q=` и `?scope=`
  - Требует обертки в Suspense
- `useRouter()`: программная навигация
  - Используется для: обновления URL при изменении области поиска

### Константы:
- `SCOPE_LABELS`: маппинг областей поиска на отображаемые названия
  - Используется для: отображения понятного названия области поиска
  - Валидации области поиска

### Вычисляемые значения:
- `q`: поисковый запрос из URL параметра `?q=` (по умолчанию пустая строка)
  - Вычисляется: `searchParams.get('q') ?? ''`
- `scope`: область поиска из URL параметра `?scope=` (по умолчанию 'all')
  - Вычисляется: `searchParams.get('scope') ?? 'all'`
- `scopeLabel`: понятное название области поиска
  - Вычисляется: `(SCOPE_LABELS[scope] ?? scope) || 'Во всех данных'`

## Функции и обработчики

### setScope(s)
- **Назначение**: Установка области поиска
- **Параметры**: 
  - `s`: string - новая область поиска ('all', 'vacancies', 'candidates', 'companies')
- **Логика**: 
  1. Создает новый URLSearchParams из текущих параметров
  2. Сохраняет поисковый запрос `?q=` если он есть
  3. Устанавливает новую область поиска `?scope=`
  4. Выполняет навигацию на обновленный URL через router.push()
- **Поведение**: 
  - Вызывается при клике на кнопку области поиска
  - Сохраняет текущий запрос q в URL
  - Обновляет scope в URL
  - Выполняет переход на обновленный URL
- **Используется для**: 
  - Переключения области поиска
  - Синхронизации области поиска с URL

### scopeLabel (вычисляемое значение)
- **Назначение**: Получение отображаемого названия области поиска
- **Логика**: 
  1. Получает понятное название области из SCOPE_LABELS
  2. Если область не найдена - возвращает саму область или 'Во всех данных'
- **Используется для**: 
  - Отображения текущей области поиска
  - Гарантирует, что всегда есть отображаемое название

## URL параметры

### ?q=QUERY
- **Тип**: string - поисковый запрос
- **Назначение**: Поисковый запрос для поиска
- **Пример**: `/search?q=react` - поиск "react"
- **Обработка**: 
  - Читается через `useSearchParams().get('q')`
  - По умолчанию: пустая строка ('')
  - Отображается в заголовке страницы
  - Сохраняется при переключении области поиска
- **Поведение**: 
  - Если запрос пустой, отображается '—'
  - При переключении области поиска запрос сохраняется в URL

### ?scope=SCOPE
- **Тип**: string - область поиска ('all' | 'vacancies' | 'candidates' | 'companies')
- **Назначение**: Фильтрация результатов по области
- **Пример**: `/search?q=react&scope=vacancies` - поиск "react" только в вакансиях
- **Обработка**: 
  - Читается через `useSearchParams().get('scope')`
  - По умолчанию: 'all' (все данные)
  - Обновляется при клике на кнопку области
- **Поведение**: 
  - При изменении области обновляется URL параметр
  - Сохраняется поисковый запрос `?q=` при переключении области
