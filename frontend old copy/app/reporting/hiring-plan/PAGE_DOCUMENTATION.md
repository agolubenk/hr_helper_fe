# Документация страницы: План найма (reporting/hiring-plan/page.tsx)

## Общее описание

Страница плана найма (заявки на найм). Отображает заявки на найм с фильтрацией и поиском, статистику по заявкам (всего, планируется, в процессе, закрыто, отменено), метрики SLA и Time-to-Hire, а также позволяет перейти к годовой таблице плана найма.

**Путь**: `/reporting/hiring-plan`

## Компоненты страницы

### 1. HiringPlanPage (основной компонент)
- **Расположение**: `@/app/reporting/hiring-plan/page.tsx`
- **Назначение**: Главный компонент страницы плана найма
- **Функциональность**: 
  - Управление состоянием фильтрации и поиска
  - Вычисление статистики по заявкам
  - Вычисление метрик SLA и Time-to-Hire
  - Пагинация таблицы заявок
  - Переход к годовой таблице плана найма
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - Link: навигация к годовой таблице плана найма

### 2. Заголовок страницы
- **Компонент**: `Flex` с иконкой и текстом
- **Назначение**: Заголовок страницы с кнопкой перехода к годовой таблице
- **Логика**: 
  - Отображает иконку ListBulletIcon и текст "Заявки на найм"
  - Кнопка "Годовая таблица" переходит на `/reporting/hiring-plan/yearly`
- **Связи**: Навигация на `/reporting/hiring-plan/yearly`

### 3. Статистика по заявкам
- **Компонент**: Карточки с метриками (Card из Radix UI)
- **Назначение**: Отображение статистики по заявкам на найм
- **Метрики**: 
  - **Всего заявок**: общее количество отфильтрованных заявок
  - **Планируется**: количество запланированных заявок (status === 'planned')
  - **В процессе**: количество заявок в процессе (status === 'in_progress')
  - **Закрыто**: количество закрытых заявок (status === 'closed')
  - **Отменено**: количество отмененных заявок (status === 'cancelled')
  - **Выполнение**: процент выполнения (закрытые из общего числа)
  - **Среднее SLA (план)**: среднее значение SLA плана в днях для заявок со SLA
  - **Средний T2H (факт)**: среднее время до найма в днях для закрытых заявок
- **Особенности**: 
  - Каждая карточка имеет градиентный фон для визуального различия
  - Метрики обновляются в реальном времени при изменении фильтров

### 4. Фильтры
- **Компонент**: Card с полями фильтрации
- **Назначение**: Фильтрация заявок по различным критериям
- **Поля фильтрации**:
  - **Поиск**: поиск по вакансии, имени кандидата или проекту (TextField)
  - **Вакансия**: выбор вакансии из списка (Select)
  - **Рекрутер**: выбор рекрутера из списка (Select)
  - **Статус**: выбор статуса заявки (Select)
  - **Период**: выбор месяца в формате YYYY-MM (TextField type="month")
  - **Кнопка "Сбросить"**: сброс всех фильтров
- **Логика**: 
  - Все фильтры работают одновременно (логическое И)
  - Фильтрация происходит в реальном времени при изменении фильтров
  - Кнопка "Сбросить" очищает все фильтры и сбрасывает лимит пагинации

### 5. Таблица заявок
- **Компонент**: HTML table внутри Card
- **Назначение**: Отображение заявок на найм в табличном виде
- **Колонки**: 
  - **Вакансия / Грейд**: название вакансии с бейджем грейда и ссылкой на Huntflow
  - **Проект**: название проекта (или "—" если нет)
  - **Рекрутер**: список рекрутеров с количеством дней работы каждого
  - **Статус**: статус заявки с цветовой индикацией
  - **Сроки**: дата открытия — дедлайн (или "нет SLA")
  - **Факт / SLA**: фактическое количество дней / SLA в днях, статус SLA
  - **T2H | SLA**: время до найма в днях (или "—"), бейдж SLA
  - **Кандидат**: имя найденного кандидата со ссылкой на Huntflow (или "—")
- **Особенности**: 
  - Сортировка по дате открытия (новые сверху)
  - Пагинация с настраиваемым лимитом записей
  - Цветовая индикация статусов и SLA
  - Ссылки на Huntflow для вакансий и кандидатов

### 6. Пагинация
- **Компонент**: Select для выбора лимита записей
- **Назначение**: Ограничение количества отображаемых заявок
- **Опции**: 5, 10, 15, 25, 50, 100 заявок
- **Логика**: 
  - Показывает первые N заявок из отсортированного списка
  - Отображает текст "Показано: последние X из Y заявок"
  - При изменении лимита обновляется количество отображаемых заявок

## Интерфейсы и типы

### HiringRequest (интерфейс заявки на найм)
```typescript
interface HiringRequest {
  id: number                                    // Уникальный идентификатор заявки
  vacancy: string                               // Название вакансии
  grade: string                                 // Грейд (полный)
  grade_short: string                           // Грейд (сокращенный, например "M", "M+")
  project: string | null                        // Проект (может быть null)
  priority: 1 | 2 | 3                          // Приоритет (1 - высокий, 2 - средний, 3 - низкий)
  status: 'planned' | 'in_progress' | 'closed' | 'cancelled'  // Статус заявки
  opening_date: string | null                  // Дата открытия (YYYY-MM-DD)
  deadline: string | null                       // Дедлайн (YYYY-MM-DD)
  closed_date: string | null                    // Дата закрытия (YYYY-MM-DD)
  days_in_progress: number                      // Количество дней в работе
  sla_to_offer: number                          // SLA по времени до оффера (в днях)
  sla_status_display: string                    // Статус SLA ('В срок', 'Просрочено', 'Риск просрочки', 'Нормально', 'Нет SLA')
  time2hire: number | null                     // Время до найма в днях (может быть null)
  recruiters: Array<{                           // Массив рекрутеров
    name: string                                // Имя рекрутера
    days: number                                // Количество дней работы
  }>
  candidate_name: string | null                 // Имя найденного кандидата (может быть null)
  candidate_id: string | null                   // ID кандидата в Huntflow (может быть null)
  is_overdue: boolean                           // Флаг просрочки заявки
}
```

## Состояние компонента

### State переменные:
- `search` (string): поисковый запрос для фильтрации заявок
  - Инициализация: пустая строка
  - Обновление: при вводе в поле поиска
  - Использование: фильтрация по вакансии, имени кандидата или проекту

- `status` (string): выбранный статус для фильтрации
  - Инициализация: пустая строка (все статусы)
  - Обновление: при выборе в фильтре по статусу
  - Использование: фильтрация заявок по статусу

- `period` (string): выбранный период для фильтрации
  - Инициализация: пустая строка (все периоды)
  - Обновление: при выборе месяца в формате YYYY-MM
  - Использование: фильтрация заявок по месяцу открытия

- `vacancy` (string): выбранная вакансия для фильтрации
  - Инициализация: пустая строка (все вакансии)
  - Обновление: при выборе в фильтре по вакансиям
  - Использование: фильтрация заявок по вакансии

- `recruiter` (string): выбранный рекрутер для фильтрации
  - Инициализация: пустая строка (все рекрутеры)
  - Обновление: при выборе в фильтре по рекрутерам
  - Использование: фильтрация заявок по рекрутеру

- `tableLimit` (number): лимит записей на странице для пагинации
  - Инициализация: 10
  - Обновление: при выборе в Select лимита
  - Использование: ограничение количества отображаемых заявок

## Функции и обработчики

### fmt(d)
- **Назначение**: Форматирование даты из формата YYYY-MM-DD в DD.MM.YYYY
- **Параметры**: 
  - `d`: дата в формате YYYY-MM-DD или null
- **Возвращает**: string - отформатированная дата в формате DD.MM.YYYY или '—'
- **Логика**: 
  1. Проверяет наличие даты
  2. Разбивает строку по разделителю '-'
  3. Переставляет части: год, месяц, день → день, месяц, год
  4. Объединяет через точку
- **Используется для**: Отображения дат в таблице заявок

### statusBadge(s)
- **Назначение**: Получение CSS класса для бейджа статуса заявки
- **Параметры**: 
  - `s`: статус заявки (string)
- **Возвращает**: string - CSS класс для бейджа статуса
- **Логика**: 
  - Маппинг статусов на CSS классы через объект map
  - Возвращает соответствующий класс или класс по умолчанию
- **Используется для**: Цветовой индикации статуса заявки в таблице

### slaBadge(s)
- **Назначение**: Получение CSS класса для бейджа статуса SLA
- **Параметры**: 
  - `s`: статус SLA (string)
- **Возвращает**: string - CSS класс для бейджа статуса SLA
- **Логика**: 
  - Проверяет статус SLA и возвращает соответствующий CSS класс
  - 'В срок' → badgeSlaOk (зеленый)
  - 'Просрочено' → badgeSlaOver (красный)
  - 'Риск просрочки' → badgeSlaRisk (желтый/оранжевый)
  - другие → badgeSlaNormal (серый)
- **Используется для**: Цветовой индикации статуса SLA в таблице

## Вычисляемые значения

### filtered
- **Назначение**: Отфильтрованный список заявок на найм
- **Вычисление**: `MOCK_REQUESTS.filter((r) => { ... })`
- **Логика фильтрации** (все условия должны выполняться одновременно):
  1. **Поиск**: проверяет наличие запроса в названии вакансии, имени кандидата или проекте (без учета регистра)
  2. **Фильтр по вакансии**: если выбрана вакансия - проверяет точное совпадение
  3. **Фильтр по статусу**: если выбран статус - проверяет совпадение
  4. **Фильтр по периоду**: если выбран период (YYYY-MM) - проверяет совпадение месяца открытия
  5. **Фильтр по рекрутеру**: если выбран рекрутер - проверяет наличие в массиве recruiters
- **Используется для**: Сортировки и отображения в таблице

### sorted
- **Назначение**: Отсортированный список заявок
- **Вычисление**: `[...filtered].sort((a, b) => ...)`
- **Логика сортировки**: 
  - Сортирует по дате открытия (opening_date) в порядке убывания (новые сверху)
  - Использует localeCompare для сравнения дат в формате YYYY-MM-DD
- **Используется для**: Отображения заявок в хронологическом порядке

### tableRows
- **Назначение**: Список заявок для отображения в таблице (с учетом пагинации)
- **Вычисление**: `sorted.slice(0, tableLimit)`
- **Логика**: 
  - Берет первые tableLimit заявок из отсортированного списка
  - При изменении tableLimit обновляется количество отображаемых заявок
- **Используется для**: Отображения в таблице

### Статистика (s)
- **Назначение**: Статистика по заявкам
- **Вычисление**: 
  - `total_requests`: filtered.length
  - `planned_requests`: filtered.filter((r) => r.status === 'planned').length
  - `in_progress_requests`: filtered.filter((r) => r.status === 'in_progress').length
  - `closed_requests`: filtered.filter((r) => r.status === 'closed').length
  - `cancelled_requests`: filtered.filter((r) => r.status === 'cancelled').length
- **Используется для**: Отображения в карточках метрик

### fulfillment
- **Назначение**: Процент выполнения заявок
- **Вычисление**: `(closed_requests / total_requests) * 100`
- **Логика**: 
  - Округляется до целого числа
  - Если total_requests === 0, возвращает 0
- **Используется для**: Отображения метрики "Выполнение" в процентах

### Метрики SLA
- **withSla**: заявки со SLA (исключая запланированные и без SLA)
- **avgSlaPlan**: среднее значение SLA плана (в днях) для заявок со SLA
- **closedWithSla**: закрытые заявки
- **avgSlaFact**: среднее фактическое время выполнения (в днях) для закрытых заявок

### Метрики Time-to-Hire (T2H)
- **closedWithT2H**: закрытые заявки с указанным временем до найма
- **avgT2H**: среднее время до найма (в днях) для закрытых заявок

### vacancies
- **Назначение**: Массив уникальных названий вакансий
- **Вычисление**: `[...new Set(MOCK_REQUESTS.map((r) => r.vacancy).filter(Boolean))]`
- **Используется для**: Заполнения выпадающего списка фильтра по вакансиям

### recruiters
- **Назначение**: Массив уникальных имен рекрутеров
- **Вычисление**: `[...new Set(MOCK_REQUESTS.flatMap((r) => (r.recruiters || []).map((re) => re.name)).filter(Boolean))]`
- **Используется для**: Заполнения выпадающего списка фильтра по рекрутерам

## Фильтрация и поиск

### Поиск (search)
- **Компонент**: TextField в секции фильтров
- **Placeholder**: "Вакансия, кандидат, проект…"
- **Логика**: 
  - Ищет по названию вакансии (без учета регистра): `r.vacancy.toLowerCase().includes(q)`
  - Ищет по имени кандидата (без учета регистра): `(r.candidate_name || '').toLowerCase().includes(q)`
  - Ищет по проекту (без учета регистра): `(r.project || '').toLowerCase().includes(q)`
  - Любое из условий достаточно для прохождения фильтра
- **Поведение**: Фильтрация происходит в реальном времени при вводе текста

### Фильтр по вакансии (vacancy)
- **Компонент**: Select в секции фильтров
- **Опции**: 
  - "Все вакансии" (value: '__all__')
  - Список уникальных вакансий из MOCK_REQUESTS
- **Логика**: 
  - Если выбрано "Все вакансии" - пропускает все заявки
  - Иначе фильтрует по точному совпадению: `r.vacancy !== vacancy`

### Фильтр по рекрутеру (recruiter)
- **Компонент**: Select в секции фильтров
- **Опции**: 
  - "Все рекрутеры" (value: '__all__')
  - Список уникальных рекрутеров из всех заявок
- **Логика**: 
  - Если выбрано "Все рекрутеры" - пропускает все заявки
  - Иначе проверяет наличие рекрутера в массиве: `(r.recruiters || []).some((re) => re.name === recruiter)`

### Фильтр по статусу (status)
- **Компонент**: Select в секции фильтров
- **Опции**: 
  - "Все статусы" (value: '')
  - "Планируется" (value: 'planned')
  - "В процессе" (value: 'in_progress')
  - "Закрыта" (value: 'closed')
  - "Отменена" (value: 'cancelled')
- **Логика**: 
  - Если выбрано "Все статусы" - пропускает все заявки
  - Иначе фильтрует по совпадению: `r.status !== status`

### Фильтр по периоду (period)
- **Компонент**: TextField type="month" в секции фильтров
- **Формат**: YYYY-MM (например, "2025-12")
- **Логика**: 
  - Если период не выбран - пропускает все заявки
  - Иначе проверяет совпадение месяца открытия: сравнивает год и месяц из opening_date

### Кнопка "Сбросить"
- **Компонент**: Button с иконкой Cross2Icon
- **Назначение**: Сброс всех фильтров
- **Логика**: 
  - Очищает все фильтры: search, status, period, vacancy, recruiter
  - Сбрасывает лимит пагинации в 10
  - Обновляет отображаемые заявки

## Особенности

- **Множественная фильтрация**: 
  - Все фильтры работают одновременно (логическое И)
  - Фильтрация происходит в реальном времени при изменении фильтров
  - Кнопка "Сбросить" очищает все фильтры одним кликом

- **Статистика в реальном времени**: 
  - Статистика обновляется автоматически при изменении фильтров
  - Метрики вычисляются на основе отфильтрованных заявок
  - Градиентные фоны карточек для визуального различия метрик

- **Метрики SLA и T2H**: 
  - Отображение плановых и фактических метрик
  - Средние значения вычисляются только для релевантных заявок
  - Отображение "—" если метрика не применима

- **Пагинация**: 
  - Настраиваемый лимит записей на странице
  - Отображение количества показанных и общих заявок
  - Опции: 5, 10, 15, 25, 50, 100 заявок

- **Сортировка**: 
  - Автоматическая сортировка по дате открытия (новые сверху)
  - Использует localeCompare для корректного сравнения дат

- **Ссылки на Huntflow**: 
  - Ссылки на вакансии и кандидатов в Huntflow
  - Открываются в новой вкладке
  - Формат ссылок: `https://app.huntflow.ru/vacancy/{id}` и `https://app.huntflow.ru/candidate/{id}`

- **Цветовая индикация**: 
  - Статусы заявок с цветовой индикацией
  - Статусы SLA с цветовой индикацией (зеленый - в срок, красный - просрочено, желтый - риск)
  - Бейджи грейдов для визуального различия
