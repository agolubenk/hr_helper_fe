# Документация компонентов: aichat

## Общее описание

Директория `components/aichat` содержит компоненты для AI чата - интерфейса общения с искусственным интеллектом. Компоненты обеспечивают отображение сообщений, форматирование текста, ввод сообщений, управление историей чатов и заголовком чата.

**Расположение**: `@/components/aichat`

## Компоненты

### 1. FormattedText
- **Файл**: `FormattedText.tsx`
- **Назначение**: Компонент для отображения форматированного текста с markdown-подобным синтаксисом
- **Пропсы**:
  - `content`: текст для форматирования (string)
  - `disableHeadings`: отключить обработку заголовков (boolean, опционально, по умолчанию false)
- **Функциональность**: 
  - Парсинг markdown-подобного синтаксиса
  - Поддержка жирного, курсивного, подчеркнутого, зачеркнутого текста
  - Поддержка заголовков (# H1, ## H2, ### H3, и т.д.)
  - Поддержка блоков кода (```) с кнопкой копирования
  - Поддержка inline кода (`)
  - Поддержка ссылок [текст](url) с превью
  - Поддержка цитат (>)
  - Сохранение переносов строк
- **Особенности**: 
  - Рекурсивная обработка вложенного форматирования
  - Приоритетная обработка блоков кода
  - Превью ссылок отображается под сообщением через сепаратор
  - Кнопка копирования для блоков кода с подтверждением
- **Используется в**: 
  - ChatMessages: для отображения сообщений от ассистента
  - Telegram чатах: для форматирования сообщений (с disableHeadings=true)

### 2. ChatMessages
- **Файл**: `ChatMessages.tsx`
- **Назначение**: Компонент отображения списка сообщений в чате
- **Пропсы**:
  - `messages`: массив сообщений (Message[])
- **Функциональность**: 
  - Отображение сообщений пользователя и ассистента
  - Разные стили для сообщений пользователя и ассистента
  - Отображение аватаров (G для ассистента, PersonIcon для пользователя)
  - Отображение прикрепленных файлов
  - Tooltip для списка файлов (если файлов больше одного)
  - Отображение временных меток сообщений
  - Пустое состояние при отсутствии сообщений
- **Особенности**: 
  - Сообщения ассистента используют FormattedText для форматирования
  - Сообщения пользователя отображаются как обычный текст
  - Файлы отображаются с размером (форматирование через formatFileSize)
  - Использует Radix UI Tooltip для отображения списка файлов
- **Используется в**: 
  - `/app/aichat/page.tsx`: основной компонент страницы AI чата

### 3. ChatInput
- **Файл**: `ChatInput.tsx`
- **Назначение**: Обертка для AnimatedAIInput
- **Пропсы**:
  - `onSend`: обработчик отправки сообщения ((message: string, modelId?: string, files?: File[]) => void)
  - `onFileAttach`: обработчик прикрепления файла ((file: File) => void, опционально)
- **Функциональность**: 
  - Простая обертка, передает все пропсы в AnimatedAIInput
- **Используется в**: 
  - `/app/aichat/page.tsx`: для ввода сообщений

### 4. AnimatedAIInput
- **Файл**: `AnimatedAIInput.tsx`
- **Назначение**: Компонент ввода сообщений с выбором AI модели и прикреплением файлов
- **Пропсы**:
  - `onSend`: обработчик отправки сообщения ((message: string, modelId?: string, files?: File[]) => void)
  - `onFileAttach`: обработчик прикрепления файла ((file: File) => void, опционально)
  - `placeholder`: placeholder для поля ввода (string, опционально, по умолчанию "Введите ваше сообщение...")
  - `disabled`: отключить ввод (boolean, опционально, по умолчанию false)
- **Функциональность**: 
  - Поле ввода сообщения (TextArea с автоизменением высоты)
  - Выбор AI модели через Popover (o3-mini, Gemini 2.5 Flash, Claude 3.5 Sonnet, GPT-4-1 Mini, GPT-4-1)
  - Прикрепление файлов (drag-and-drop и через кнопку)
  - Отображение списка прикрепленных файлов
  - Удаление прикрепленных файлов
  - Отправка сообщения по Enter (без Shift) или по кнопке
  - Автоматическое изменение высоты textarea
- **Особенности**: 
  - Поддержка drag-and-drop для файлов
  - Визуальные иконки для разных AI моделей
  - Отображение выбранной модели в кнопке
  - Валидация перед отправкой (сообщение или файлы должны быть)
- **Используется в**: 
  - ChatInput: обертка для этого компонента

### 5. ChatHeader
- **Файл**: `ChatHeader.tsx`
- **Назначение**: Заголовок чата с редактированием названия и удалением
- **Пропсы**:
  - `title`: название чата (string)
  - `createdAt`: дата создания (string, опционально)
  - `onTitleChange`: обработчик изменения названия ((newTitle: string) => void)
  - `onDelete`: обработчик удаления чата (() => void)
- **Функциональность**: 
  - Отображение названия чата
  - Редактирование названия по клику (inline редактирование)
  - Кнопки сохранения и отмены при редактировании
  - Кнопка удаления чата (показывается при hover)
  - Поддержка клавиш Enter (сохранить) и Escape (отменить)
- **Особенности**: 
  - Автоматический фокус и выделение текста при начале редактирования
  - Валидация: название не может быть пустым
  - Иконки появляются при hover
- **Используется в**: 
  - `/app/aichat/page.tsx`: заголовок чата

### 6. ChatHistory
- **Файл**: `ChatHistory.tsx`
- **Назначение**: Компонент истории чатов (боковая панель)
- **Пропсы**:
  - `chats`: массив чатов (Chat[])
  - `selectedChatId`: ID выбранного чата (string)
  - `onChatSelect`: обработчик выбора чата ((chatId: string) => void)
  - `onNewChat`: обработчик создания нового чата (() => void)
  - `onChatDelete`: обработчик удаления чата ((chatId: string) => void)
- **Функциональность**: 
  - Отображение списка чатов
  - Выделение выбранного чата
  - Отображение названия и даты создания чата
  - Кнопка удаления чата (показывается при hover)
  - Кнопка "Новый чат"
  - Заголовок "История чатов"
- **Особенности**: 
  - Фиксированная боковая панель
  - Hover эффекты для элементов списка
  - Кнопка удаления появляется при hover на элементе
- **Используется в**: 
  - `/app/aichat/page.tsx`: боковая панель с историей чатов

## Поддерживаемые форматы форматирования (FormattedText)

### Inline форматирование:
- **Жирный**: `**текст**` или `__текст__`
- **Курсив**: `*текст*` или `_текст_`
- **Жирный и курсив**: `***текст***` или `___текст___`
- **Подчеркнутый**: `<u>текст</u>`
- **Зачеркнутый**: `~~текст~~`
- **Inline код**: `` `код` ``
- **Ссылки**: `[текст](url)`

### Блочные элементы:
- **Заголовки**: `# H1`, `## H2`, `### H3`, `#### H4`, `##### H5`, `###### H6`
- **Блок кода**: ` ```код``` `
- **Цитаты**: `> текст цитаты`
- **Переносы строк**: автоматически сохраняются

### Особенности парсинга:
- Приоритет обработки: блоки кода > ссылки > подчеркнутый > зачеркнутый > inline код > жирный+курсив > жирный > курсив
- Рекурсивная обработка вложенного форматирования
- Превью ссылок отображается под сообщением через сепаратор

## Функции и утилиты

### parseFormattedText
- **Назначение**: Парсинг текста с markdown-подобным синтаксисом
- **Параметры**: 
  - `text`: текст для парсинга (string)
  - `disableHeadings`: отключить обработку заголовков (boolean)
- **Возвращает**: массив токенов (Token[])
- **Логика**: 
  1. Сначала обрабатывает блоки кода (приоритет)
  2. Затем обрабатывает заголовки и цитаты
  3. Затем обрабатывает inline форматирование

### parseTextWithHeadings
- **Назначение**: Парсинг текста с заголовками и цитатами
- **Параметры**: 
  - `text`: текст для парсинга (string)
  - `disableHeadings`: отключить обработку заголовков (boolean)
- **Возвращает**: массив токенов (Token[])
- **Логика**: 
  - Обрабатывает строки построчно
  - Определяет заголовки, цитаты и обычный текст
  - Сохраняет переносы строк

### parseInlineFormatting
- **Назначение**: Парсинг inline форматирования
- **Параметры**: 
  - `text`: текст для парсинга (string)
- **Возвращает**: массив токенов (Token[])
- **Логика**: 
  - Рекурсивно обрабатывает вложенное форматирование
  - Приоритетная обработка различных типов форматирования

### renderToken
- **Назначение**: Рендеринг токена в React элемент
- **Параметры**: 
  - `token`: токен для рендеринга (Token)
  - `key`: ключ для React (number)
- **Возвращает**: React элемент (ReactNode)
- **Логика**: 
  - Рендерит токен с соответствующим форматированием
  - Для форматированных токенов рекурсивно обрабатывает содержимое

### formatFileSize
- **Назначение**: Форматирование размера файла
- **Параметры**: 
  - `bytes`: размер в байтах (number)
- **Возвращает**: отформатированная строка (string)
- **Логика**: 
  - < 1024 байт: "X B"
  - < 1024 KB: "X.X KB"
  - >= 1024 KB: "X.X MB"
