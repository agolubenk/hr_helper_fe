# API Pre-Specifications

## Общее описание

В директории `frontend` созданы JSON-файлы `pre-specification.json` для каждой страницы и компонента, которые содержат предварительную спецификацию API на основе моковых данных и документации.

## Статистика

- **Всего спецификаций**: 76
- **Всего API эндпоинтов**: 501
- **Всего интерфейсов**: 113
- **Всего моковых данных**: 104
- **Страниц с API эндпоинтами**: 52
- **Компонентов с API эндпоинтами**: 15

## Структура спецификации

Каждый файл `pre-specification.json` содержит следующую структуру:

```json
{
  "name": "название_страницы_или_компонента",
  "type": "page" | "component",
  "description": "Описание страницы/компонента",
  "route_path": "Путь маршрута (только для страниц)",
  "documentation_file": "Путь к файлу документации",
  "mock_data": {
    "название_моковых_данных": {
      "description": "Описание моковых данных",
      "extracted_from": "documentation"
    }
  },
  "interfaces": [
    {
      "name": "Название интерфейса",
      "definition": "Полное определение интерфейса",
      "fields": [
        {
          "name": "название_поля",
          "type": "тип_поля",
          "optional": true/false,
          "comment": "Комментарий к полю"
        }
      ]
    }
  ],
  "api_endpoints": [
    {
      "task_number": 1,
      "description": "Описание задачи",
      "method": "GET" | "POST" | "PUT" | "PATCH" | "DELETE",
      "endpoint": "/api/path/to/endpoint/",
      "parameters": "Описание параметров",
      "returns": "Описание возвращаемых данных",
      "status": "pending"
    }
  ],
  "generated_at": "2026-01-28T17:59:39.786636"
}
```

## Расположение файлов

Спецификации находятся в тех же директориях, что и соответствующие файлы документации:

- **Страницы**: `app/{путь_к_странице}/pre-specification.json`
- **Компоненты**: `components/{путь_к_компоненту}/pre-specification.json`

## Использование

### Для разработчиков Backend

1. Откройте `pre-specification.json` для нужной страницы/компонента
2. Изучите раздел `api_endpoints` - там перечислены все необходимые эндпоинты
3. Используйте раздел `interfaces` для понимания структуры данных
4. Используйте раздел `mock_data` для понимания текущих моковых данных

### Для разработчиков Frontend

1. Используйте спецификации как справочник при интеграции с API
2. Обновляйте статус эндпоинтов (`status`) по мере реализации:
   - `pending` - не реализовано
   - `in_progress` - в разработке
   - `completed` - реализовано
   - `testing` - в тестировании

### Пример использования

```typescript
// Загрузка спецификации
import spec from './pre-specification.json';

// Получение всех эндпоинтов
const endpoints = spec.api_endpoints;

// Фильтрация по методу
const getEndpoints = endpoints.filter(e => e.method === 'GET');

// Получение интерфейсов
const interfaces = spec.interfaces;
const ChatInterface = interfaces.find(i => i.name === 'TelegramChat');
```

## Генерация спецификаций

Спецификации генерируются автоматически скриптом `generate_api_specs.py`:

```bash
cd frontend
python3 generate_api_specs.py
```

Скрипт:
1. Находит все файлы `PAGE_DOCUMENTATION.md` и `COMPONENTS_DOCUMENTATION.md`
2. Извлекает информацию о моковых данных
3. Извлекает интерфейсы и типы
4. Извлекает задачи из раздела "TODO: Интеграция с API"
5. Создает JSON-файлы `pre-specification.json`

## Обновление спецификаций

При изменении документации необходимо перегенерировать спецификации:

```bash
cd frontend
python3 generate_api_specs.py
```

## Структура данных

### Mock Data

Раздел `mock_data` содержит описание всех моковых данных, используемых на странице/в компоненте. Каждая запись включает:
- Название моковых данных
- Описание структуры
- Поля и их типы
- TODO для замены на реальные данные

### Interfaces

Раздел `interfaces` содержит все TypeScript интерфейсы, используемые на странице/в компоненте. Каждый интерфейс включает:
- Название
- Полное определение
- Список полей с типами, опциональностью и комментариями

### API Endpoints

Раздел `api_endpoints` содержит список всех необходимых API эндпоинтов. Каждый эндпоинт включает:
- Номер задачи
- Описание
- HTTP метод (GET, POST, PUT, PATCH, DELETE)
- Путь эндпоинта
- Параметры запроса
- Возвращаемые данные
- Статус реализации

## Примечания

- Спецификации созданы автоматически на основе документации
- Методы и эндпоинты извлекаются из раздела "TODO: Интеграция с API"
- Некоторые эндпоинты могут быть описаны не полностью - требуется дополнительная спецификация
- При реализации API рекомендуется обновлять статусы эндпоинтов в спецификациях
