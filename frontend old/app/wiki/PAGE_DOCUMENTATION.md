# Документация страницы: Вики (wiki/page.tsx)

## Общее описание

Главная страница вики (базы знаний). Отображает список статей вики, позволяет фильтровать по тегам и поисковому запросу, группирует статьи по категориям. Страница является центральным хабом для доступа к документации и знаниям о системе HR Helper.

**Путь**: `/wiki`

## Интерфейсы и типы

### WikiPage (интерфейс статьи вики)
```typescript
interface WikiPage {
  id: string              // Уникальный идентификатор статьи
  title: string           // Название статьи
  description: string     // Краткое описание статьи
  tags: string[]         // Массив тегов статьи (с префиксом #, например, ['#настройка', '#финансы'])
  date: string           // Дата создания или последнего обновления (формат DD.MM.YYYY)
  category: string       // Категория статьи (например, 'Архитектура', 'Настройка')
}
```

### Tag (интерфейс тега)
```typescript
interface Tag {
  id: string              // Уникальный идентификатор тега (без префикса #)
  label: string          // Отображаемое название тега (с префиксом #)
  color: string          // Цвет тега для визуального отображения (hex код)
}
```

## Функции и обработчики

### Фильтрация статей (filteredPages)
- **Назначение**: Отфильтрованный список страниц вики по выбранным критериям
- **Вычисление**: `wikiPages.filter(page => { ... })`
- **Логика фильтрации** (оба условия должны выполняться одновременно):
  1. **Фильтр по тегу** (`matchesTag`): 
     - Если `selectedTag === null` - пропускает все статьи (не фильтрует по тегу)
     - Иначе проверяет наличие выбранного тега в массиве тегов статьи
     - Сравнение выполняется без учета регистра: `tag.toLowerCase() === selectedTag.toLowerCase()`
     - Использует `page.tags.some()` для проверки наличия тега
  2. **Фильтр по поиску** (`matchesSearch`): 
     - Если `searchQuery` пустой - пропускает все статьи (не фильтрует по поиску)
     - Иначе проверяет наличие поискового запроса в названии или описании
     - Сравнение выполняется без учета регистра (toLowerCase)
     - Проверяет: `page.title.toLowerCase().includes(searchQuery.toLowerCase())` ИЛИ `page.description.toLowerCase().includes(searchQuery.toLowerCase())`
- **Использование**: 
  - Используется для группировки по категориям
  - Отображается в WikiCategory компонентах
- **Особенности**: 
  - Оба фильтра применяются одновременно (логическое И)
  - Фильтрация происходит в реальном времени при изменении фильтров

### Группировка по категориям (categories)
- **Назначение**: Массив уникальных категорий из отфильтрованных страниц
- **Вычисление**: `Array.from(new Set(filteredPages.map(page => page.category)))`
- **Логика**: 
  1. Извлекает все категории из отфильтрованных страниц через `map(page => page.category)`
  2. Создает Set для получения уникальных значений
  3. Преобразует Set в массив через `Array.from()`
- **Использование**: 
  - Используется для определения порядка отображения категорий
  - Передается в sortedCategories для сортировки

### Сортировка категорий (sortedCategories)
- **Назначение**: Отсортированный массив категорий для отображения
- **Вычисление**: `[...categories].sort((a, b) => { ... })`
- **Логика сортировки**: 
  1. Категория "Архитектура" всегда первая (приоритет)
     - Если `a === 'Архитектура'` - возвращает -1 (a идет раньше)
     - Если `b === 'Архитектура'` - возвращает 1 (b идет раньше)
  2. Остальные категории сортируются по алфавиту
     - Использует `a.localeCompare(b)` для алфавитной сортировки
- **Использование**: 
  - Определяет порядок отображения категорий на странице
  - Обеспечивает, что категория "Архитектура" всегда видна первой
- **Особенности**: 
  - Создает копию массива через spread оператор `[...categories]`
  - Не изменяет оригинальный массив categories

### Группировка страниц по категориям (groupedPages)
- **Назначение**: Объект сгруппированных страниц по категориям
- **Вычисление**: `sortedCategories.reduce((acc, category) => { ... }, {} as Record<string, WikiPage[]>)`
- **Структура**: 
  - Ключ: название категории (string)
  - Значение: массив страниц этой категории (WikiPage[])
- **Логика**: 
  1. Проходит по отсортированному списку категорий
  2. Для каждой категории фильтрует страницы: `filteredPages.filter(page => page.category === category)`
  3. Добавляет категорию в объект только если в ней есть хотя бы одна страница (`if (pages.length > 0)`)
  4. Возвращает объект `Record<string, WikiPage[]>`
- **Использование**: 
  - Используется для рендеринга WikiCategory компонентов
  - Порядок категорий соответствует sortedCategories
- **Особенности**: 
  - Включает только категории, в которых есть хотя бы одна страница после фильтрации
  - Если все статьи категории отфильтрованы - категория не отображается

## Группировка и отображение

### Группировка по категориям
- **Логика**: 
  - Все отфильтрованные статьи группируются по полю `category`
  - Каждая категория отображается отдельным блоком через `WikiCategory`
  - Внутри категории статьи отображаются через `WikiCard`
- **Структура**: 
  - Объект `groupedPages` где ключ - категория, значение - массив статей
  - Используется для рендеринга: `Object.entries(groupedPages).map(([category, pages]) => ...)`

### Сортировка категорий
- **Правило**: 
  - Категория "Архитектура" всегда первая (приоритет)
  - Остальные категории сортируются по алфавиту (localeCompare)
- **Реализация**: 
  ```typescript
  const sortedCategories = [...categories].sort((a, b) => {
    if (a === 'Архитектура') return -1  // "Архитектура" всегда первая
    if (b === 'Архитектура') return 1
    return a.localeCompare(b)  // Остальные по алфавиту
  })
  ```
- **Особенности**: 
  - Обеспечивает, что категория "Архитектура" всегда видна первой
  - Остальные категории в алфавитном порядке

## Особенности

- **Группировка по категориям**: 
  - Все статьи автоматически группируются по категориям
  - Каждая категория отображается отдельным блоком

- **Специальная сортировка категорий**: 
  - Категория "Архитектура" всегда первая (приоритет)
  - Остальные категории в алфавитном порядке

- **Фильтрация по тегам и поиску одновременно**: 
  - Оба фильтра применяются одновременно (логическое И)
  - Фильтрация происходит в реальном времени

- **Поддержка множественных тегов**: 
  - Каждая статья может иметь несколько тегов
  - Теги отображаются с цветовой индикацией

- **Выделение первой статьи**: 
  - Первая страница в категории "Архитектура" с id='1' выделяется (класс highlighted)
  - Используется для акцентирования важной статьи

- **Цветовая индикация тегов**: 
  - Каждый тег имеет свой цвет для визуального различия
  - Цвета используются в WikiFilters и WikiCard

- **Реальная фильтрация**: 
  - Фильтрация происходит в реальном времени при изменении фильтров
  - Не требует нажатия кнопки "Поиск"
