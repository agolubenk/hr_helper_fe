# Документация страницы: Шаблон оффера (company-settings/recruiting/offer-template/page.tsx)

## Общее описание

Страница управления шаблоном оффера (предложения о работе). Позволяет загружать шаблоны в форматах DOCX, PPTX, Figma, настраивать переменные для подстановки и просматривать предпросмотр с подставленными значениями.

## Компоненты страницы

### 1. Загрузка файла шаблона
- **Компонент**: Область drag & drop или кнопка выбора файла
- **Назначение**: Загрузка файла шаблона оффера
- **Поддерживаемые форматы**: 
  - `.docx` - документы Word
  - `.pptx` - презентации PowerPoint
  - `.figma` - файлы Figma (TODO: реализовать обработку на сервере)
- **Логика**: 
  - Drag & drop для перетаскивания файла
  - Кнопка выбора файла
  - Валидация формата файла
  - Автоматическая генерация предпросмотра после загрузки
- **Связи**: 
  - mammoth.js: конвертация DOCX в HTML
  - JSZip: парсинг PPTX файлов

### 2. Предпросмотр шаблона
- **Компонент**: Контейнер с HTML содержимым
- **Назначение**: Отображение предпросмотра шаблона с подставленными переменными
- **Логика**: 
  - Для DOCX: конвертация в HTML через mammoth.js
  - Для PPTX: парсинг и отображение слайдов
  - Автоматическое масштабирование под размер контейнера
  - На мобильных: открывается в модальном окне
- **Связи**: Использует загруженный файл и переменные

### 3. Настройка переменных
- **Компонент**: Форма с полями для переменных
- **Назначение**: Настройка переменных для подстановки в шаблон
- **Логика**: 
  - Автоматическое извлечение переменных из шаблона (формат: {variable_name})
  - Поля для ввода значений переменных
  - Автоматическое обновление предпросмотра при изменении переменных
- **Связи**: Используется для подстановки значений в предпросмотр

### 4. Модальное окно предпросмотра (мобильные)
- **Компонент**: Dialog с предпросмотром
- **Назначение**: Отображение предпросмотра в модальном окне на мобильных устройствах
- **Логика**: 
  - Открывается при клике на кнопку предпросмотра
  - Показывает полный предпросмотр шаблона
  - Закрывается при клике вне окна или на кнопку закрытия
- **Связи**: Используется только на мобильных устройствах

## Состояние компонента

### State переменные:
- `uploadedFile`: загруженный файл шаблона (File | null)
- `fileType`: тип файла ('docx' | 'pptx' | 'figma' | null)
- `previewHtml`: HTML содержимое для предпросмотра
- `variables`: объект переменных для подстановки (ключ - имя переменной, значение - значение)
- `isPreviewModalOpen`: флаг открытия модального окна предпросмотра (для мобильных)
- `previewScale`: масштаб предпросмотра (для автоматического масштабирования)

## Функции и обработчики

### handleFileUpload(file)
- **Назначение**: Обработка загрузки файла шаблона
- **Логика**: 
  1. Валидирует формат файла (.docx, .pptx, .figma)
  2. Определяет тип файла
  3. Генерирует предпросмотр:
     - Для DOCX: конвертирует в HTML через mammoth.js
     - Для PPTX: парсит через JSZip и отображает слайды
     - Для Figma: показывает сообщение о необходимости обработки на сервере
  4. Извлекает переменные из шаблона (формат: {variable_name})
  5. Обновляет состояние
- **TODO**: Реализовать обработку Figma файлов на сервере

### handleVariableChange(variableName, value)
- **Назначение**: Изменение значения переменной
- **Логика**: 
  - Обновляет значение переменной в объекте `variables`
  - Автоматически обновляет предпросмотр с новыми значениями
- **Поведение**: Вызывается при изменении поля переменной

### updatePreview()
- **Назначение**: Обновление предпросмотра с подставленными переменными
- **Логика**: 
  - Заменяет переменные в формате {variable_name} на их значения
  - Обновляет HTML предпросмотра
- **Вызывается**: 
  - При изменении переменных
  - При загрузке файла

### calculatePreviewScale()
- **Назначение**: Расчет масштаба предпросмотра для автоматического масштабирования
- **Логика**: 
  - Определяет оптимальный масштаб для отображения содержимого
  - Учитывает размер контейнера и размер содержимого
- **Используется**: Для автоматического масштабирования предпросмотра

### handleSave()
- **Назначение**: Сохранение шаблона на сервер
- **Логика**: 
  1. Отправляет файл и переменные на сервер
  2. При успехе показывает toast
  3. При ошибке показывает сообщение об ошибке
