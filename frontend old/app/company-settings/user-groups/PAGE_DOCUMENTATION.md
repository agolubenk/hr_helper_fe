# Документация страницы: Группы пользователей (company-settings/user-groups/page.tsx)

## Общее описание

Страница управления группами пользователей. Позволяет создавать, редактировать и удалять группы, управлять правами доступа групп.

## Компоненты страницы

### 1. Список групп
- **Компонент**: Таблица или список
- **Назначение**: Отображение всех групп пользователей
- **Логика**: 
  - Отображает список групп
  - Показывает количество пользователей в группе
  - Кнопки редактирования и удаления

### 2. GroupAccessModal
- **Расположение**: `@/components/company-settings/GroupAccessModal`
- **Назначение**: Модальное окно управления правами доступа группы
- **Пропсы**: 
  - `isOpen`: флаг открытости
  - `onClose`: обработчик закрытия
  - `groupId`: ID группы
  - `currentAccess`: текущие права доступа
  - `onSave`: обработчик сохранения прав
- **Логика**: 
  - Отображение текущих прав доступа
  - Редактирование прав доступа
  - Сохранение через API
- **Связи**: Используется для управления правами групп

### 3. Форма создания/редактирования группы
- **Компонент**: Dialog с формой
- **Назначение**: Создание или редактирование группы
- **Поля**:
  - Название группы
  - Описание
  - Права доступа

## Состояние компонента

### State переменные:
- `groups`: массив всех групп
- `editingGroup`: группа для редактирования
- `showAddForm`: флаг отображения формы добавления

## Функции и обработчики

### CRUD операции:
- **Создание**: Добавление новой группы
- **Чтение**: Загрузка списка групп
- **Обновление**: Редактирование существующей группы
- **Удаление**: Удаление группы с подтверждением

## Моковые данные

### mockGroups
Массив объектов групп с полями:
- `id`: уникальный идентификатор
- `name`: название группы (Администраторы, Рекрутеры, Менеджеры, HR)
- `description`: описание группы
- `userCount`: количество пользователей в группе
- `access`: права доступа группы
