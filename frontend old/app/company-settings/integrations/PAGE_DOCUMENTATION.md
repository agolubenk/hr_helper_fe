# Документация страницы: Интеграции (company-settings/integrations/page.tsx)

## Общее описание

Страница управления интеграциями компании. Позволяет настраивать интеграции с внешними сервисами (Huntflow, Google, Telegram, ИИ сервисы и т.д.), управлять их активностью и для части интеграций выбирать режим хранения учетных данных (общий/у каждого свой/оба). Для Google выбора режима нет — подключение всегда индивидуальное (включение выполняется в профиле, у каждого пользователя своё).

**Путь**: `/company-settings/integrations`

## Компоненты страницы

### 1. IntegrationsSettingsPage (основной компонент)
- **Расположение**: `@/app/company-settings/integrations/page.tsx`
- **Назначение**: Главный компонент страницы управления интеграциями
- **Функциональность**: 
  - Отображение списка всех доступных интеграций
  - Фильтрация интеграций по группам
  - Управление активностью интеграций
  - Открытие модального окна выбора области применения
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - IntegrationScopeModal: модальное окно выбора области применения
  - IntegrationCard: компонент карточки интеграции

### 2. IntegrationCard (внутренний компонент)
- **Расположение**: `@/app/company-settings/integrations/page.tsx` (внутри файла)
- **Назначение**: Карточка отдельной интеграции
- **Пропсы**:
  - `item`: объект интеграции (Integration)
  - `isActive`: текущее состояние активности (boolean)
  - `onActiveChange`: обработчик изменения активности ((value: boolean) => void)
  - `onOpenScopeModal`: обработчик открытия модального окна выбора области ((item: Integration) => void)
- **Функциональность**: 
  - Отображение информации об интеграции (название, иконка, статус)
  - Переключатель активности (Switch)
  - Кнопка "Настроить"/"Подключить"
  - Определение действия при клике на кнопку (переход на страницу настройки или открытие модального окна)
- **Логика**: 
  - При клике на Switch вызывает onActiveChange
  - При клике на кнопку:
    - Если есть href - переходит на страницу настройки
    - Если allowsScopeChoice - открывает модальное окно выбора области
    - Иначе показывает сообщение о разработке
- **Особенности**: 
  - ИИ-интеграции (gemini, openai, cloud-ai, n8n) отображают специальную иконку LightningBoltIcon
  - Остальные интеграции отображают shortName или первые 2 символа названия
  - Цвет иконки зависит от активности (зеленый для активных, серый для неактивных)

### 3. IntegrationScopeModal
- **Расположение**: `@/app/company-settings/integrations/IntegrationScopeModal.tsx`
- **Назначение**: Модальное окно выбора области применения учетных данных интеграции
- **Пропсы**: 
  - `open`: флаг открытости модального окна (boolean)
  - `onOpenChange`: обработчик изменения открытости ((open: boolean) => void)
  - `integrationId`: ID интеграции (string)
  - `integrationName`: название интеграции (string)
  - `onSave`: обработчик сохранения области (опционально)
- **Функциональность**: 
  - Выбор режима хранения учетных данных: общий, у каждого свой, оба
  - Настройка API-ключей и токенов (через IntegrationSettingsForm)
  - Инструкции для пользователей при режиме "у каждого свой"
  - Отправка email с инструкциями (в разработке)
- **Связи**: 
  - IntegrationSettingsForm: форма настройки API-ключей
  - useToast: для отображения уведомлений
  - localStorage: сохранение выбранного режима и настроек
- **Подробная документация**: См. `IntegrationScopeModal.tsx` с детальными комментариями

## Интерфейсы и типы

### IntegrationId
```typescript
type IntegrationId =
  | 'google'      // Google (включение сервисов в профиле — календарь, Drive, таблицы, личные данные; у каждого пользователя своё)
  | 'telegram'    // Telegram (мессенджер)
  | 'hh'          // hh.ru и rabota.by (job-сайты)
  | 'huntflow'    // Huntflow (ATS система)
  | 'gemini'      // Google Gemini (ИИ)
  | 'openai'      // OpenAI (ИИ)
  | 'cloud-ai'    // Cloud AI (ИИ)
  | 'clickup'     // ClickUp (task tracker)
  | 'notion'      // Notion (task tracker)
  | 'n8n'         // n8n (автоматизация)
```

### IntegrationGroupId
```typescript
type IntegrationGroupId =
  | 'all'         // Все интеграции
  | 'ai'          // ИИ интеграции
  | 'auth'        // Авторизация
  | 'messengers'  // Мессенджеры
  | 'job-sites'   // Job-сайты
  | 'hrm-ats'     // HRM и ATS системы
  | 'task-trackers' // Task trackers
```

### Integration
```typescript
interface Integration {
  id: IntegrationId                    // Уникальный идентификатор
  name: string                         // Полное название
  shortName?: string                   // Короткое название
  active: boolean                      // Флаг активности (по умолчанию)
  href?: string                        // URL для перехода к настройкам
  group: IntegrationGroupId            // Группа интеграции
  allowsScopeChoice?: boolean          // Разрешает выбор области применения
}
```

## Состояние компонента

### State переменные:
- `scopeModalItem` (Integration | null): интеграция, для которой открыто модальное окно выбора области
  - Инициализация: null
  - Обновление: при открытии модального окна через onOpenScopeModal
  - Использование: передается в IntegrationScopeModal для отображения настроек

- `integrationActive` (Record<string, boolean>): объект с состоянием активности всех интеграций
  - Инициализация: пустой объект {}
  - Обновление: при монтировании загружается из localStorage, при переключении Switch обновляется
  - Структура: ключ - ID интеграции, значение - активна/неактивна
  - Использование: для отображения текущего состояния активности каждой интеграции

- `selectedGroup` (IntegrationGroupId): выбранная группа интеграций для фильтрации
  - Инициализация: 'all' (все интеграции)
  - Обновление: при клике на вкладку группы
  - Использование: для фильтрации отображаемых интеграций

## Функции и обработчики

### getIntegrationActive(id, fallback)
- **Назначение**: Получение состояния активности интеграции из localStorage
- **Параметры**: 
  - `id`: ID интеграции (string)
  - `fallback`: значение по умолчанию (boolean)
- **Возвращает**: boolean - состояние активности интеграции
- **Логика**: 
  - Проверяет наличие window (для SSR безопасности)
  - Читает значение из localStorage по ключу `integrationActive_${id}`
  - Парсит строковое значение в boolean
  - Если значение не найдено или некорректно - возвращает fallback
- **Используется для**: Восстановления состояния активности при загрузке страницы

### setIntegrationActive(id, value)
- **Назначение**: Сохранение состояния активности интеграции в localStorage
- **Параметры**: 
  - `id`: ID интеграции (string)
  - `value`: новое состояние активности (boolean)
- **Логика**: 
  - Проверяет наличие window (для SSR безопасности)
  - Сохраняет значение в localStorage по ключу `integrationActive_${id}`
  - Преобразует boolean в строку для хранения
- **Используется для**: Сохранения состояния активности при переключении Switch

### filterByGroup(items, group)
- **Назначение**: Фильтрация интеграций по группе
- **Параметры**: 
  - `items`: массив интеграций (Integration[])
  - `group`: ID группы для фильтрации (IntegrationGroupId)
- **Возвращает**: Integration[] - отфильтрованный массив интеграций
- **Логика**: 
  - Если group === 'all' - возвращает исходный массив
  - Иначе фильтрует интеграции, оставляя только те, у которых group совпадает с указанным
- **Используется для**: Отображения интеграций выбранной группы

### useEffect: загрузка состояния активности
- **Назначение**: Загрузка состояния активности интеграций из localStorage при монтировании
- **Зависимости**: [] (пустой массив - выполняется только при монтировании)
- **Логика**: 
  1. Создает объект next для хранения состояний
  2. Для каждой интеграции из INTEGRATIONS:
     - Загружает состояние из localStorage через getIntegrationActive
     - Использует значение по умолчанию из INTEGRATIONS, если в localStorage нет значения
  3. Устанавливает состояние integrationActive
- **Используется для**: Восстановления состояния активности всех интеграций при загрузке страницы

### handleActiveChange(id, value)
- **Назначение**: Обработчик изменения активности интеграции
- **Параметры**: 
  - `id`: ID интеграции (string)
  - `value`: новое состояние активности (boolean)
- **Логика**: 
  1. Сохраняет новое состояние в localStorage через setIntegrationActive
  2. Обновляет состояние компонента через setIntegrationActiveState
- **Используется для**: Сохранения состояния активности при переключении Switch

### handleAction (в IntegrationCard)
- **Назначение**: Обработчик клика на кнопку "Настроить"/"Подключить"
- **Логика**: 
  - Если у интеграции есть href - переходит на страницу настройки через router.push
  - Если интеграция позволяет выбрать область применения (allowsScopeChoice) - открывает модальное окно
  - Иначе показывает сообщение о разработке (TODO)
- **Используется для**: Определения действия при клике на кнопку интеграции

## Константы

### ACTIVE_STORAGE_KEY
- **Значение**: `'integrationActive'`
- **Назначение**: Ключ для хранения состояния активности интеграций в localStorage
- **Формат ключа**: `${ACTIVE_STORAGE_KEY}_${id}` (например, `integrationActive_google`)

### INTEGRATION_GROUPS
- **Тип**: `{ id: IntegrationGroupId; label: string }[]`
- **Назначение**: Массив групп интеграций для фильтрации
- **Используется для**: Отображения вкладок фильтрации в интерфейсе
- **Группы**:
  - `all`: Все
  - `ai`: ИИ
  - `auth`: Вход
  - `messengers`: Мессенджеры
  - `job-sites`: Job-сайты
  - `hrm-ats`: HRM&ATS
  - `task-trackers`: Task Tracker's

### INTEGRATIONS
- **Тип**: `Integration[]`
- **Назначение**: Массив всех доступных интеграций
- **Используется для**: Отображения списка интеграций в интерфейсе
- **Структура каждой интеграции**:
  - `id`: уникальный идентификатор (IntegrationId)
  - `name`: полное название интеграции
  - `shortName`: короткое название для компактного отображения
  - `active`: флаг активности (true - включена, false - выключена)
  - `group`: группа интеграции для фильтрации
  - `allowsScopeChoice`: разрешает ли выбор области применения учетных данных
  - `href`: опционально, URL для перехода к настройкам интеграции

## Доступные интеграции

  - Переключатель Switch для быстрого включения/выключения интеграции
  - Визуальная индикация активности (Badge и цвет иконки)

- **Выбор области применения**: 
  - Для интеграций с allowsScopeChoice можно выбрать режим хранения ключей
  - Режимы: общий (один набор ключей на всю компанию), у каждого свой, оба
  - Настройка API-ключей через IntegrationScopeModal

- **Адаптивная сетка**: 
  - На мобильных: 1 колонка
  - На планшетах: 2 колонки
  - На десктопе: 3 колонки

- **Иконки интеграций**: 
  - ИИ-интеграции отображают специальную иконку LightningBoltIcon
  - Остальные интеграции отображают shortName или первые 2 символа названия
  - Цвет иконки зависит от активности (зеленый для активных, серый для неактивных)

- **Сохранение состояния**: 
  - Состояние активности сохраняется в localStorage для восстановления при следующей загрузке
  - Ключ: `integrationActive_${id}`
  - Безопасно работает на сервере (SSR) - проверяет наличие window
