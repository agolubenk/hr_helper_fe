# Документация страницы: Telegram — Вход (telegram/page.tsx)

## Общее описание

Главная страница входа в Telegram. Предоставляет два способа подключения Telegram аккаунта к HR Helper: через сканирование QR-кода в приложении Telegram на телефоне или вход по номеру телефона. Страница является точкой входа для интеграции с Telegram и дальнейшей работы с чатами.

**Путь**: `/telegram`

## Интерфейсы и типы

### QR_PATTERN
- **Тип**: boolean[][]
- **Структура**: Двумерный массив 21×21, где true означает черную ячейку, false - белую
- **Назначение**: Предвычисленный паттерн QR-кода для отображения
- **Генерация**: Вычисляется один раз при загрузке модуля функцией getQrPattern()

## Функции и обработчики

### getQrPattern()
- **Назначение**: Генерация детерминированного паттерна QR-кода
- **Параметры**: Нет
- **Возвращает**: boolean[][] - двумерный массив 21×21
- **Логика**: 
  1. Создается сетка 21×21, заполненная false (белые ячейки)
  2. Добавляются углы 7×7 (finder pattern) в трех углах (true - черные ячейки)
     - Левый верхний угол [0, 0]
     - Правый верхний угол [0, n-7]
     - Левый нижний угол [n-7, 0]
  3. Вырезается внутренний квадрат в левом верхнем углу (2,2-4,4)
  4. Добавляется timing pattern по 6-й строке и столбцу (чередующиеся ячейки)
  5. Заполняются дополнительные ячейки по алгоритму:
     - Для области 8-20,8-20: если (i + j) % 3 === 0 и не timing pattern
     - Для области 0-5,8-20: если (i * 2 + j) % 5 === 0 и не timing pattern
     - Для области 8-20,0-5: если (i + j * 2) % 5 === 1 и не timing pattern
- **Используется для**: 
  - Генерации визуального паттерна, напоминающего QR-код
  - Отображения на странице входа
- **TODO**: Заменить на генерацию реального QR-кода с данными для Telegram через API

### handlePhoneInput(value)
- **Назначение**: Обработчик ввода номера телефона
- **Параметры**: 
  - `value`: string - введенное значение
- **Логика**: 
  1. Фильтрует введенное значение, оставляя только допустимые символы
  2. Разрешает только цифры (0-9), круглые скобки () и знак +
  3. Удаляет все остальные символы через replace(/[^0-9()+]/g, '')
  4. Обновляет состояние phoneNumber отфильтрованным значением
- **Используется для**: 
  - Валидации ввода номера телефона
  - Предотвращения ввода недопустимых символов

### handlePhoneLogin()
- **Назначение**: Обработчик входа по номеру телефона
- **Параметры**: Нет
- **Логика**: 
  1. Проверяет, что номер телефона не пустой (trim())
  2. Если пустой - выходит из функции
  3. TODO: Отправляет запрос на вход по номеру телефона через Telegram API
  4. В текущей реализации только логирует номер в консоль
- **Используется для**: 
  - Отправки запроса на вход по номеру телефона
  - Вызывается при клике на кнопку "Войти" или нажатии Enter

## Алгоритм генерации QR-кода

### Структура паттерна:
1. **Инициализация**: Создается сетка 21×21, заполненная false (белые ячейки)
2. **Finder Pattern (углы 7×7)**: 
   - Добавляются в трех углах: левый верхний [0,0], правый верхний [0,14], левый нижний [14,0]
   - Каждый угол - квадрат 7×7 черных ячеек
3. **Внутренний белый квадрат**: 
   - Вырезается в левом верхнем углу (2,2-4,4)
   - Создает характерный паттерн finder pattern
4. **Timing Pattern**: 
   - Линия по 6-й строке (m[6][k] = k % 2 === 0)
   - Линия по 6-му столбцу (m[k][6] = k % 2 === 0)
   - Чередующиеся черные и белые ячейки
5. **Дополнительное заполнение**: 
   - Область 8-20,8-20: (i + j) % 3 === 0 (исключая timing pattern)
   - Область 0-5,8-20: (i * 2 + j) % 5 === 0 (исключая timing pattern)
   - Область 8-20,0-5: (i + j * 2) % 5 === 1 (исключая timing pattern)

### Визуальное представление:
- Черные ячейки: background #000, класс qrCell
- Белые ячейки: background #fff, класс qrCellEmpty
- Размер ячейки: 8px × 8px
- Общий размер QR-кода: 168px × 168px (21 × 8px)
