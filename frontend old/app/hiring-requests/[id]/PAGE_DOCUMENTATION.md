# Документация страницы: Детальный просмотр заявки (hiring-requests/[id]/page.tsx)

## Общее описание

Страница детального просмотра заявки на подбор. Отображает полную информацию о заявке, позволяет редактировать и управлять заявкой. Страница является детальным представлением заявки из списка на странице `/hiring-requests`.

**Путь**: `/hiring-requests/[id]`

**Примечание**: Файл `page.tsx` не существует, страница существует только в виде документации. Вместо этого используется модальное окно или страница редактирования.

## Компоненты страницы

### 1. Заголовок страницы
- **Компонент**: `Flex` с кнопкой "Назад"
- **Назначение**: Навигация и заголовок страницы
- **Логика**: 
  - Кнопка "Назад" возвращает на страницу списка заявок (`/hiring-requests`)
  - Отображает название заявки (из данных заявки)
  - Может содержать breadcrumbs для навигации
- **Связи**: Навигация на `/hiring-requests` через useRouter

- **Секции**:
  - **Основная информация**: 
    - Название вакансии
    - Грейд (Junior+, Middle, Middle+, Senior)

  - **Рекрутер и сроки**: 
    - Рекрутер, ведущий заявку
    - Количество дней работы рекрутера

  - **Статус и приоритет**:

    - Дата изменения статуса (для закрытых/отмененных)
  - **Технологии**:

## Интерфейсы и типы

### HiringRequest (интерфейс заявки)
```typescript
interface HiringRequest {
  id: number                                    // Уникальный идентификатор заявки
  title: string                                 // Название вакансии
  grade: string                                 // Грейд (Junior+, Middle, Middle+, Senior)
  project: string | null                         // Проект (может быть null)
  recruiter: string                             // Рекрутер, ведущий заявку
  recruiterDays: number                         // Количество дней работы рекрутера
  status: 'planned' | 'in_process' | 'closed' | 'cancelled'  // Статус заявки
  statusDate?: string                           // Дата изменения статуса (опционально)
  startDate: string                             // Дата начала подбора (DD.MM.YYYY)
  endDate: string                               // Дата окончания подбора (DD.MM.YYYY)
  deadline?: string                             // Дедлайн (опционально)
  isOverdue: boolean                            // Флаг просрочки заявки
  factDays: number                              // Фактическое количество дней подбора
  slaDays: number                               // SLA по количеству дней
  slaStatus: 'normal' | 'risk' | 'overdue' | 'on_time'  // Статус SLA
  t2hDays?: number                              // Время до найма в днях (опционально)
  t2hSlaDays: number                            // SLA по времени до найма
  candidate?: {                                  // Информация о найденном кандидате (опционально)
    name: string
    id: string
  }
  department: string                            // Отдел/департамент
  priority: 'high' | 'medium' | 'low'           // Приоритет заявки
  technologies: string[]                        // Массив технологий
  candidates: number                             // Количество кандидатов
  date: string                                  // Дата создания заявки
  hasWarning: boolean                            // Флаг наличия предупреждения
}
```

## Состояние компонента

### State переменные:
- `request` (HiringRequest | null): данные заявки для отображения
  - Инициализация: null
  - Обновление: при загрузке данных через loadRequest
  - Использование: для отображения информации о заявке

- `loading` (boolean): флаг загрузки данных
  - Инициализация: false
  - Обновление: устанавливается в true при начале загрузки, false при завершении
  - Использование: для отображения индикатора загрузки

## Функции и обработчики

### loadRequest()
- **Назначение**: Загрузка данных заявки по ID из URL параметра
- **Логика**: 
  1. Читает ID заявки из URL через useParams()
  2. Устанавливает loading в true
  3. Вызывает API для получения данных заявки
  4. Обновляет состояние request с полученными данными
  5. Устанавливает loading в false
  6. Обрабатывает ошибки загрузки
- **Используется для**: Загрузки данных заявки при монтировании компонента

### handleEdit()
- **Назначение**: Обработчик перехода к редактированию заявки
- **Логика**: 
  - Переходит на страницу редактирования через router.push(`/hiring-requests/${id}/edit`)
- **Используется для**: Перехода к редактированию заявки

### handleStatusChange(newStatus)
- **Назначение**: Обработчик изменения статуса заявки
- **Параметры**: 
  - `newStatus`: новый статус заявки ('planned' | 'in_process' | 'closed' | 'cancelled')
- **Логика**: 
  1. Показывает подтверждение изменения статуса
  2. Вызывает API для обновления статуса
  3. Обновляет состояние request
  4. Показывает уведомление об успехе

## URL параметры

### [id]
- **Тип**: Динамический параметр маршрута
- **Назначение**: ID заявки для просмотра
- **Пример**: `/hiring-requests/123` отображает заявку с ID 123
- **Обработка**: 
  - Читается через `useParams()` из Next.js
  - Используется для загрузки данных заявки через loadRequest()
  - Используется для навигации к редактированию
