# Документация страницы: Главная (page.tsx)

## Общее описание

Главная страница приложения HR Helper. Точка входа после авторизации, навигационный хаб с карточками быстрого доступа к разделам и приветственным туром.

## Компоненты страницы

### 1. Приветственное сообщение
- **Компонент**: `Text` с заголовком
- **Назначение**: Приветствие пользователя
- **Логика**: Отображает заголовок "Добро пожаловать в HR Helper"
- **Связи**: Используется в приветственном туре

### 2. Кнопка приветственного тура
- **Компонент**: `Button` с иконкой
- **Назначение**: Запуск интерактивного тура по приложению
- **Логика**: 
  - При клике запускает тур через библиотеку driver.js
  - Тур проходит по основным разделам приложения
  - Сохраняет прогресс в localStorage
- **Связи**: 
  - driver.js: библиотека для создания тура
  - Использует data-tour атрибуты на других страницах

### 3. Карточки навигации
- **Компонент**: `Card` с ссылками
- **Назначение**: Быстрый доступ к разделам приложения
- **Структура**: 
  - Иконка раздела
  - Название раздела
  - Ссылка на страницу раздела
- **Логика**: 
  - При клике происходит переход на соответствующую страницу
  - Используется в приветственном туре
- **Разделы**: 
  - Чат, Рекрутинг, Вакансии, Заявки на подбор
  - ЗП вилки, Бенчмарки, Интервьюеры
  - ИИ чат, Вики, Отчетность, Настройки

## Состояние компонента

### State переменные:
- Нет (все данные статические)

## Функции и обработчики

### startTour()
- **Назначение**: Запуск приветственного тура
- **Логика**: 
  1. Создает экземпляр driver.js
  2. Определяет шаги тура на основе BLOCKS
  3. Настраивает сохранение прогресса в localStorage
  4. Запускает тур
  5. При переходе между страницами использует задержку для плавности
- **Связи**: 
  - driver.js: библиотека для создания тура
  - localStorage: сохранение прогресса
  - useRouter: навигация между страницами во время тура

### Восстановление тура
- **Логика**: 
  - При загрузке проверяет localStorage на наличие сохраненного прогресса
  - Если есть - предлагает продолжить тур

## Константы

### BLOCKS
Массив разделов приложения:
- `id`: уникальный идентификатор (используется для data-tour)
- `label`: название раздела
- `href`: URL для перехода
- `icon`: иконка из Radix UI

**Используется для**: 
- Рендеринга карточек навигации
- Создания шагов в приветственном туре

## Приветственный тур

### Шаги тура:
1. Приветствие на главной странице
2. Переход по разделам через карточки
3. Объяснение функционала каждого раздела
4. Сохранение прогресса в localStorage

### Сохранение прогресса:
- **Ключ**: `'tour-progress'`
- **Значение**: текущий шаг тура
- **Логика**: 
  - Сохраняется при каждом шаге
  - Восстанавливается при следующем запуске
