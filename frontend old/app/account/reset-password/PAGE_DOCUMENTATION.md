# Документация страницы: Сброс пароля (account/reset-password/page.tsx)

## Общее описание

Страница сброса пароля. Позволяет установить новый пароль после перехода по ссылке из email. Требует токен из URL параметра `?token=...` для работы.

**Путь**: `/account/reset-password?token=TOKEN`

## Компоненты страницы

### 1. Форма сброса пароля
- **Компонент**: Форма с двумя полями пароля
- **Назначение**: Установка нового пароля
- **Поля**:
  - Новый пароль (обязательное, type="password", минимум 8 символов)
  - Подтверждение пароля (обязательное, type="password", должно совпадать с новым паролем)
- **Логика**: 
  - Валидация пароля (минимальная длина, совпадение полей)
  - Отправка нового пароля и токена на сервер
  - Показ сообщения об успехе после сброса

## Состояние компонента

### State переменные:
- `password` (string): значение поля нового пароля
  - Инициализация: `useState('')`
  - Обновление: через `onChange` обработчик поля ввода
  - Использование: отправка в API для сброса пароля
- `confirmPassword` (string): значение поля подтверждения пароля
  - Инициализация: `useState('')`
  - Обновление: через `onChange` обработчик поля ввода
  - Использование: проверка совпадения с новым паролем
- `isLoading` (boolean): флаг загрузки
  - Инициализация: `useState(false)`
  - Обновление: `true` при отправке запроса, `false` после завершения
  - Использование: блокировка кнопки и изменение текста на "Сохранение..."
- `isSuccess` (boolean): флаг успешного сброса
  - Инициализация: `useState(false)`
  - Обновление: `true` после успешного сброса пароля
  - Использование: переключение между формой и сообщением об успехе
- `error` (string): текст ошибки валидации или запроса
  - Инициализация: `useState('')`
  - Обновление: при ошибках валидации или запроса
  - Использование: отображение сообщения об ошибке
- `token` (string | null): токен сброса пароля из URL
  - Инициализация: `useState<string | null>(null)`
  - Обновление: из URL параметра `?token=...` через `useSearchParams`
  - Использование: отправка в API для сброса пароля

### Hooks:
- `useSearchParams()`: получение параметров из URL
  - Используется для получения токена: `searchParams.get('token')`
- `useTheme()`: получение темы и функции переключения
- `useRouter()`: программная навигация
  - Используется для перехода на `/account/login`

## Функции и обработчики

### useEffect - получение токена из URL
- **Назначение**: Получение токена сброса пароля из URL параметров
- **Зависимости**: `[searchParams]`
- **Логика**: 
  1. Читает параметр 'token' из URL через `searchParams.get('token')`
  2. Сохраняет токен в состоянии: `setToken(tokenParam)`
  3. Если токена нет - устанавливает ошибку: `setError('Недействительная ссылка...')`
  4. Блокирует форму при отсутствии токена
- **Поведение**: 
  - Выполняется при монтировании компонента и при изменении searchParams
  - Позволяет открывать страницу с токеном через URL: `/account/reset-password?token=...`
  - Без токена форма недоступна для использования
- **Связи**: 
  - Используется для глубоких ссылок из email с токеном сброса пароля

### handleSubmit()
- **Назначение**: Обработчик отправки формы сброса пароля
- **Параметры**: Нет
- **Логика**: 
  1. `setError('')` - очищает предыдущие ошибки

  3. Валидация минимальной длины: проверяет `password.length < 8`
  4. Валидация совпадения: проверяет `password !== confirmPassword`
  5. Валидация токена: проверяет наличие `token`

## URL параметры

### ?token=TOKEN
- **Тип**: Токен сброса пароля (string)
- **Назначение**: Токен для сброса пароля из email
- **Пример**: `/account/reset-password?token=abc123def456` использует токен abc123def456
- **Обработка**: 
  - Читается через `useSearchParams().get('token')`
  - Сохраняется в состоянии `token`
  - Используется для отправки запроса на сброс пароля
  - Без токена форма недоступна
- **Безопасность**: 
  - Токен должен быть одноразовым
  - Токен должен иметь срок действия (например, 1 час)
  - Токен должен быть криптографически стойким
