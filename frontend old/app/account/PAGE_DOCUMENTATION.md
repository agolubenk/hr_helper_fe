# Документация модуля: Аккаунт и аутентификация (account/)

## Общее описание

Модуль `account/` содержит все страницы, связанные с аутентификацией пользователя и управлением аккаунтом. Все страницы объединены в единую структуру для удобства управления и поддержки.

## Структура модуля

```
account/
├── login/              # Страница входа в систему
├── forgot-password/    # Страница восстановления пароля
├── reset-password/     # Страница сброса пароля
├── profile/            # Страница профиля пользователя
└── PAGE_DOCUMENTATION.md  # Этот файл
```

## Общие характеристики

### Единый дизайн
Все страницы аутентификации (login, forgot-password, reset-password) используют единый визуальный стиль:
- Центрированная форма с полупрозрачным фоном
- Плавающие метки полей ввода (FloatingLabelInput)
- Кнопка переключения темы (светлая/темная)
- Адаптивный дизайн для мобильных устройств
- Поддержка темной темы

### Общие компоненты

#### FloatingLabelInput
Кастомный компонент поля ввода с плавающей меткой, используемый на всех страницах аутентификации:
- Метка перемещается вверх при фокусе или наличии значения
- Валидация и отображение ошибок
- Поддержка различных типов полей (email, password, text)

#### Theme Toggle
Кнопка переключения темы доступна на всех страницах:
- Для десктопа: справа от формы
- Для мобильных: в углу формы
- Использует `useTheme` из ThemeProvider

**Особенности**:

- Кнопка входа через Google OAuth
- Кнопка переключения темы
- Ссылка на восстановление пароля (`/account/forgot-password`)
- Индикатор загрузки при отправке формы

**Использование**:

### Reset Password - Сброс пароля (`account/reset-password/`)

**Назначение**: Сброс пароля пользователя по токену из email.

**Особенности**:

- Форма с двумя полями: новый пароль и подтверждение пароля
- Валидация пароля (минимум 8 символов, совпадение полей)
- Индикатор загрузки при отправке запроса
- Сообщение об успешном изменении пароля
- Отображение ошибок валидации
- Кнопка возврата на страницу входа (`/account/login`)
- Кнопка переключения темы

**Использование**:

### Profile - Профиль пользователя (`account/profile/`)

**Назначение**: Управление профилем пользователя и настройками аккаунта.

**Особенности**:
- **UserCard**: карточка пользователя с основной информацией
- **ProfileNavigation**: навигация между вкладками профиля
- **ProfileInfo**: просмотр информации о пользователе
- **ProfileEditForm**: редактирование данных пользователя
- **IntegrationsPage**: настройка интеграций и API
- **QuickButtonsPage**: управление быстрыми кнопками
- **AccentColorSettings**: настройка акцентного цвета темы
- Синхронизация активной вкладки через localStorage и URL параметры
- Синхронизация между вкладками браузера через события

**Вкладки профиля**:
1. **Профиль** (`?tab=profile`) - просмотр и редактирование информации
2. **Интеграции и API** (`?tab=integrations`) - настройка интеграций
3. **Быстрые кнопки** (`?tab=quick-buttons`) - управление быстрыми кнопками
4. **Цвет темы** (`?tab=accent-color`) - настройка акцентного цвета

**Использование**:
- При клике на имя пользователя в Header
- При переходе из Sidebar
- При переходе из других компонентов (например, FloatingActions)

## Технические детали

### CSS Модули
Каждая страница использует свой CSS модуль:
- `login.module.css` - стили для страницы входа
- `forgot-password.module.css` - стили для страницы восстановления пароля
- `reset-password.module.css` - стили для страницы сброса пароля
- `profile.module.css` - стили для страницы профиля

### Навигация
Все страницы используют `useRouter` из Next.js для программной навигации:
- `router.push('/account/login')` - переход на страницу входа
- `router.push('/account/forgot-password')` - переход на восстановление пароля
- `router.push('/account/reset-password')` - переход на сброс пароля
- `router.push('/account/profile')` - переход на профиль
- `router.push('/account/profile?tab=integrations')` - переход на профиль с вкладкой интеграций

### Тема
Все страницы поддерживают переключение темы через `useTheme`:
- Получение текущей темы
- Функция переключения темы
- Сохранение выбранной темы в localStorage

### Валидация
Страницы аутентификации используют клиентскую валидацию:
- Email: проверка формата
- Пароль: минимум 8 символов
- Подтверждение пароля: совпадение с новым паролем

## Связи с другими модулями

### AppLayout
Страница профиля обернута в `AppLayout` для единообразного отображения:
- Заголовок страницы в браузере
- Общий layout приложения
- Навигация и меню

### Header
При клике на имя пользователя в Header происходит переход на `/account/profile`.

### Sidebar
Sidebar содержит ссылки на профиль:
- "Профиль" - переход на `/account/profile`
- "Интеграции и API" - переход на `/account/profile?tab=integrations`

### FloatingActions
Компонент быстрых действий может открывать страницу профиля.

### Errors
Страница ошибки 401 содержит кнопку перехода на `/account/login`.

## TODO: Интеграция с API

Все страницы модуля требуют интеграции с API:

1. **Login**:
   - POST `/api/auth/login` - авторизация пользователя
   - POST `/api/auth/google` - авторизация через Google
   - Сохранение токена доступа в localStorage/cookies
   - Редирект на главную страницу после успешной авторизации

2. **Forgot Password**:
   - POST `/api/auth/forgot-password` - отправка email с инструкциями
   - Валидация email на сервере
   - Обработка ошибок

3. **Reset Password**:
   - POST `/api/auth/reset-password` - сброс пароля по токену
   - Валидация токена на сервере
   - Обработка ошибок (невалидный или истекший токен)

4. **Profile**:
   - GET `/api/user/profile` - получение данных пользователя
   - PUT `/api/user/profile` - обновление данных пользователя
   - GET `/api/user/integrations` - получение списка интеграций
   - POST `/api/user/integrations` - добавление интеграции
   - DELETE `/api/user/integrations/:id` - удаление интеграции
   - GET `/api/user/quick-buttons` - получение быстрых кнопок
   - PUT `/api/user/quick-buttons` - обновление быстрых кнопок
   - PUT `/api/user/theme` - обновление акцентного цвета темы

## Рекомендации по использованию

1. **Безопасность**: Все запросы к API должны использовать HTTPS
2. **Валидация**: Валидация должна выполняться как на клиенте, так и на сервере
3. **Обработка ошибок**: Все ошибки должны обрабатываться и отображаться пользователю
4. **UX**: Индикаторы загрузки должны отображаться при всех асинхронных операциях
5. **Доступность**: Все формы должны быть доступны для клавиатурной навигации
