## 📖 Обзор модуля

### Назначение

Модуль `account/` содержит все страницы, связанные с:
- ✅ Аутентификацией пользователей
- ✅ Управлением аккаунтом
- ✅ Восстановлением доступа
- ✅ Настройками профиля

### Структура

```
account/
├── login/                    # Вход в систему
│   ├── page.tsx             # Компонент страницы входа
```

## 🏗️ Архитектура

### Принципы проектирования

1. **Единый дизайн**
   - Все страницы аутентификации используют единый визуальный стиль
   - Центрированные формы с полупрозрачным фоном
   - Плавающие метки полей ввода
   - Адаптивный дизайн

2. **Переиспользование компонентов**
   - `FloatingLabelInput` - используется на всех страницах аутентификации
   - `ThemeProvider` - единое управление темой
   - Общие стили и паттерны

3. **Безопасность**
   - Валидация на клиенте и сервере
   - Защита от CSRF
   - Secure storage для токенов
   - Rate limiting

### Технологии

- **Next.js 14** - App Router, Server Components
- **React 18** - хуки, состояние
- **Radix UI** - компоненты интерфейса
- **TypeScript** - типизация
- **CSS Modules** - изолированные стили

## 🔐 Страница входа

**Путь:** `/account/login`  
**Файл:** `account/login/page.tsx`  
**Документация:** `account/login/PAGE_DOCUMENTATION.md`

### Функциональность

#### Методы авторизации

1. **Email/Пароль**
   - Форма с полями email и password
   - HTML5 валидация
   - Отправка на `/api/auth/login`
   - Сохранение токена

2. **Google OAuth**
   - Кнопка "Войти через Google"
   - Редирект на Google OAuth
   - Callback обработка
   - Автоматический вход

#### Состояние

```typescript
{
  email: string           // Email пользователя
  password: string        // Пароль пользователя
  isLoading: boolean      // Флаг загрузки при отправке формы
}
```

#### Обработчики

- `handleEmailLogin(e: React.FormEvent)` - вход через email/пароль
- `handleGoogleLogin()` - вход через Google OAuth

#### Валидация

- **Email**: формат email (HTML5 `type="email"`)
- **Password**: обязательное поле (`required`)

## 📧 Страница восстановления пароля

**Путь:** `/account/forgot-password`  
**Файл:** `account/forgot-password/page.tsx`  
**Документация:** `account/forgot-password/PAGE_DOCUMENTATION.md`

### Функциональность

#### Процесс восстановления

1. **Запрос восстановления**
   - Пользователь вводит email
   - Отправка запроса на сервер
   - Генерация токена сброса пароля
   - Отправка email с инструкциями

2. **Email с инструкциями**
   - Ссылка: `/account/reset-password?token=TOKEN`
   - Срок действия токена (обычно 1 час)
   - Инструкции по сбросу пароля

#### Состояние

```typescript
{
  email: string           // Email для восстановления
  isLoading: boolean      // Флаг загрузки при отправке

}
```

#### Обработчики

- `handleSubmit()` - отправка запроса на восстановление
- `handleBackToLogin()` - возврат на страницу входа

#### Валидация

- **Email**: формат email (HTML5 `type="email"`)

## 🔑 Страница сброса пароля

**Путь:** `/account/reset-password?token=TOKEN`  
**Файл:** `account/reset-password/page.tsx`  
**Документация:** `account/reset-password/PAGE_DOCUMENTATION.md`

### Функциональность

#### Процесс сброса

1. **Получение токена**
   - Чтение токена из URL параметра `?token=...`
   - Валидация наличия токена
   - Блокировка формы при отсутствии токена

2. **Установка нового пароля**
   - Ввод нового пароля
   - Подтверждение пароля
   - Валидация совпадения
   - Отправка на сервер с токеном

#### Состояние

```typescript
{
  password: string         // Новый пароль
  confirmPassword: string  // Подтверждение пароля
  isLoading: boolean       // Флаг загрузки
  isSuccess: boolean       // Флаг успешного сброса
  error: string            // Текст ошибки
  token: string | null     // Токен из URL
}
```

#### Обработчики

- `useEffect` - получение токена из URL
- `handleSubmit()` - отправка нового пароля
- `handleBackToLogin()` - возврат на страницу входа

#### Валидация

1. **Заполненность полей**
2. **Минимальная длина**: 8 символов
3. **Совпадение паролей**: `password === confirmPassword`
4. **Наличие токена**: токен должен быть в URL

#### Требования к паролю

- Минимум 8 символов
- Рекомендуется: буквы, цифры, символы

## 👤 Страница профиля

**Путь:** `/account/profile` или `/account/profile?tab=integrations`  
**Файл:** `account/profile/page.tsx`  
**Документация:** `account/profile/PAGE_DOCUMENTATION.md`

### Функциональность

#### Вкладки профиля

1. **Профиль** (`activeTab === 'profile'`)
   - Просмотр информации о пользователе
   - Режим только чтения
   - Компонент: `ProfileInfo`

2. **Редактирование** (`activeTab === 'edit'`)
   - Редактирование личных данных
   - Настройка акцентного цвета темы
   - Компоненты: `ProfileEditForm`, `AccentColorSettings`

3. **Интеграции и API** (`activeTab === 'integrations'`)
   - Настройка интеграций (Huntflow, Google Calendar)
   - Управление API ключами
   - Компонент: `IntegrationsPage`

4. **Быстрые кнопки** (`activeTab === 'quick-buttons'`)
   - Добавление/удаление быстрых кнопок
   - Настройка ссылок и иконок
   - Компонент: `QuickButtonsPage`

#### Состояние

```typescript
{
  activeTab: TabType        // Текущая активная вкладка
  userData: UserData        // Данные пользователя
}
```

#### Синхронизация вкладок

- **localStorage**: сохранение активной вкладки
- **URL параметры**: `?tab=integrations` для глубоких ссылок
- **Storage events**: синхронизация между вкладками браузера
- **Custom events**: синхронизация в той же вкладке

#### Обработчики

- `getInitialTab()` - определение начальной вкладки
- `handleSave(data)` - сохранение данных пользователя
- `handleCancel()` - отмена редактирования
- `renderContent()` - рендеринг содержимого вкладки

## 🧩 Общие компоненты

### FloatingLabelInput

**Расположение:** `@/components/FloatingLabelInput`

**Назначение:** Поле ввода с плавающей меткой

**Использование:**
- Страница входа (email, password)
- Страница восстановления пароля (email)
- Страница сброса пароля (password, confirmPassword)

**Особенности:**
- Метка перемещается вверх при фокусе или заполнении
- Анимация перемещения метки
- Поддержка иконок слева от поля
- HTML5 валидация

### ThemeProvider

**Расположение:** `@/components/ThemeProvider`

**Назначение:** Управление темой приложения

**Использование:**
- Все страницы аккаунта
- Переключение светлой/темной темы
- Сохранение выбора в localStorage

**API:**
```typescript
const { theme, toggleTheme } = useTheme()
```

**Расположение:** `@/components/AppLayout`

**Назначение:** Общий layout приложения

**Использование:**

## 🔌 API интеграция

### Endpoints

#### Авторизация

```typescript
// POST /api/auth/login
{
  email: string
  password: string
}
// Response: { token: string, user: User }
```

#### Восстановление пароля

```typescript
// POST /api/auth/forgot-password
{
  email: string
}
// Response: { success: boolean, message: string }
```

#### Сброс пароля

```typescript
// POST /api/auth/reset-password
{
  token: string
  password: string
}
// Response: { success: boolean, message: string }
```

#### Профиль пользователя

```typescript
// GET /api/user/profile
// Response: { user: User }

// PUT /api/user/profile
{
  firstName: string
  lastName: string
  email: string
  // ... другие поля
}
// Response: { user: User }
```

### Обработка ошибок

#### Типы ошибок

1. **401 Unauthorized**
   - Неверный email/пароль
   - Истекшая сессия

2. **403 Forbidden**
   - Аккаунт заблокирован
   - Недостаточно прав

3. **404 Not Found**
   - Email не найден
   - Токен не найден

4. **422 Validation Error**
   - Ошибки валидации полей
   - Неверный формат данных

5. **429 Too Many Requests**
   - Превышен лимит запросов
   - Rate limiting

### Безопасность

1. **Токены**
   - Хранение в httpOnly cookies
   - Или localStorage с защитой от XSS
   - Обновление заголовков запросов

2. **Пароли**
   - Передача только через HTTPS
   - Хеширование на сервере
   - Проверка сложности

3. **Rate Limiting**
   - Ограничение попыток входа
   - Ограничение запросов восстановления пароля
   - Защита от brute force

## 🔒 Безопасность

### Защита от атак

1. **CSRF**
   - CSRF токены
   - SameSite cookies

2. **XSS**
   - Санитизация данных
   - Content Security Policy

3. **Brute Force**
   - Rate limiting
   - Блокировка после N попыток

4. **SQL Injection**
   - Параметризованные запросы
   - ORM защита

### Рекомендации

- ✅ Использовать HTTPS в продакшене
- ✅ Валидировать данные на сервере
- ✅ Логировать подозрительную активность
- ✅ Регулярно обновлять зависимости
- ✅ Использовать secure storage для токенов
