# Документация страницы: Интервьюеры (interviewers/page.tsx)

## Общее описание

Страница управления интервьюерами компании. Позволяет управлять базой интервьюеров: добавлять, редактировать, удалять интервьюеров, управлять их активностью, просматривать календари и статистику.

**Путь**: `/interviewers`

## Компоненты страницы

### 1. InterviewersPage (основной компонент)
- **Расположение**: `@/app/interviewers/page.tsx`
- **Назначение**: Главный компонент страницы управления интервьюерами
- **Функциональность**: 
  - Отображение списка интервьюеров в таблице
  - Поиск и фильтрация интервьюеров
  - Пагинация списка
  - Управление модальными окнами (просмотр, редактирование)
  - Удаление интервьюеров с подтверждением
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - useToast: для отображения уведомлений
  - Sidebar: содержит ссылку на эту страницу в разделе "Рекрутинг"
  - Google Calendar: ссылки на календари интервьюеров

### 2. Поиск и фильтры
- **Компонент**: Card с TextField и кнопками фильтров
- **Назначение**: Поиск и фильтрация интервьюеров по статусу
- **Функциональность**: 
  - Поиск по имени, фамилии, отчеству, email
  - Фильтрация по статусу: "Всего", "Активных", "Неактивных"
  - Кнопка поиска (TODO: реализовать функциональность)
  - Отображение статистики (общее количество, активные, неактивные)
- **Поведение**: 
  - При вводе в поле поиска фильтрует список интервьюеров
  - При изменении фильтра статуса обновляет список
  - При изменении поиска или фильтра сбрасывает текущую страницу на первую

### 3. Кнопка добавления интервьюера
- **Компонент**: Button с иконкой PlusIcon
- **Назначение**: Открытие модального окна для создания нового интервьюера
- **Функциональность**: 
  - При клике открывает модальное окно редактирования в режиме создания
  - Сбрасывает editingInterviewer в null

### 4. Таблица интервьюеров
- **Компонент**: Table.Root с колонками
- **Назначение**: Отображение списка интервьюеров с пагинацией
- **Колонки**:
  - **ФИО**: 
    - Аватар с инициалами (круглый, цветной фон)
    - Полное имя (Фамилия Имя)
    - Короткое имя (только Имя)
    - Кликабельно: открывает модальное окно просмотра
  - **Email**: 
    - Email адрес интервьюера
  - **Календарь**: 
    - Иконка CalendarIcon
    - Текст "Календарь" (кликабельно)
    - При клике открывает календарь интервьюера в новой вкладке
  - **Статус**: 
    - Для активных: иконка CheckIcon (зеленая) + бейдж "Активен" (зеленый)
    - Для неактивных: бейдж "Неактивен" (серый)
  - **Действия**: 
    - Кнопка редактирования (Pencil1Icon)
    - Кнопка удаления (TrashIcon)
- **Поведение**: 
  - Отображает только интервьюеров текущей страницы (пагинация)
  - При клике на строку открывает модальное окно просмотра
  - При клике на действия останавливает всплытие события (stopPropagation)

### 5. Пагинация
- **Компонент**: Flex с кнопками навигации
- **Назначение**: Навигация по страницам списка интервьюеров
- **Элементы**: 
  - Кнопка "Первая страница" (DoubleArrowLeftIcon)
  - Кнопка "Предыдущая страница" (ChevronLeftIcon)
  - Информация о странице: "Страница X из Y" (в центре, на цветном фоне)
  - Кнопка "Следующая страница" (ChevronRightIcon)
  - Кнопка "Последняя страница" (DoubleArrowRightIcon)
- **Поведение**: 
  - Отображается только если totalPages > 1
  - Кнопки "Первая" и "Предыдущая" отключены на первой странице
  - Кнопки "Следующая" и "Последняя" отключены на последней странице
  - При клике обновляет currentPage

### 6. Модальное окно просмотра интервьюера
- **Компонент**: Dialog.Root с содержимым
- **Назначение**: Отображение подробной информации об интервьюере
- **Структура**: 
  - **Заголовок**: 
    - Иконка PersonIcon и текст "Информация об интервьюере"
    - Кнопка "Редактировать" (открывает модальное окно редактирования)
    - Кнопка "Активен" (если интервьюер активен)
  - **Основная информация** (левая часть): 
    - Аватар с инициалами (большой, 80x80px)
    - Полное имя и короткое имя
    - Детали:
      - Имя
      - Email (кликабельно, открывает mailto)
      - Дата создания
      - Фамилия
      - Статус (с иконкой и бейджем)
      - Последнее обновление
    - Ссылка на календарь:
      - TextField (readonly) с ссылкой
      - Кнопка копирования (CopyIcon)
  - **Боковая панель** (правая часть, 250px): 
    - **Карточка "Действия"**: 
      - Кнопка "Написать email" (открывает mailto)
      - Кнопка "Открыть календарь" (открывает календарь в новой вкладке)
      - Кнопка "Записаться на интервью" (открывает календарь)
      - Кнопка "Удалить" (красная, с подтверждением)
    - **Карточка "Статистика"**: 
      - Количество интервью (TODO: загружать из API)
      - Количество завершенных интервью (TODO: загружать из API)
- **Поведение**: 
  - Открывается при клике на строку таблицы или на имя интервьюера
  - При закрытии сбрасывает selectedInterviewer
  - При клике на "Редактировать" закрывает это окно и открывает модальное окно редактирования

### 7. InterviewerEditModal (модальное окно редактирования/создания)
- **Компонент**: Dialog.Root с формой
- **Назначение**: Создание нового или редактирование существующего интервьюера
- **Режимы**: 
  - **Создание**: когда interviewer === null
  - **Редактирование**: когда interviewer !== null
- **Структура**: 
  - **Заголовок**: 
    - Градиентный фон (accent-9 → accent-10)
    - Иконка Pencil1Icon в круглом фоне
    - Заголовок: "Создание интервьюера" или "Редактирование интервьюера [ФИО]"
  - **Поля формы**: 
    - **Имя** (обязательное): TextField для имени
    - **Фамилия** (обязательное): TextField для фамилии
    - **Отчество** (опциональное): TextField для отчества
    - **Email** (обязательное): TextField типа email
    - **Активен**: Switch для переключения активности
    - **Ссылка на календарь**: 
      - TextField для ссылки
      - Кнопка "Автозаполнить" (генерирует ссылку из email)
  - **Футер**: 
    - Кнопка "Отмена" / "Назад к интервьюеру" (зависит от returnToView)
    - Кнопка "Создать интервьюера" / "Сохранить изменения" (градиентный фон)
- **Валидация**: 
  - Проверяет обязательные поля: firstName, lastName, email
  - Показывает alert при отсутствии обязательных полей
- **Поведение**: 
  - При открытии заполняет форму данными интервьюера (если редактирование) или сбрасывает (если создание)
  - При сохранении вызывает onSave с данными формы
  - При закрытии может вернуться к модальному окну просмотра (если returnToView === true)

## Интерфейсы и типы

### Interviewer
```typescript
interface Interviewer {
  id: number                    // Уникальный идентификатор интервьюера
  firstName: string             // Имя интервьюера
  lastName: string              // Фамилия интервьюера
  middleName?: string           // Отчество интервьюера (опционально)
  email: string                 // Email интервьюера
  isActive: boolean             // Флаг активности интервьюера
  createdAt: string             // Дата добавления интервьюера (строка)
  calendarLink?: string          // Ссылка на календарь интервьюера в Google Calendar (опционально)
}
```

### InterviewerEditModalProps
```typescript
interface InterviewerEditModalProps {
  isOpen: boolean               // Флаг открытия модального окна
  onClose: () => void           // Функция закрытия модального окна
  interviewer: Interviewer | null // Интервьюер для редактирования (null для создания)
  returnToView?: boolean        // Флаг возврата к модальному окну просмотра после закрытия
  onSave: (data: any) => void   // Функция сохранения данных интервьюера
}
```

## Состояние компонента

### State переменные:
- `searchQuery` (string): поисковый запрос
  - Инициализация: пустая строка
  - Обновление: при вводе в поле поиска
  - Использование: фильтрация интервьюеров по имени, фамилии, отчеству, email
- `statusFilter` ('all' | 'active' | 'inactive'): фильтр по статусу
  - Инициализация: 'active'
  - Обновление: при клике на кнопки фильтров
  - Использование: фильтрация интервьюеров по активности
- `currentPage` (number): текущая страница пагинации
  - Инициализация: 1
  - Обновление: при клике на кнопки пагинации
  - Использование: вычисление startIndex и endIndex для пагинации
- `selectedInterviewer` (Interviewer | null): выбранный интервьюер для просмотра
  - Инициализация: null
  - Обновление: при клике на строку таблицы или имя интервьюера
  - Использование: отображение информации в модальном окне просмотра
- `isModalOpen` (boolean): флаг открытия модального окна просмотра
  - Инициализация: false
  - Обновление: при открытии/закрытии модального окна
  - Использование: управление видимостью Dialog просмотра
- `isEditModalOpen` (boolean): флаг открытия модального окна редактирования
  - Инициализация: false
  - Обновление: при открытии/закрытии модального окна редактирования
  - Использование: управление видимостью Dialog редактирования
- `editingInterviewer` (Interviewer | null): интервьюер для редактирования
  - Инициализация: null
  - Обновление: при открытии модального окна редактирования
  - Использование: передача данных в InterviewerEditModal
- `returnToViewModal` (boolean): флаг возврата к модальному окну просмотра
  - Инициализация: false
  - Обновление: при открытии редактирования из модального окна просмотра
  - Использование: управление навигацией между модальными окнами

### Константы:
- `itemsPerPage` (number = 10): количество элементов на странице
  - Используется для вычисления totalPages, startIndex, endIndex

### Вычисляемые значения:
- `filteredInterviewers` (Interviewer[]): отфильтрованные интервьюеры
  - Фильтрация по searchQuery (имя, фамилия, отчество, email)
  - Фильтрация по statusFilter (all/active/inactive)
- `totalPages` (number): общее количество страниц
  - Вычисляется: Math.ceil(filteredInterviewers.length / itemsPerPage)
- `startIndex` (number): начальный индекс для текущей страницы
  - Вычисляется: (currentPage - 1) * itemsPerPage
- `endIndex` (number): конечный индекс для текущей страницы
  - Вычисляется: startIndex + itemsPerPage
- `paginatedInterviewers` (Interviewer[]): интервьюеры текущей страницы
  - Вычисляется: filteredInterviewers.slice(startIndex, endIndex)
- `totalCount` (number): общее количество интервьюеров
  - Вычисляется: mockInterviewers.length
- `activeCount` (number): количество активных интервьюеров
  - Вычисляется: mockInterviewers.filter(i => i.isActive).length
- `inactiveCount` (number): количество неактивных интервьюеров
  - Вычисляется: mockInterviewers.filter(i => !i.isActive).length

### Hooks:
- `useToast()`: для отображения уведомлений
  - Используется для: подтверждения удаления интервьюера

## Функции и обработчики

### getInitials(interviewer)
- **Назначение**: Получение инициалов интервьюера для аватара
- **Параметры**: 
  - `interviewer`: Interviewer - интервьюер
- **Возвращает**: string - инициалы (например, "AA" для Artur Akimau)
- **Логика**: 
  1. Берет первую букву имени и преобразует в верхний регистр
  2. Берет первую букву фамилии и преобразует в верхний регистр
  3. Возвращает конкатенацию двух букв
- **Используется для**: 
  - Отображения инициалов в аватаре в таблице
  - Отображения инициалов в аватаре в модальном окне просмотра

### getFullName(interviewer)
- **Назначение**: Получение полного имени интервьюера (Фамилия Имя)
- **Параметры**: 
  - `interviewer`: Interviewer - интервьюер
- **Возвращает**: string - полное имя
- **Логика**: 
  1. Конкатенирует lastName и firstName через пробел
  2. Возвращает строку в формате "Фамилия Имя"
- **Используется для**: 
  - Отображения полного имени в таблице
  - Отображения полного имени в модальном окне просмотра
  - Отображения в заголовке модального окна редактирования

### getShortName(interviewer)
- **Назначение**: Получение короткого имени интервьюера (только Имя)
- **Параметры**: 
  - `interviewer`: Interviewer - интервьюер
- **Возвращает**: string - короткое имя
- **Логика**: 
  1. Возвращает firstName
- **Используется для**: 
  - Отображения короткого имени под полным именем в таблице
  - Отображения короткого имени в модальном окне просмотра

### handleDeleteInterviewer(interviewer, closeModal)
- **Назначение**: Удаление интервьюера с подтверждением
- **Параметры**: 
  - `interviewer`: Interviewer - интервьюер для удаления
  - `closeModal`: boolean - закрыть ли модальное окно после удаления (по умолчанию false)
- **Логика**: 
  1. Показывает toast с предупреждением и подтверждением
  2. Toast содержит два действия:
     - "Отмена" (серая кнопка, ничего не делает)
     - "Удалить" (красная кнопка, выполняет удаление)
  3. При подтверждении удаления:
     - TODO: Реализовать удаление через API
     - Если closeModal === true, закрывает модальное окно и сбрасывает selectedInterviewer
- **Используется для**: 
  - Удаления интервьюера из таблицы (кнопка удаления)
  - Удаления интервьюера из модального окна просмотра (кнопка "Удалить")

### Обработчики изменения состояния
- **setSearchQuery**: обновление поискового запроса
  - При изменении сбрасывает currentPage на 1
- **setStatusFilter**: обновление фильтра статуса
  - При изменении сбрасывает currentPage на 1
- **setCurrentPage**: обновление текущей страницы
  - Используется при клике на кнопки пагинации
- **setSelectedInterviewer**: установка выбранного интервьюера
  - Используется при клике на строку таблицы или имя интервьюера
- **setIsModalOpen**: управление видимостью модального окна просмотра
  - Используется при открытии/закрытии модального окна
- **setIsEditModalOpen**: управление видимостью модального окна редактирования
  - Используется при открытии/закрытии модального окна редактирования
- **setEditingInterviewer**: установка интервьюера для редактирования
  - Используется при открытии модального окна редактирования
- **setReturnToViewModal**: установка флага возврата к модальному окну просмотра
  - Используется при открытии редактирования из модального окна просмотра

### Обработчики в InterviewerEditModal

#### handleSave()
- **Назначение**: Сохранение данных интервьюера
- **Логика**: 
  1. Проверяет обязательные поля (firstName, lastName, email)
  2. Если поля не заполнены, показывает alert
  3. Если все поля заполнены, вызывает onSave с данными формы
- **Используется для**: 
  - Сохранения нового интервьюера
  - Сохранения изменений существующего интервьюера

#### handleAutofill()
- **Назначение**: Автозаполнение ссылки на календарь из email
- **Логика**: 
  1. Если email заполнен, генерирует ссылку на Google Calendar
  2. Формат ссылки: `https://calendar.google.com/calendar/embed?src=${email}`
  3. Обновляет calendarLink в formData
- **Используется для**: 
  - Быстрого заполнения ссылки на календарь при создании интервьюера

#### useEffect для инициализации формы
- **Назначение**: Заполнение формы данными интервьюера при открытии модального окна
- **Логика**: 
  1. Если interviewer !== null (режим редактирования):
     - Заполняет форму данными интервьюера
     - middleName и calendarLink могут быть пустыми (используется || '')
  2. Если interviewer === null (режим создания):
     - Сбрасывает форму в начальное состояние
     - isActive устанавливается в true
- **Зависимости**: [interviewer, isOpen]

## Фильтрация и поиск

### Поиск (searchQuery)
- **Поле**: TextField с иконкой MagnifyingGlassIcon
- **Логика**: 
  - Ищет по имени, фамилии, отчеству, email
  - Без учета регистра (toLowerCase)
  - Фильтрация происходит в реальном времени при вводе
  - При изменении поиска сбрасывает currentPage на 1
- **Поведение**: 
  - При вводе обновляется filteredInterviewers
  - Обновляется отображаемый список интервьюеров

### Фильтр по статусу (statusFilter)
- **Кнопки**: "Всего", "Активных", "Неактивных"
- **Логика**: 
  - "Всего": показывает всех интервьюеров (statusFilter === 'all')
  - "Активных": показывает только активных (statusFilter === 'active')
  - "Неактивных": показывает только неактивных (statusFilter === 'inactive')
  - При изменении фильтра сбрасывает currentPage на 1
- **Поведение**: 
  - Активная кнопка имеет variant="solid" и color="blue"
  - Неактивные кнопки имеют variant="soft" и color="gray"
  - При клике обновляется filteredInterviewers

## Пагинация

- **Размер страницы**: 10 элементов (itemsPerPage = 10)
- **Кнопки**: 
  - Первая страница (DoubleArrowLeftIcon): переход на страницу 1
  - Предыдущая страница (ChevronLeftIcon): переход на предыдущую страницу
  - Информация о странице: "Страница X из Y" (в центре, на цветном фоне)
  - Следующая страница (ChevronRightIcon): переход на следующую страницу
  - Последняя страница (DoubleArrowRightIcon): переход на последнюю страницу
- **Логика**: 
  - `totalPages` вычисляется на основе `filteredInterviewers.length`
  - `startIndex` и `endIndex` вычисляются на основе `currentPage` и `itemsPerPage`
  - `paginatedInterviewers` содержит только интервьюеров текущей страницы
- **Поведение**: 
  - Отображается только если totalPages > 1
  - Кнопки "Первая" и "Предыдущая" отключены на первой странице
  - Кнопки "Следующая" и "Последняя" отключены на последней странице
  - При переключении страницы обновляется отображаемый список
