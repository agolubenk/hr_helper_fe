# Документация страницы: Чат рекрутера (recr-chat/page.tsx)

## Общее описание

Страница чата рекрутера с кандидатами. Централизованный интерфейс для общения со всеми кандидатами из разных мессенджеров (Telegram, WhatsApp, Viber), управления контактами, информацией о кандидатах, настройками вакансий и workflow процессами.

**Путь**: `/recr-chat`

**Размер файла**: ~9754 строки кода (очень большой компонент)

## Компоненты страницы

### 1. RecrChatPage (основной компонент)
- **Расположение**: `@/app/recr-chat/page.tsx`
- **Назначение**: Главный компонент страницы чата рекрутера
- **Функциональность**: 
  - Управление тремя основными вкладками (Candidates, Chat, Настройки вакансии)
  - Отображение списка кандидатов с фильтрацией
  - Чат с кандидатами через WorkflowChat
  - Управление информацией о кандидатах
  - Настройки вакансий
  - Управление workflow процессами
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - WorkflowChat: компонент чата с кандидатами
  - SlotsPanel: панель свободных слотов для интервью
  - useToast: для отображения уведомлений
  - Sidebar: содержит ссылку на эту страницу в разделе "Рекрутинг"

### 2. Левая колонка (leftColumn)
- **Компонент**: Box с тремя вкладками
- **Назначение**: Навигация и список кандидатов/чатов/настроек
- **Вкладки**:
  - **Candidates**: список всех кандидатов
    - Поиск по имени
    - Фильтрация по статусу просмотра
    - Отображение непрочитанных сообщений
    - Отображение подозрений на дубликаты
  - **Chat**: список диалогов
    - Непрочитанные сообщения
    - Последнее сообщение
    - Время последней активности
  - **Настройки вакансии**: настройки вакансии
    - Текст вакансии
    - Рекрутеры
    - Заказчики
    - Вопросы
    - Интеграции
    - Статусы
    - Зарплата
    - Интервью
    - Scorecard
    - Обработка данных
    - История

### 3. Правая колонка (rightColumn)
- **Компонент**: Box с информацией о кандидате
- **Назначение**: Отображение и редактирование информации о выбранном кандидате
- **Вкладки**:
  - **Info**: основная информация о кандидате
    - Аватар с возможностью загрузки и редактирования
    - Личные данные
    - Контакты (email, телефоны, мессенджеры, социальные сети)
    - Информация о заявке
    - Вакансии кандидата
    - Подозрения на дубликаты
  - **History**: история взаимодействий
    - История изменений статуса
    - История сообщений
    - История интервью
  - **Activity**: активность кандидата
    - События и действия
  - **Documents**: документы кандидата
    - Загруженные документы
    - Управление видимостью документов
    - Группы документов

### 4. WorkflowChat (компонент чата)
- **Расположение**: `@/components/workflow/WorkflowChat`
- **Назначение**: Компонент чата с кандидатами
- **Функциональность**: 
  - Отображение истории сообщений
  - Отправка сообщений
  - Обработка команд через теги
  - Интеграция с мессенджерами
- **Связи**: Используется для общения с кандидатами

### 5. SlotsPanel (панель слотов)
- **Расположение**: `@/components/workflow/SlotsPanel`
- **Назначение**: Панель свободных слотов для интервью
- **Функциональность**: 
  - Отображение свободных слотов по неделям
  - Копирование доступного времени
- **Связи**: Используется для быстрого копирования доступного времени

### 6. Workflow кнопки и тогглеры этапов
- **Компонент**: Flex с кнопками быстрых действий и тогглерами этапов
- **Назначение**: Быстрые действия для workflow процессов и выбор типа процесса
- **Тогглеры этапов процесса**:
  - **Скрининг** (Screening) - всегда доступен, 30 минут
  - **Динамические кнопки этапов-встреч** - формируются автоматически из этапов найма с меткой "встреча" (`isMeeting = true`)
    - Количество кнопок: 0 и более (зависит от количества этапов с `isMeeting = true`)
    - Названия кнопок: берутся из названий этапов найма, отмеченных чекбоксом "встреча"
    - Если этапов-встреч нет, тогглер может быть пустым или содержать только "Скрининг"
    - Настройки этапов-встреч задаются на странице `/company-settings/recruiting/stages`
- **Панель настроек встречи** (отображается только при выборе этапа-встречи):
  - **Тогглер формата встречи** (Онлайн/Офис):
    - Отображается только если `showOffices = true` для выбранного этапа
    - Позволяет выбрать формат: Онлайн или Офис
  - **Выбор офиса** (Минск, Варшава, Гомель):
    - Отображается только если выбран формат "Офис" и `showOffices = true`
    - Показывается сразу после кнопки выбора формата "Офис"
  - **Выбор интервьюеров**:
    - Отображается только если `showInterviewers = true` для выбранного этапа
    - Чекбоксы для выбора нескольких интервьюеров
    - Автор (текущий пользователь) добавляется в начало списка
- **Кнопка "Слоты"**: открывает панель свободных слотов для интервью
- **Связь с настройками этапов**:
  - Настройки этапов (`isMeeting`, `showOffices`, `showInterviewers`) задаются на странице `/company-settings/recruiting/stages`
  - Эти настройки определяют, какие элементы отображаются в панели настроек встречи

### 7. Модальное окно редактирования аватара
- **Компонент**: Dialog с формой редактирования фото
- **Назначение**: Загрузка и редактирование аватара кандидата
- **Функциональность**: 
  - Загрузка фото (drag-and-drop или кнопка)
  - Обрезка фото в квадрат
  - Карусель фото (несколько фото для одного кандидата)
  - Выбор главного фото
  - Удаление фото

### 8. Модальное окно редактирования кандидата
- **Компонент**: Dialog с формой редактирования
- **Назначение**: Редактирование информации о кандидате
- **Функциональность**: 
  - Редактирование личных данных
  - Управление контактами (email, телефоны, мессенджеры, социальные сети)
  - Отметка как просмотренного
  - Управление подозрениями на дубликаты

### 9. Модальное окно дубликатов
- **Компонент**: Dialog с информацией о дубликатах
- **Назначение**: Управление подозрениями на дубликаты кандидатов
- **Функциональность**: 
  - Сравнение данных кандидатов
  - Подсветка совпадающих полей
  - Объединение кандидатов
  - Отметка как разных кандидатов

### 10. Модальное окно загрузки документов
- **Компонент**: Dialog с формой загрузки
- **Назначение**: Загрузка документов кандидата
- **Функциональность**: 
  - Загрузка файлов (drag-and-drop или кнопка)
  - Выбор видимости документа (только я, группа, все)
  - Выбор группы для документа

### 11. Настройки вакансии
- **Компонент**: Множество вкладок с настройками
- **Назначение**: Полная настройка вакансии
- **Вкладки**:
  - **Текст**: текст вакансии (название, отдел, заголовок, обязанности, требования и т.д.)
  - **Рекрутеры**: выбор рекрутеров для вакансии
  - **Заказчики**: выбор заказчиков (интервьюеров)
  - **Вопросы**: настройка вопросов для кандидатов
  - **Интеграции**: интеграции с внешними системами (Huntflow)
  - **Статусы**: настройка статусов кандидатов
  - **Зарплата**: настройка зарплатных вилок по грейдам
  - **Интервью**: настройка интервью (формат, интервьюеры)
  - **Scorecard**: настройка Scorecard
  - **Обработка данных**: настройка обработки данных
  - **История**: история изменений вакансии с возможностью восстановления версий

### Candidate (кандидат)
```typescript
interface Candidate {
  id: string                                    // Уникальный идентификатор кандидата
  name: string                                 // Имя кандидата
  position: string                             // Позиция/должность
  status: string                                // Статус кандидата (Interview, New, Rejected, Offer и т.д.)
  statusColor: string                           // Цвет статуса
  avatar: string                               // Инициалы для аватара
  timeAgo: string                              // Время последней активности
  unread: number                               // Количество непрочитанных сообщений
  unreadSources: Record<string, number>        // Количество непрочитанных сообщений по источникам
  isViewed: boolean                            // Флаг просмотренности информации
  hasUnviewedChanges: boolean                  // Флаг наличия непросмотренных изменений
  email: string                                // Основной email
  phone: string                                // Основной телефон
  emails: string[]                             // Массив всех email
  phones: string[]                             // Массив всех телефонов
  location: string                             // Местоположение

  // Мессенджеры (старые поля для обратной совместимости)

  viber?: string
  telegram?: string
  // Массивы контактов мессенджеров
  whatsapps?: string[]
  vibers?: string[]
  telegrams?: string[]
  // Социальные сети (старые поля)
  vk?: string
  linkedin?: string
  dribbble?: string
  behance?: string
  pinterest?: string
  habrCareer?: string
  github?: string
  instagram?: string

  twitter?: string

  linkedins?: string[]
  dribbbles?: string[]
  behances?: string[]
  pinterests?: string[]
  habrCareers?: string[]
  githubs?: string[]
  instagrams?: string[]
  facebooks?: string[]
  twitters?: string[]
  kaggles?: string[]

  // Дополнительные поля
```

## Состояние компонента

### State переменные (основные):
- `leftTab` ('candidates' | 'chat' | 'vacancy-settings'): активная вкладка левой колонки
- `rightTab` ('info' | 'history' | 'activity' | 'documents'): активная вкладка правой колонки
- `selectedCandidate`: выбранный кандидат для чата
- `searchQuery`: поисковый запрос
- `isRightColumnOpen`: флаг открытия правой колонки (для мобильных)
- `isMobile`: флаг мобильного устройства
- `windowWidth`: ширина окна

### State для управления аватаром:
- `avatarModalOpen`: флаг открытия модального окна аватара
- `isEditingPhoto`: флаг редактирования фото
- `uploadedPhotoForEdit`: загруженное фото для редактирования
- `cropArea`: область обрезки фото { x, y, size }
- `isDragging`: флаг перетаскивания фото
- `imageDimensions`: размеры изображения
- `candidatePhotos`: объект с фото кандидатов (Record<string, string[]>)

### State для управления документами:
- `pendingDocumentFile`: файл документа для загрузки
- `documentUploadModalOpen`: флаг открытия модального окна загрузки документа
- `documentVisibilityGroup`: видимость документа ('only-me' | 'group' | 'all')
- `selectedGroup`: выбранная группа для документа
- `uploadedDocuments`: массив загруженных документов
- `isDraggingDocument`: флаг перетаскивания документа

### State для workflow:
- `selectedWorkflow`: выбранный тип workflow ('screening' | string) - 'screening' для скрининга, или ID этапа найма с `isMeeting = true`
- `interviewFormat`: формат интервью ('online' | 'office')
- `selectedInterviewers`: выбранные интервьюеры (массив ID)
- `slotsOpen`: флаг открытия панели слотов
- `selectedOffice`: выбранный офис ('minsk' | 'warsaw' | 'gomel')
- `recruitmentStagesWithMeetings`: этапы найма с настройками встреч (массив объектов с полями `id`, `name`, `description`, `color`, `isMeeting`, `showOffices`, `showInterviewers`)
- `meetingStages`: вычисляемое значение - этапы-встречи для формирования тогглера (этапы с `isMeeting = true`)
- `selectedMeetingStage`: вычисляемое значение - настройки выбранного этапа-встречи (null если выбран "Скрининг")
- `showOfficesForSelectedStage`: вычисляемое значение - флаг отображения офисов для выбранного этапа (`selectedMeetingStage?.showOffices ?? false`)
- `showInterviewersForSelectedStage`: вычисляемое значение - флаг отображения интервьюеров для выбранного этапа (`selectedMeetingStage?.showInterviewers ?? false`)

### State для настроек вакансии:
- `selectedSettingTab`: активная вкладка настроек вакансии
- `isPublished`: флаг публикации вакансии
- `publicationUrl`: URL опубликованной вакансии
- `editHistory`: история изменений вакансии
- `selectedHistoryItem`: выбранный элемент истории
- `restoreConfirmationOpen`: флаг открытия подтверждения восстановления
- `versionToRestore`: версия для восстановления
- Множество полей для текста вакансии (vacancyTitle, vacancyDepartment, vacancyHeader и т.д.)
- `fieldSettings`: настройки полей вакансии
- `selectedRecruiters`: выбранные рекрутеры
- `mainRecruiter`: главный рекрутер
- `selectedVacancyInterviewers`: выбранные интервьюеры для вакансии
- `finalInterviewInterviewers`: интервьюеры для финального интервью
- `activeStages`: активные этапы найма
- `activeGrades`: активные грейды
- `salaryRanges`: зарплатные вилки по грейдам и валютам
- `isGrossFormat`: формат зарплаты (gross/net)
- `interviewMeetings`: настройки интервью
- `scorecardLinkUrl`: URL ссылки на Scorecard
- `scorecardLinkTitle`: название ссылки на Scorecard
- `scorecardLinkPosition`: позиция ссылки ('start' | 'end')
- `scorecardLocalActive`: флаг активности локального Scorecard
- `useUnifiedPrompt`: использование единого промпта
- `analysisPrompt`: промпт для анализа
- `integrationPartner`: партнер интеграции ('' | 'huntflow')
- `huntflowVacancyId`: ID вакансии в Huntflow
- `questionsLinksByOffice`: ссылки на вопросы по офисам

### State для дубликатов:
- `duplicateModalOpen`: флаг открытия модального окна дубликатов
- `duplicateCandidates`: массив кандидатов-дубликатов

### Refs:
- `uploadedPhotoRef`: ссылка на элемент изображения для обрезки
- `rightColumnRef`: ссылка на правую колонку

## Функции и обработчики

### Управление фотографиями

#### getInitials(name)
- **Назначение**: Получение инициалов из имени кандидата
- **Параметры**: 
  - `name`: string - полное имя кандидата
- **Возвращает**: string - инициалы (2 заглавные буквы)
- **Логика**: 
  - Если одно слово: берет первые 2 буквы
  - Если несколько слов: берет первую букву первого и второго слова
- **Используется для**: Отображения инициалов в аватаре, если нет фото

#### handleAvatarClick()
- **Назначение**: Открытие модального окна для управления фотографиями
- **Поведение**: Устанавливает avatarModalOpen в true

#### handlePhotoUpload(file)
- **Назначение**: Загрузка фотографии кандидата
- **Параметры**: 
  - `file`: File - файл изображения
- **Поведение**: 
  - Проверяет тип файла (должен быть изображением)
  - Читает файл через FileReader
  - Открывает режим редактирования с обрезкой
  - Инициализирует область обрезки по центру изображения

#### handleSaveCroppedPhoto()
- **Назначение**: Сохранение обрезанного фото
- **Поведение**: 
  - Обрезает фото в квадрат через cropImageToSquare
  - Добавляет фото в начало массива фотографий кандидата
  - Закрывает режим редактирования

#### handleSelectMainPhoto(photoIndex)
- **Назначение**: Выбор главного фото в карусели
- **Параметры**: 
  - `photoIndex`: number - индекс фото в массиве
- **Поведение**: Перемещает выбранное фото в начало массива

### Управление документами

#### handleDocumentFileSelect(file)
- **Назначение**: Обработка выбора файла документа
- **Параметры**: 
  - `file`: File - файл документа
- **Поведение**: 
  - Сохраняет файл в pendingDocumentFile
  - Открывает модальное окно загрузки
  - Переключает на вкладку Documents

#### handleDocumentUploadConfirm()
- **Назначение**: Подтверждение загрузки документа
- **Поведение**: 
  - Создает объект документа с метаданными
  - Применяет настройки видимости
  - Добавляет документ в список
  - Закрывает модальное окно

### Управление контактами

#### getEmails()
- **Назначение**: Получение массива email адресов кандидата
- **Возвращает**: string[] - массив email адресов
- **Поведение**: Возвращает массив emails или создает из старого поля email

#### getPhones()
- **Назначение**: Получение массива телефонов кандидата
- **Возвращает**: string[] - массив телефонов
- **Поведение**: Возвращает массив phones или создает из старого поля phone

#### handleEmailSave(index)
- **Назначение**: Сохранение изменений email адреса
- **Параметры**: 
  - `index`: number - индекс email в массиве
- **Поведение**: Обновляет email по индексу и обновляет основное поле email

#### handlePhoneSave(index)
- **Назначение**: Сохранение изменений телефона
- **Параметры**: 
  - `index`: number - индекс телефона в массиве
- **Поведение**: Обновляет телефон по индексу и обновляет основное поле phone

#### handleAddContact()
- **Назначение**: Добавление нового контакта (email или телефон)
- **Поведение**: 
  - Проверяет на дубликаты
  - Добавляет контакт в массив
  - Обновляет основное поле

#### getSocialContacts(platform)
- **Назначение**: Получение массива контактов для социальной платформы

- **Параметры**: 
  - `currentStatus`: string - текущий статус
- **Возвращает**: string - следующий статус
- **Поведение**: Возвращает следующий статус из массива statusOrder

#### handleStatusChange(newStatus)
- **Назначение**: Изменение статуса кандидата
- **Параметры**: 
  - `newStatus`: string - новый статус
- **Поведение**: 
  - Создает запись в активности
  - Для статуса "Rejected" откладывает добавление до выбора причины
  - Обновляет статус и цвет статуса кандидата

#### handleRejectionReasonChange(reason)
- **Назначение**: Обработка изменения причины отказа
- **Параметры**: 
  - `reason`: string - выбранная причина отказа
- **Поведение**: 
  - Добавляет отложенную запись о статусе Rejected в активность
  - Объединяет изменение статуса, комментарий и причину отказа

- **Назначение**: Получение цвета для статуса
- **Параметры**:

- **Поведение**: Возвращает цвет из маппинга статусов

### Поиск дубликатов

#### isMatch(value1, value2)
- **Назначение**: Проверка совпадения двух значений
- **Параметры**: 
  - `value1`: any - первое значение
  - `value2`: any - второе значение
- **Возвращает**: boolean - true если совпадают
- **Поведение**: 
  - Сравнивает строки без учета регистра и пробелов
  - Сравнивает массивы на пересечение

#### calculateDuplicateProbability(candidate1, candidate2)
- **Назначение**: Расчет вероятности дубликата двух кандидатов
- **Параметры**: 
  - `candidate1`: any - первый кандидат
  - `candidate2`: any - второй кандидат
- **Возвращает**: number - вероятность в процентах (0-100)
- **Поведение**: 
  - Сравнивает кандидатов по множеству полей с разными весами
  - Вычисляет процент совпадений
