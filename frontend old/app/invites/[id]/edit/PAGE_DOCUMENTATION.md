# Документация страницы: Редактирование инвайта (invites/[id]/edit/page.tsx)

## Общее описание

Страница редактирования инвайта на интервью. Позволяет изменить данные инвайта: кандидата, вакансию, время интервью, формат и другие параметры.

## Компоненты страницы

### 1. Заголовок страницы
- **Компонент**: `Flex` с кнопкой "Назад"
- **Назначение**: Навигация и заголовок страницы
- **Логика**: 
  - Кнопка "Назад" возвращает на страницу детального просмотра
  - Отображает название страницы "Редактирование инвайта"
- **Связи**: Навигация на `/invites/[id]`

### 2. Форма редактирования
- **Компонент**: Card с полями формы
- **Назначение**: Редактирование данных инвайта
- **Поля**:
  - **Название кандидата**: TextField для ввода имени кандидата
  - **Дата и время интервью**: TextField type="datetime-local" (обязательное поле)
  - **Статус**: Select с опциями из STATUS_OPTIONS (pending, sent, completed, cancelled)
  - **Формат интервью**: Select с опциями (online, office)
  - **Офис**: Select с списком офисов (отображается только для офисного формата, обязательное поле)
  - **Длительность (минуты)**: TextField type="number" (опционально, по умолчанию используется стандартная длительность)
  - **Уровень кандидата**: Select с опциями из CANDIDATE_GRADE_OPTIONS (Intern, Junior, Middle, Senior, Lead, Principal)
  - **Ссылка на кандидата (Huntflow)**: TextField для ввода ссылки на кандидата
  - **Вакансия**: Select с списком вакансий
  - **Ссылка на шаблон scorecard**: TextField для ввода ссылки на шаблон
- **Логика**: 
  - Заполнение полей текущими данными инвайта при загрузке
  - Преобразование даты интервью в локальный формат для input[type="datetime-local"]
  - Валидация обязательных полей перед сохранением
  - Условное отображение поля "Офис" только для офисного формата
  - Сохранение изменений через API
- **Валидация**: 
  - interview_datetime: обязательное поле
  - office_id: обязательное поле для офисного интервью (проверяется на сервере)

## Состояние компонента

### State переменные:
- `invite` (Invite | null): данные редактируемого инвайта
  - Инициализация: null
  - Обновление: при загрузке данных через loadData()
  - Использование: отображение текущих данных и заполнение формы
- `vacancies` (Vacancy[]): список вакансий для выбора
  - Инициализация: пустой массив
  - Обновление: при загрузке данных через loadData()
  - Использование: выбор вакансии в Select
- `offices` (Array<{ id: number; name: string; address?: string }>): список офисов для выбора
  - Инициализация: пустой массив
  - Обновление: при загрузке данных через loadData()
  - Использование: выбор офиса для очного интервью
- `loading` (boolean): флаг загрузки данных
  - Инициализация: true
  - Обновление: устанавливается в false при завершении загрузки
  - Использование: блокировка интерфейса и отображение индикатора загрузки
- `saving` (boolean): флаг сохранения данных
  - Инициализация: false
  - Обновление: устанавливается в true при начале сохранения, false при завершении
  - Использование: блокировка кнопки сохранения и отображение индикатора
- `formData` (объект): данные формы редактирования
  - Структура:
    - candidate_name: string - имя кандидата
    - interview_datetime: string - дата и время интервью (локальный формат для input)
    - status: string - статус инвайта
    - interview_format: 'online' | 'office' - формат интервью
    - custom_duration_minutes: string - длительность интервью в минутах
    - candidate_grade: string - грейд кандидата
    - candidate_url: string - ссылка на кандидата в Huntflow
    - vacancy_id: string - ID вакансии
    - scorecard_template_url: string - ссылка на шаблон Scorecard
    - office_id: string - ID офиса (для очного интервью)
  - Инициализация: пустые значения
  - Обновление: при загрузке данных инвайта и изменении полей формы
  - Использование: отправка данных на сервер при сохранении

## Функции и обработчики

### loadData()
- **Назначение**: Загрузка данных для редактирования инвайта
- **Параметры**: Нет (использует inviteId из состояния)
- **Логика**: 
  1. Проверяет наличие inviteId, если нет - выходит
  2. Устанавливает `loading = true`
  3. Имитирует задержку API (300мс)
  4. Загружает моковые данные инвайта, вакансий и офисов
  5. Преобразует дату интервью в локальный формат для input[type="datetime-local"]
  6. Заполняет форму данными инвайта
  7. Устанавливает `loading = false` в finally
  8. При ошибке показывает toast и редиректит на список
- **Вызывается**: 
  - При монтировании компонента (useEffect)
  - При изменении inviteId из URL
- **Связи**: 
  - invitesApi.getById: API для получения инвайта по ID
  - vacanciesApi.getAll: API для получения списка вакансий

### handleSave()
- **Назначение**: Сохранение изменений инвайта
- **Параметры**: Нет (использует formData и inviteId из состояния)
- **Логика**: 
  1. Проверяет наличие inviteId, если нет - выходит
  2. Валидирует обязательные поля (interview_datetime)
  3. Преобразует данные формы в формат API:
     - interview_datetime: локальный формат → ISO строка
     - custom_duration_minutes: строка → число или null
     - office_id: строка → число или null (только для офисного интервью)
  4. Устанавливает `saving = true`
  5. Отправляет запрос на обновление через invitesApi.update (TODO: раскомментировать)
  6. При успехе показывает toast и редиректит на детальную страницу
  7. При ошибке показывает toast
  8. Устанавливает `saving = false` в finally
- **Валидация**: 
  - interview_datetime: обязательное поле
  - office_id: обязательное поле для офисного интервью (проверяется на сервере)
- **Поведение**: 
  - Вызывается при клике на кнопку "Сохранить"
  - Блокирует кнопку сохранения во время сохранения
- **Связи**: 
  - invitesApi.update: API для обновления инвайта
  - router.push: навигация на страницу детального просмотра после успешного сохранения

## URL параметры

### [id]
- **Тип**: Динамический параметр маршрута
- **Назначение**: ID инвайта для редактирования
- **Пример**: `/invites/123/edit` редактирует инвайт с ID 123
- **Обработка**: 
  - Читается через `useParams()`
  - Используется для загрузки данных инвайта
