# Документация страницы: Детальный просмотр вакансии (vacancies/[id]/page.tsx)

## Общее описание

Страница редиректа с URL `/vacancies/[id]` на `/vacancies?edit=[id]`. Обеспечивает обратную совместимость со старыми ссылками и автоматически открывает модальное окно редактирования вакансии.

## Компоненты страницы

### Компонент редиректа
- **Тип**: Функциональный компонент без UI
- **Назначение**: Выполнение редиректа на страницу списка вакансий с параметром редактирования
- **Логика**: 
  - Читает ID вакансии из URL параметра `[id]`
  - Выполняет редирект на `/vacancies?edit=[id]`
  - Если ID отсутствует - редирект на `/vacancies`
- **Поведение**: 
  - Не рендерит никакого контента (возвращает `null`)
  - Выполняется один раз при монтировании компонента

## Состояние компонента

### State переменные:
- Нет (компонент только выполняет редирект)

## Функции и обработчики

### useEffect для редиректа
- **Назначение**: Выполнение редиректа при загрузке компонента
- **Логика**: 
  - Если ID вакансии существует - редирект на `/vacancies?edit=[id]`
  - Если ID отсутствует - редирект на `/vacancies`
- **Поведение**: 
  - Выполняется один раз при монтировании
  - Использует `router.replace()` (не добавляет запись в историю)
  - Кодирует ID через `encodeURIComponent()` для безопасной передачи

## URL параметры

### [id]
- **Тип**: Динамический параметр маршрута
- **Назначение**: ID вакансии для редактирования
- **Пример**: `/vacancies/123` → редирект на `/vacancies?edit=123`
- **Обработка**: 
  - Читается через `useParams()`
  - Передается в URL параметр `?edit=` на странице списка

## Связи с другими страницами

- `/vacancies/page.tsx` - страница списка вакансий, которая обрабатывает параметр `?edit=` и открывает модальное окно редактирования

## Особенности

- Не рендерит контент (возвращает `null`)
- Использует `router.replace()` вместо `router.push()` (не добавляет запись в историю)
- Обеспечивает обратную совместимость со старыми ссылками
- Автоматически открывает модальное окно редактирования на странице списка

## Причина существования

- Обеспечивает обратную совместимость со старыми ссылками на детальную страницу вакансии
- Упрощает навигацию: вместо отдельной страницы используется модальное окно на странице списка
- Позволяет использовать глубокие ссылки на редактирование вакансии
