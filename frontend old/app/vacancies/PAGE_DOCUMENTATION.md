# Документация страницы: Вакансии (vacancies/page.tsx)

## Общее описание

Страница управления вакансиями компании. Позволяет просматривать, фильтровать, создавать и редактировать вакансии в двух режимах отображения: карточки и список. Страница является центральным хабом для управления всеми вакансиями компании и предоставляет быстрый доступ к основным операциям с вакансиями.

**Путь**: `/vacancies`

## Интерфейсы и типы

### Vacancy (интерфейс вакансии)
```typescript
interface Vacancy {
  id: number                    // Уникальный идентификатор вакансии
  title: string                 // Название вакансии
  status: 'active' | 'inactive' // Статус вакансии
  recruiter: string             // Имя рекрутера, ведущего вакансию
  locations: string[]           // Массив локаций (города, удаленная работа)
  interviewers: number          // Количество назначенных интервьюеров
  date: string | null           // Дата создания/обновления (может быть null)
  hasWarning: boolean           // Флаг наличия предупреждения
  warningText?: string          // Текст предупреждения (опционально)
}
```

## Состояние компонента

### State переменные:
- `viewMode` ('cards' | 'list'): режим отображения вакансий
  - Инициализация: 'cards'
  - Обновление: при клике на переключатель вида
  - Использование: определяет, какой компонент использовать для отображения (VacancyCard или VacancyListItem)

- `searchQuery` (string): поисковый запрос для фильтрации вакансий
  - Инициализация: пустая строка
  - Обновление: при вводе в поле поиска
  - Использование: фильтрация по названию или ID вакансии

- `selectedRecruiter` (string): выбранный рекрутер для фильтрации
  - Инициализация: 'all'
  - Обновление: при выборе в фильтре по рекрутеру
  - Использование: фильтрация вакансий по рекрутеру

- `selectedStatus` (string): выбранный статус для фильтрации
  - Инициализация: 'all'
  - Обновление: при выборе в фильтре по статусу
  - Использование: фильтрация вакансий по статусу

- `viewVacancyId` (number | null): ID вакансии для просмотра в модальном окне
  - Инициализация: null
  - Обновление: при клике на вакансию для просмотра
  - Использование: открывает модальное окно в режиме просмотра

- `editVacancyId` (number | null): ID вакансии для редактирования в модальном окне
  - Инициализация: null
  - Обновление: при клике на кнопку редактирования или через URL параметр ?edit
  - Использование: открывает модальное окно в режиме редактирования

- `statusOverrides` (Record<number, 'active' | 'inactive'>): объект переопределений статусов

  - Структура: ключ - ID вакансии, значение - новый статус

## Функции и обработчики

### getStatus(v)
- **Назначение**: Получение актуального статуса вакансии с учетом переопределений
- **Параметры**: 
  - `v`: объект вакансии из mockVacancies
- **Возвращает**: 'active' | 'inactive' - актуальный статус вакансии
- **Логика**: 
  1. Проверяет наличие переопределения статуса в `statusOverrides[v.id]`
  2. Если переопределение есть - возвращает его
  3. Если переопределения нет - возвращает оригинальный статус `v.status`
- **Используется для**: 
  - Отображения актуального статуса с учетом временных изменений
  - Фильтрации вакансий по статусу
  - Подсчета статистики (активные/неактивные)
- **Особенности**: 
  - Позволяет изменять статус без немедленного сохранения на сервер
  - Переопределения применяются только локально в UI

### useEffect для URL параметра ?edit
- **Назначение**: Открытие модального окна редактирования через URL параметр
- **Зависимости**: [searchParams]
- **Логика**: 
  1. Читает параметр 'edit' из URL через `searchParams.get('edit')`
  2. Если параметр существует - парсит его в число
  3. Если ID валиден (не NaN) - открывает модальное окно редактирования
  4. Закрывает модальное окно просмотра, если оно было открыто
- **Используется для**: 
  - Глубоких ссылок на редактирование вакансии
  - Открытия страницы с уже открытым модальным окном редактирования
- **Пример использования**: `/vacancies?edit=123` открывает редактирование вакансии с ID 123

## Вычисляемые значения

### Статистика по вакансиям:
- `totalVacancies`: общее количество вакансий
  - Вычисление: `mockVacancies.length`
  - Использование: передается в компонент VacanciesStats

- `activeVacancies`: количество активных вакансий
  - Вычисление: `mockVacancies.filter(v => getStatus(v) === 'active').length`
  - Использование: передается в компонент VacanciesStats
  - Особенности: учитывает переопределения статусов из `statusOverrides`

- `inactiveVacancies`: количество неактивных вакансий
  - Вычисление: `mockVacancies.filter(v => getStatus(v) === 'inactive').length`
  - Использование: передается в компонент VacanciesStats
  - Особенности: учитывает переопределения статусов из `statusOverrides`

### filteredVacancies
- **Назначение**: Отфильтрованный список вакансий по всем критериям
- **Вычисление**: `mockVacancies.filter(vacancy => { ... })`
- **Логика фильтрации** (все три условия должны выполняться одновременно):
  1. **Поиск** (`matchesSearch`): 
     - Если `searchQuery` пустой - пропускает все вакансии
     - Иначе проверяет, содержит ли название вакансии поисковый запрос (без учета регистра)
     - Или проверяет, содержит ли ID вакансии поисковый запрос
  2. **Фильтр по рекрутеру** (`matchesRecruiter`): 
     - Если `selectedRecruiter === 'all'` - пропускает все вакансии
     - Иначе проверяет точное совпадение имени рекрутера
  3. **Фильтр по статусу** (`matchesStatus`): 
     - Если `selectedStatus === 'all'` - пропускает все вакансии
     - Иначе проверяет совпадение статуса (с учетом переопределений через `getStatus()`)
- **Использование**: 
  - Отображается в виде карточек (VacancyCard) или списка (VacancyListItem)
  - Обновляется в реальном времени при изменении фильтров
- **Особенности**: 
  - Использует `getStatus()` для получения актуального статуса с учетом переопределений
  - Все условия применяются через логическое И (&&)

## Управление статусами

### Переопределение статусов (statusOverrides)
- **Назначение**: Временное изменение статуса вакансии без сохранения на сервер
- **Структура**: `Record<number, 'active' | 'inactive'>`
  - Ключ: ID вакансии (number)
  - Значение: новый статус ('active' | 'inactive')
- **Логика**: 
  - При клике на статус в карточке/списке обновляется `statusOverrides`
  - `getStatus()` учитывает переопределения при отображении
  - Позволяет быстро переключать статусы без немедленного сохранения
- **Обновление**: 
  ```typescript
  setStatusOverrides(prev => ({ 
    ...prev, 
    [vacancy.id]: currentStatus === 'active' ? 'inactive' : 'active' 
  }))
  ```
- **Использование**: 
  - Отображение актуального статуса в UI
  - Фильтрация по статусу
  - Подсчет статистики

## Модальное окно редактирования

### Открытие модального окна
- **Режим просмотра** (`viewVacancyId`): 
  - Открывается при клике на вакансию (onClick)
  - Открывается при клике на кнопку "Просмотр" (EyeOpenIcon)
  - Режим: `mode='view'`
- **Режим редактирования** (`editVacancyId`): 
  - Открывается при клике на кнопку редактирования (onEditClick)
  - Открывается через URL параметр `?edit=123`
  - Режим: `mode='edit'`
- **Логика открытия**: 
  - При открытии в одном режиме закрывается другой режим
  - `setEditVacancyId(vacancy.id); setViewVacancyId(null)` - для редактирования
  - `setViewVacancyId(vacancy.id); setEditVacancyId(null)` - для просмотра

### Закрытие модального окна
- **Обработчик**: `onOpenChange={(open) => { if (!open) { setEditVacancyId(null); setViewVacancyId(null) } }}`
- **Логика**: 
  - При закрытии (open === false) очищаются оба состояния
  - Закрывает модальное окно в любом режиме

### Переключение между режимами
- **Из просмотра в редактирование**: 
  - Функция `onSwitchToEdit` передается только если открыт режим просмотра
  - При вызове: `setEditVacancyId(viewVacancyId); setViewVacancyId(null)`
- **Из редактирования в просмотр**: 
  - Не реализовано (TODO: добавить кнопку "Отменить" в режиме редактирования)

### Передача данных в модальное окно
- **vacancyId**: `editVacancyId ?? viewVacancyId` - ID вакансии для работы
- **vacancy**: объект вакансии с актуальным статусом (с учетом переопределений)
- **vacancyStatus**: актуальный статус вакансии (с учетом переопределений)
- **onVacancyStatusChange**: обновляет `statusOverrides` при изменении статуса в модальном окне
- **vacancyTitle**: название вакансии для отображения в заголовке модального окна
