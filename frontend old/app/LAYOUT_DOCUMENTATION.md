# Документация: Корневой Layout (layout.tsx)

## Общее описание

Корневой layout компонент приложения HR Helper. Оборачивает все страницы приложения и предоставляет глобальные провайдеры для темы и уведомлений.

## Компоненты layout

### 1. ThemeProvider
- **Расположение**: `@/components/ThemeProvider`
- **Назначение**: Управление темой приложения (светлая/темная)
- **Функциональность**: 
  - Предоставляет контекст темы через `useTheme()` хук
  - Сохраняет выбранную тему в localStorage
  - Применяет тему ко всем дочерним компонентам
  - Поддерживает переключение между светлой и темной темой
- **Связи**: 
  - Header компонент: использует для переключения темы
  - Все компоненты: могут использовать тему через `useTheme()`

### 2. ToastProvider
- **Расположение**: `@/components/Toast/ToastContext`
- **Назначение**: Управление системой уведомлений
- **Функциональность**: 
  - Предоставляет контекст для отображения toast-уведомлений
  - Используется через `useToast()` хук в компонентах
  - Показывает информационные, успешные, ошибочные и предупреждающие сообщения
  - Автоматическое скрытие уведомлений через заданное время
- **Связи**: 
  - Все компоненты: могут показывать уведомления через `useToast()`

## Метаданные

### Metadata
- **title**: 'HR Helper' - отображается во вкладке браузера и в результатах поиска
- **description**: 'HR Helper Application' - используется поисковыми системами для описания приложения

## Структура layout

```tsx
<html lang="ru">
  <body>
    <ThemeProvider>
      <ToastProvider>
        {children}
      </ToastProvider>
    </ThemeProvider>
  </body>
</html>
```

## Поведение

- **Применение**: Автоматически применяется ко всем страницам через Next.js App Router
- **Инициализация**: Провайдеры инициализируются один раз при загрузке приложения
- **Метаданные**: Используются для SEO и отображения в браузере

## Связи

- **Next.js App Router**: Используется для оборачивания всех страниц
- **ThemeProvider**: Связан с Header компонентом для переключения темы
- **ToastProvider**: Связан с компонентами, которые показывают уведомления через `useToast()`

## Глобальные стили (базовые для всех страниц)

Подключаются в `app/layout.tsx` — автоматически применяются ко всем страницам.

- **@radix-ui/themes/styles.css** — стили компонентов Radix UI.
- **./globals.css** — базовые стили приложения. Сюда добавлять новые общие стили и переменные.

В **globals.css** заданы:
- **Переменные**: `--page-padding` (24px), `--page-padding-mobile` (16px), `--content-max-width` (1200px), `--content-gap` (16px) — использовать в модулях для единообразия.
- **Класс** `.page-content-container` — контейнер контента (padding, max-width, margin auto) для обычных страниц; в модулях можно использовать его или свои классы с теми же переменными.
- Тема (светлая/тёмная), z-index (меню, модалки, тосты), reset для html/body.

Страницы настроек компании уже подтягивают переменные (например, `company-settings.module.css` использует `var(--page-padding)` и `var(--content-max-width)`).

## Особенности

- Единая точка входа для всех страниц
- Глобальные провайдеры для темы и уведомлений
- Метаданные для SEO
- Поддержка русского языка (lang="ru")
