# Документация модуля: Страницы ошибок (errors/)

## Общее описание

Модуль `errors/` содержит все страницы ошибок приложения, объединенные в единую структуру для удобства управления и поддержки. Все страницы ошибок следуют единому дизайну и функциональности, обеспечивая консистентный пользовательский опыт при возникновении различных типов ошибок.

## Структура модуля

```
errors/
├── 401/          # Страница ошибки 401 (Unauthorized - требуется авторизация)
├── 402/          # Страница ошибки 402 (Payment Required - требуется оплата)
├── 404/          # Страница ошибки 404 (Not Found - страница не найдена)
├── 500/          # Страница ошибки 500 (Internal Server Error - внутренняя ошибка сервера)
├── forbidden/    # Страница ошибки 403 (Forbidden - доступ запрещен)
└── PAGE_DOCUMENTATION.md  # Этот файл
```

## Общие характеристики

### Единый дизайн
Все страницы ошибок используют единый визуальный стиль:
- Анимированный фон с плавающими иконками (35-40 иконок в зависимости от страницы)
- Крупный код ошибки с градиентной заливкой
- Центрированный контентный блок с полупрозрачным фоном и размытием
- Адаптивный дизайн для мобильных устройств
- Поддержка темной темы

### Общие компоненты

#### FloatingIcon
Компонент плавающей иконки с анимацией, используемый на всех страницах ошибок:
- **Параметры анимации**: случайные для каждой иконки
- **Группы скоростей**: быстрая (8-12с), средняя (12-18с), медленная (18-25с)
- **Движение**: плавное плавание с вращением и масштабированием
- **CSS переменные**: управление через CSS для оптимизации производительности

#### Структура страницы
Каждая страница ошибки содержит:
1. **Фон с анимированными иконками** - создает визуально привлекательный фон
2. **Контентный блок** - центрированный блок с информацией об ошибке
3. **Код ошибки** - крупный текст с кодом ошибки (401, 402, 403, 404, 500)
4. **Заголовок** - название ошибки с соответствующей иконкой
5. **Описание** - текстовое описание ошибки для пользователя
6. **Кнопки навигации** - кнопки для возврата в приложение

## Страницы ошибок

### 401 - Требуется авторизация (`errors/401/`)

**Назначение**: Отображается при попытке доступа к защищенным ресурсам без авторизации.

**Особенности**:
- Иконки: замки, пользователи, запреты, информация
- Цветовая схема: синий (accent-9)
- Кнопки: "Войти" (переход на `/account/login`), "На главную" (переход на `/workflow`)

**Использование**:
- При истечении сессии
- При попытке доступа без авторизации
- При отсутствии валидного токена доступа

**Назначение**: Отображается при необходимости оплаты подписки или при истечении подписки.

**Использование**:
- При истечении подписки
- При попытке доступа к платному контенту без оплаты
- При необходимости обновления подписки

### 403 - Доступ запрещен (`errors/forbidden/`)

**Назначение**: Отображается при отсутствии прав доступа к запрашиваемому ресурсу.

**Особенности**:

- Кнопки: "На главную" (переход на `/workflow`), "Назад" (возврат в истории)

**Использование**:
- При отсутствии необходимых прав доступа
- При попытке доступа к ресурсам другого пользователя

### 404 - Страница не найдена (`errors/404/`)

**Назначение**: Отображается при обращении к несуществующему URL.

**Особенности**:
- Иконки: иконки из меню приложения (связь с основным приложением)
- Цветовая схема: синий (accent-9)
- Кнопки: "На главную" (переход на `/workflow`), "Назад" (возврат в истории)
- Анимация иконки поиска: вращение и покачивание

**Использование**:
- При переходе на несуществующий URL
- При удалении страницы
- При неправильном вводе адреса

**Примечание**: Файл `not-found.tsx` также находится в корне `app/` для автоматического использования Next.js при 404 ошибках.

## Технические детали

### CSS Модули
Каждая страница использует свой CSS модуль:
- `error-401.module.css` - стили для страницы 401
- `error-402.module.css` - стили для страницы 402
- `error-500.module.css` - стили для страницы 500
- `forbidden.module.css` - стили для страницы 403
- `not-found.module.css` - стили для страницы 404

### Анимации
Все страницы используют CSS анимации:
- `@keyframes float` - плавание иконок по экрану
- `@keyframes pulse` - пульсация кода ошибки
- `@keyframes bounce` - подпрыгивание иконки (401)
- `@keyframes flip` - переворот иконки (402)
- `@keyframes shake` - тряска иконки (403, 500)
- `@keyframes searchRotate` и `@keyframes searchWrapper` - анимация поиска (404)

### Адаптивность
Все страницы адаптированы для мобильных устройств:
- Уменьшенный размер кода ошибки (120px → 80px)
- Уменьшенный размер иконок (32px → 24px)
- Уменьшенные отступы контентного блока
- Оптимизированные размеры шрифтов

### Темная тема
Все страницы поддерживают темную тему через атрибут `[data-theme="dark"]`:
- Адаптированные цвета фона
- Улучшенная видимость иконок
- Оптимизированные тени и эффекты

## Связи с другими модулями

### AppLayout
Все страницы ошибок обернуты в `AppLayout` для единообразного отображения:
- Заголовок страницы в браузере
- Общий layout приложения
- Навигация и меню

### useRouter
Используется для программной навигации:
- `router.push('/workflow')` - переход на главную страницу
- `router.push('/account/login')` - переход на страницу входа (401)
- `router.back()` - возврат в истории браузера
- `window.location.reload()` - перезагрузка страницы (500)
