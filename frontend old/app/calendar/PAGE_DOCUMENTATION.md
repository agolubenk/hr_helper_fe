# Документация страницы: Календарь (calendar/page.tsx)

## Общее описание

Страница календаря событий для управления интервью, скринингами и встречами. Поддерживает синхронизацию с Google Calendar, просмотр событий в разных режимах (месяц, неделя, день, список), управление слотами и детальный просмотр событий.

**Путь**: `/calendar`

## Компоненты страницы

### 1. SlotsPanel
- **Расположение**: `@/components/workflow/SlotsPanel`
- **Назначение**: Панель свободных слотов для интервью
- **Пропсы**: Нет (использует внутреннее состояние)
- **Логика**: 
  - Отображает свободные слоты по неделям (текущая и следующая)
  - Раскрытие/сворачивание секций недель
  - Карточки слотов с датой, днем недели и временными интервалами
  - Если слотов нет - показывает "Нет свободных слотов"
- **Связи**: 
  - Синхронизируется с Google Calendar (TODO: через API)
  - Используется на вкладке "Слоты"

### 2. Календарь (месячный вид)
- **Компонент**: Кастомная сетка дней месяца
- **Назначение**: Отображение событий в календарном виде
- **Логика**: 
  - Сетка 7xN (7 дней недели, N недель в месяце)
  - Заголовки дней недели: Пн, Вт, Ср, Чт, Пт, Сб, Вс
  - Отображение дней месяца с пустыми ячейками для дней до начала месяца
  - Выделение текущего дня (border и фон)
  - Отображение событий в днях (до 3 событий, остальные показываются как "+N еще")
  - Бейджи с количеством событий
  - Цвета событий зависят от типа и статуса
- **Связи**: 
  - Использует данные из `mockEvents`
  - При клике на событие открывает модальное окно

### 3. Таблица событий (недельный/дневной вид)
- **Компонент**: `Table` из Radix UI
- **Назначение**: Отображение событий в табличном виде для недели или дня
- **Колонки**:
  - **Время**: время начала и окончания события (с иконкой ClockIcon)
  - **Событие**: тип события (Badge с цветом по типу)
  - **Кандидат**: имя кандидата (или "-" если нет)
  - **Интервьюер**: имя интервьюера (или "-" если нет)
  - **Формат**: онлайн (VideoIcon) или офис (BoxIcon)
  - **Вакансия**: название вакансии (или "-" если нет)
- **Логика**: 
  - Фильтрация событий по выбранному периоду (неделя или день)
  - При клике на строку открывает модальное окно с деталями события
- **Связи**: 
  - Использует данные из `mockEvents`
  - Отображается при `viewMode === 'week'` или `viewMode === 'day'`

### 4. Список событий (вкладка "Список событий")
- **Компонент**: `Table` из Radix UI с поиском
- **Назначение**: Отображение всех событий с поиском и фильтрацией
- **Колонки**:
  - **Название встречи**: название события со статусом (иконка CheckIcon/ExclamationTriangleIcon/Cross2Icon)
  - **Дата и время**: дата и время события (или "Весь день" для allDay событий)
  - **Google Meet**: кнопка с ссылкой на Google Meet (если есть) с количеством участников
  - **Описание**: описание события (обрезается если длинное)
  - **Ссылка**: кнопка копирования ссылки на событие в Google Calendar
- **Логика**: 
  - Поиск по названию, кандидату, вакансии, описанию
  - Фильтрация в реальном времени
  - При клике на строку открывает модальное окно
  - Пустое состояние: "События не найдены" или "События календаря не синхронизированы"
- **Связи**: 
  - Использует `filteredEvents` для отображения

### 5. Модальное окно детального просмотра события
- **Компонент**: `Dialog` из Radix UI
- **Назначение**: Просмотр детальной информации о событии
- **Секции**:
  - **Время**: время начала и окончания, длительность, флаг "Весь день"
  - **Место**: место проведения (если есть)
  - **Статус**: Badge с цветом (подтверждено/предварительно/отменено)
  - **Создатель**: имя и email создателя события
  - **Участники**: список участников с их статусами ответа на приглашение
  - **Google Meet**: кнопка "Присоединиться к встрече" (если есть ссылка)
  - **Описание**: описание события в отдельном блоке
- **Логика**: 
  - Открывается при клике на событие в календаре, таблице или списке
  - Закрывается кнопкой "Закрыть" или кликом вне модального окна
- **Связи**: 
  - Отображает данные из `selectedEvent`

## Интерфейсы и типы

### Attendee
```typescript
interface Attendee {
  email: string                                    // Email участника
  name?: string                                    // Имя участника (опционально)
  responseStatus: 'accepted' | 'declined' | 'tentative' | 'needsAction'  // Статус ответа
  organizer?: boolean                              // Флаг организатора
}
```

**Статусы ответа**:
- `accepted`: принял приглашение (зеленый Badge с CheckIcon)
- `declined`: отклонил приглашение (красный Badge с Cross2Icon)
- `tentative`: возможно придет (желтый Badge с ExclamationTriangleIcon)
- `needsAction`: не ответил (серый Badge с ClockIcon)

### CalendarEvent
```typescript
interface CalendarEvent {
  id: string                                      // Уникальный идентификатор события
  title: string                                   // Название события
  start: Date                                     // Дата и время начала
  end: Date                                       // Дата и время окончания
  type: 'interview' | 'screening' | 'meeting' | 'other'  // Тип события
  candidate?: string                              // Имя кандидата (опционально)
  interviewer?: string                            // Имя интервьюера (опционально)
  format?: 'online' | 'office'                   // Формат встречи (опционально)
  vacancy?: string                                // Название вакансии (опционально)
  status?: 'confirmed' | 'tentative' | 'cancelled'  // Статус события (опционально)
  location?: string                               // Место проведения (опционально)
  description?: string                            // Описание события (опционально)
  meetLink?: string                               // Ссылка на Google Meet (опционально)
  creatorEmail?: string                           // Email создателя (опционально)
  creatorName?: string                            // Имя создателя (опционально)
  attendees?: Attendee[]                          // Массив участников (опционально)
  allDay?: boolean                                // Флаг события на весь день (опционально)
}
```

**Типы событий**:
- `interview`: Интервью (цвет #8B5CF6 - фиолетовый)
- `screening`: HR-скрининг (цвет #A855F7 - фиолетовый)
- `meeting`: Встреча (цвет #3B82F6 - синий)
- `other`: Другое (цвет #6B7280 - серый)

**Статусы событий**:
- `confirmed`: Подтверждено (зеленый Badge)
- `tentative`: Предварительно (желтый Badge)
- `cancelled`: Отменено (красный Badge)

## Состояние компонента

### State переменные:
- `currentDate` (Date): текущая дата для отображения календаря
  - Инициализация: `new Date()` (сегодня)
  - Обновление: при навигации по месяцам/неделям/дням
  - Использование: определение, какой месяц/неделю/день отображать
- `viewMode` ('month' | 'week' | 'day'): режим отображения календаря
  - Инициализация: `'month'` (месячный вид)
  - Обновление: при выборе режима в Select
  - Использование: определение, какой вид календаря отображать
- `selectedOffice` ('minsk' | 'warsaw' | 'gomel'): выбранный офис для фильтрации
  - Инициализация: `'minsk'`
  - Обновление: при клике на кнопку офиса
  - Использование: фильтрация событий по офису (TODO: реализовать)
- `selectedEvent` (CalendarEvent | null): выбранное событие для детального просмотра
  - Инициализация: `null`
  - Обновление: при клике на событие
  - Использование: отображение в модальном окне
- `eventModalOpen` (boolean): флаг открытия модального окна события
  - Инициализация: `false`
  - Обновление: при открытии/закрытии модального окна
  - Использование: управление видимостью модального окна
- `isSyncing` (boolean): флаг синхронизации с Google Calendar
  - Инициализация: `false`
  - Обновление: при начале/завершении синхронизации
  - Использование: индикатор загрузки на кнопке синхронизации
- `activeTab` ('calendar' | 'list' | 'slots'): активная вкладка
  - Инициализация: `'calendar'`
  - Обновление: при переключении вкладок
  - Использование: определение, какой контент отображать
- `searchQuery` (string): поисковый запрос для фильтрации событий
  - Инициализация: `''`
  - Обновление: при вводе в поле поиска
  - Использование: фильтрация событий в списке

### Hooks:
- `useToast()`: хук для отображения уведомлений
  - Используется для: синхронизации, копирования ссылок, ошибок

## Функции и обработчики

### navigateMonth(direction)
- **Назначение**: Навигация по месяцам (предыдущий/следующий)
- **Параметры**: 
  - `direction`: 'prev' | 'next'
- **Логика**: 
  1. Создает новую дату на основе currentDate
  2. Увеличивает или уменьшает месяц
  3. Обновляет currentDate
- **Поведение**: 
  - Вызывается при клике на кнопки "←" и "→"
  - Обновляет отображаемый месяц в календаре
- **Связи**: 
  - Используется в кнопках навигации месяца

### goToToday()
- **Назначение**: Переход к текущей дате
- **Параметры**: Нет
- **Логика**: 
  - Устанавливает currentDate на текущую дату (new Date())
- **Поведение**: 
  - Вызывается при клике на кнопку "Сегодня"
  - Обновляет календарь на текущий месяц
- **Связи**: 
  - Используется в кнопке "Сегодня"

### handleEventClick(event)
- **Назначение**: Обработчик клика на событие
- **Параметры**: 
  - `event`: CalendarEvent - событие, на которое кликнули
- **Логика**: 
  1. Устанавливает selectedEvent в выбранное событие
  2. Открывает модальное окно (setEventModalOpen(true))
- **Поведение**: 
  - Вызывается при клике на событие в календаре, таблице или списке
  - Открывает модальное окно с деталями события
- **Связи**: 
  - Используется в календаре, таблице и списке событий

### handleSyncCalendar()
- **Назначение**: Синхронизация календаря с Google Calendar
- **Параметры**: Нет
- **Логика**: 
  1. Устанавливает isSyncing в true (показывает индикатор загрузки)
  2. Отправляет запрос на синхронизацию (TODO: реализовать)
  3. Обновляет события из Google Calendar
  4. Показывает toast об успехе или ошибке
  5. Устанавливает isSyncing в false
- **Поведение**: 
  - Вызывается при клике на кнопку "Синхронизировать"
  - Показывает индикатор загрузки (анимация вращения иконки)
  - В текущей реализации только имитирует синхронизацию (setTimeout)
- **Связи**: 
  - API: должен отправлять запрос на синхронизацию с Google Calendar
- **TODO**: Реализовать реальную синхронизацию через API

### getDuration(start, end)
- **Назначение**: Вычисление длительности события
- **Параметры**: 
  - `start`: Date - дата и время начала
  - `end`: Date - дата и время окончания
- **Возвращает**: строка с длительностью (например, "30 мин", "1,5ч", "2ч 15мин")
- **Логика**: 
  1. Вычисляет разницу в миллисекундах
  2. Конвертирует в минуты
  3. Форматирует в читаемый вид:
     - Меньше 60 минут: "N мин"
     - Ровно N часов: "Nч"
     - N часов 30 минут: "N,5ч"
     - Иначе: "Nч Mмин"
- **Поведение**: 
  - Используется в модальном окне события для отображения длительности
- **Связи**: 
  - Используется в модальном окне детального просмотра

### getAttendeeStatusBadge(status)
- **Назначение**: Получение данных для Badge статуса участника
- **Параметры**: 
  - `status`: Attendee['responseStatus'] - статус ответа участника
- **Возвращает**: объект с цветом, текстом и иконкой для Badge
- **Логика**: 
  - `accepted`: зеленый (#10b981), "Принял", CheckIcon
  - `declined`: красный (#ef4444), "Отклонил", Cross2Icon
  - `tentative`: желтый (#f59e0b), "Возможно", ExclamationTriangleIcon
  - `needsAction`: серый (#6b7280), "Не ответил", ClockIcon
- **Поведение**: 
  - Используется в модальном окне для отображения статуса участников
- **Связи**: 
  - Используется в модальном окне детального просмотра

### getDaysInMonth(date)
- **Назначение**: Получение массива дней месяца для календарной сетки
- **Параметры**: 
  - `date`: Date - дата для определения месяца
- **Возвращает**: массив (Date | null)[] - дни месяца с null для пустых ячеек
- **Логика**: 
  1. Определяет первый и последний день месяца
  2. Определяет день недели первого дня (с корректировкой для понедельника как первого дня)
  3. Добавляет null для дней до начала месяца
  4. Добавляет все дни месяца
  5. Возвращает массив для отображения в сетке
- **Поведение**: 
  - Используется для построения календарной сетки
  - Корректирует первый день недели (понедельник = 0, воскресенье = 6)
- **Связи**: 
  - Используется в месячном виде календаря

### getEventsForDate(date)
- **Назначение**: Получение событий для конкретной даты
- **Параметры**: 
  - `date`: Date | null - дата для поиска событий
- **Возвращает**: CalendarEvent[] - массив событий на эту дату
- **Логика**: 
  1. Если date === null, возвращает пустой массив
  2. Фильтрует mockEvents по дате начала события
  3. Сравнивает даты через toDateString()
- **Поведение**: 
  - Используется в календаре для отображения событий в днях
- **Связи**: 
  - Используется в месячном виде календаря

### getEventTypeColor(type)
- **Назначение**: Получение цвета для типа события
- **Параметры**: 
  - `type`: CalendarEvent['type'] - тип события
- **Возвращает**: string - hex цвет
- **Логика**: 
  - `interview`: #8B5CF6 (фиолетовый)
  - `screening`: #A855F7 (фиолетовый)
  - `meeting`: #3B82F6 (синий)
  - `other`: #6B7280 (серый)
- **Поведение**: 
  - Используется для окраски Badge событий в календаре и таблице
- **Связи**: 
  - Используется в календаре, таблице и списке событий

### getEventTypeLabel(type)
- **Назначение**: Получение текстовой метки для типа события
- **Параметры**: 
  - `type`: CalendarEvent['type'] - тип события
- **Возвращает**: string - текст метки
- **Логика**: 
  - `interview`: "Интервью"
  - `screening`: "HR-скрининг"
  - `meeting`: "Встреча"
  - `other`: "Событие"
- **Поведение**: 
  - Используется для отображения типа события в Badge
- **Связи**: 
  - Используется в календаре, таблице и списке событий

### formatTime(date)
- **Назначение**: Форматирование времени в формат HH:MM
- **Параметры**: 
  - `date`: Date - дата для форматирования
- **Возвращает**: string - время в формате "HH:MM"
- **Логика**: 
  - Использует toLocaleTimeString с опциями { hour: '2-digit', minute: '2-digit' }
- **Поведение**: 
  - Используется для отображения времени событий
- **Связи**: 
  - Используется во всех видах календаря и модальном окне

### formatDate(date)
- **Назначение**: Форматирование даты в читаемый формат
- **Параметры**: 
  - `date`: Date - дата для форматирования
- **Возвращает**: string - дата в формате "DD месяц YYYY" (например, "26 января 2026")
- **Логика**: 
  - Использует toLocaleDateString с опциями { day: 'numeric', month: 'long', year: 'numeric' }
- **Поведение**: 
  - Используется для отображения дат в заголовках и списке событий
- **Связи**: 
  - Используется в таблице событий и модальном окне

### copyEventLink(event)
- **Назначение**: Копирование ссылки на событие в Google Calendar
- **Параметры**: 
  - `event`: CalendarEvent - событие для копирования ссылки
- **Логика**: 
  1. Формирует ссылку на событие в Google Calendar
  2. Копирует ссылку в буфер обмена через navigator.clipboard
  3. Показывает toast об успехе или ошибке
- **Поведение**: 
  - Вызывается при клике на кнопку копирования ссылки
  - Показывает toast "Ссылка скопирована" при успехе
- **Связи**: 
  - Используется в списке событий и модальном окне

## Структура JSX

### AppLayout
- **Компонент**: `AppLayout`
- **Назначение**: Обертка страницы в общий layout приложения
- **Пропсы**: 
  - `pageTitle="Календарь"` - заголовок страницы в браузере

### Tabs (вкладки)
- **Компонент**: `Tabs.Root` из Radix UI
- **Вкладки**:
  1. **Календарь** (value="calendar"): месячный/недельный/дневной вид
  2. **Список событий** (value="list"): таблица всех событий с поиском
  3. **Слоты** (value="slots"): панель свободных слотов

### Вкладка "Календарь"
- **Заголовок с навигацией**:
  - Кнопки навигации: "←" (предыдущий месяц), "→" (следующий месяц)
  - Текст с названием месяца и года
  - Кнопка "Сегодня"
  - Кнопка "Синхронизировать" (с индикатором загрузки)
  - Тогглер выбора офиса (Минск, Варшава, Гомель)
  - Select для выбора режима отображения (Месяц, Неделя, День)
- **Месячный вид** (viewMode === 'month'):
  - Сетка дней месяца (7 колонок)
  - Заголовки дней недели
  - Ячейки дней с событиями
  - Выделение текущего дня
- **Недельный/дневной вид** (viewMode === 'week' || viewMode === 'day'):
  - Таблица событий с колонками: Время, Событие, Кандидат, Интервьюер, Формат, Вакансия

### Вкладка "Список событий"
- **Поиск**:
  - TextField с иконкой поиска
  - Кнопка "Очистить" (показывается только если есть поисковый запрос)
