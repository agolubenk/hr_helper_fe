# Документация страницы: Workflow (workflow/page.tsx)

## Общее описание

Основная рабочая страница для работы рекрутера с кандидатами. Центральный хаб для общения с кандидатами, внесения данных, назначения интервью и управления процессом найма. Страница объединяет все инструменты рекрутера в едином интерфейсе для эффективной работы с кандидатами.

**Путь**: `/workflow`

## Компоненты страницы

### 1. WorkflowPage (основной компонент)
- **Расположение**: `@/app/workflow/page.tsx`
- **Назначение**: Главный компонент страницы workflow
- **Функциональность**: 
  - Управление состоянием открытия/закрытия панели слотов
  - Установка специального padding для контента AppLayout
  - Организация layout страницы (заголовок, панель слотов, две колонки)
- **Связи**: 
  - AppLayout: оборачивает страницу в общий layout
  - Все дочерние компоненты workflow

### 2. WorkflowHeader
- **Расположение**: `@/components/workflow/WorkflowHeader`
- **Назначение**: Заголовок страницы с управлением вакансией и настройками
- **Пропсы**:
  - `onSlotsClick`: обработчик клика на кнопку "слоты" (() => void)
  - `slotsOpen`: флаг открытости панели слотов (boolean)
- **Функциональность**: 
  - Быстрые кнопки для коммуникации: Telegram, WhatsApp, Viber, LinkedIn, Email
  - Тогглер типа процесса: динамические кнопки на основе этапов найма с меткой "встреча"
    - Кнопка "Скрининг" - всегда доступна (30 мин)
    - Динамические кнопки этапов-встреч - формируются из этапов найма с `isMeeting = true`
    - Количество кнопок: 0 и более (зависит от количества этапов с `isMeeting = true`)
    - Названия кнопок: берутся из названий этапов найма, отмеченных чекбоксом "встреча"
    - Если этапов-встреч нет, тогглер может быть пустым или содержать только "Скрининг"
  - Выбор вакансии: выпадающий список с вакансиями
  - Кнопки управления: Календарь, Вакансия, Свободные слоты, Обновить
  - Настройки встречи (для этапов с `isMeeting = true`):
    - Формат встречи: Онлайн / Офис (отображается если `showOffices = true`)
    - Выбор интервьюеров: чекбоксы для выбора нескольких интервьюеров (отображается если `showInterviewers = true`)
    - Выбор офиса: появляется после выбора формата "Офис", если `showOffices = true`
- **Связи**: 
  - Управляет открытием/закрытием SlotsPanel через onSlotsClick
  - Передает настройки в WorkflowChat (неявно через общее состояние)
  - Google Calendar: переход к календарю
  - Huntflow: переход к вакансии
  - /company-settings/recruiting/stages: настройки этапов найма с метками "встреча"
- **Настройки этапов-встреч**:
  - `isMeeting`: метка "встреча" для этапа
  - `showOffices`: отображать ли выбор офисов (да/нет)
  - `showInterviewers`: отображать ли выбор интервьюеров (да/нет)
  - Эти настройки задаются на странице `/company-settings/recruiting/stages` в модальном окне редактирования этапа
- **Особенности**: 
  - Интерфейс меняется в зависимости от выбранного типа процесса
  - Панель настроек встречи показывается только для этапов-встреч (не для "Скрининг")
  - Содержимое панели настроек встречи зависит от настроек выбранного этапа (`showOffices`, `showInterviewers`)

### 3. WorkflowChat
- **Расположение**: `@/components/workflow/WorkflowChat`
- **Назначение**: Чат-интерфейс для работы с кандидатами
- **Пропсы**: Нет (использует внутреннее состояние)
- **Функциональность**: 
  - Отображение истории сообщений (пользователь, система, инвайты)
  - Обработка команд через теги:
    - `#add`: добавление кандидата из файла резюме
    - `#hr_screening`: HR скрининг (ссылка Huntflow + текст ответов)
    - `#tech_screening`: технический скрининг (ссылка Huntflow + дата-время)
    - `#interview`: создание интервью (ссылка Huntflow + дата-время)
    - `#delete`: отмена последнего действия
  - Управление файлами: загрузка резюме
  - Управление инвайтами: создание, копирование, пересоздание scorecard
  - Управление отклонениями: выбор причины отказа
  - Скрытие сообщений
  - Автопрокрутка к новым сообщениям
- **Связи**: 
  - API: отправка команд на сервер для обработки
  - Huntflow: интеграция для синхронизации данных
  - Google Calendar: создание событий интервью
  - Google Meet: создание встреч для онлайн интервью
  - Google Drive: создание и хранение scorecard файлов
- **Особенности**: 
  - Очень сложный компонент с множеством команд и состояний
  - Обработка команд происходит через теги в сообщениях
  - Поддержка различных типов сообщений (текст, файлы, инвайты)

### 4. WorkflowSidebar
- **Расположение**: `@/components/workflow/WorkflowSidebar`
- **Назначение**: Боковая панель с информацией и быстрыми действиями
- **Пропсы**: Нет (использует внутреннее состояние)
- **Функциональность**: 
  - Кнопка "Вики": открытие вики в новой вкладке
  - Отчеты последних недель:
    - Текущая неделя: статистика по этапам (раскрывающаяся секция)
    - Предыдущая неделя: статистика по этапам (раскрывающаяся секция)
  - Быстрые действия для коммуникации:
    - Telegram: несколько аккаунтов
    - WhatsApp: несколько номеров
    - Viber: несколько номеров
    - LinkedIn: несколько профилей
    - Email: несколько адресов
- **Связи**: 
  - `/wiki`: переход к вики при клике на кнопку
  - Внешние сервисы: открытие в новой вкладке при клике на карточки
- **Особенности**: 
  - Раскрывающиеся секции отчетов (аккордеон)
  - При открытии одной недели автоматически закрывает другую
  - Быстрые действия оформлены как карточки с иконками

### 5. SlotsPanel
- **Расположение**: `@/components/workflow/SlotsPanel`
- **Назначение**: Панель свободных слотов для интервью
- **Пропсы**: Нет (использует внутреннее состояние)
- **Функциональность**: 
  - Текущая неделя: отображение свободных слотов
  - Следующая неделя: отображение свободных слотов
  - Раскрытие/сворачивание секций недель
  - Карточки слотов: дата, день недели, список временных слотов, количество слотов
- **Связи**: 
  - Google Calendar: синхронизация слотов с календарем интервьюеров
  - WorkflowHeader: управление открытием/закрытием через slotsOpen
- **Особенности**: 
  - Условно рендерится при slotsOpen === true
  - Анимация появления (slideDown)
  - Отображает свободные слоты для быстрого копирования

## Интерфейсы и типы

### WorkflowHeaderProps
```typescript
interface WorkflowHeaderProps {
  onSlotsClick: () => void    // Обработчик клика на кнопку "слоты"
  slotsOpen: boolean           // Флаг открытости панели слотов
}
```

### WorkflowType
```typescript
type WorkflowType = 'screening' | 'interview'
// 'screening': скрининг (30 минут)
// 'interview': интервью (90 минут)
```

### InterviewFormat
```typescript
type InterviewFormat = 'online' | 'office'
// 'online': онлайн интервью
// 'office': офисное интервью
```

### Office
```typescript
type Office = 'minsk' | 'warsaw' | 'gomel'
// 'minsk': Минск
// 'warsaw': Варшава
// 'gomel': Гомель
```

### WeekSlot (из SlotsPanel)
```typescript
interface WeekSlot {
  date: string              // Дата слота (DD.MM.YYYY)
  dayOfWeek: string         // День недели (сокращенный, например "Пн", "Вт")
  slots: string[]          // Массив временных интервалов (например, ['12.00-12.45', '15.45-18.00'])
  count: number            // Количество свободных слотов
}
```

### WeekSection (из SlotsPanel)
```typescript
interface WeekSection {
  title: string            // Название секции (например, "Текущая неделя")
  slots: WeekSlot[]        // Массив слотов на неделю
}
```

## Состояние компонента

### State переменные:
- `slotsOpen` (boolean): флаг открытия/закрытия панели слотов
  - Инициализация: false (закрыта)
  - Обновление: при клике на кнопку "слоты" в WorkflowHeader
  - Использование: условный рендеринг SlotsPanel
  - Передается в WorkflowHeader для отображения активного состояния кнопки

## Функции и обработчики

### useEffect для padding контента
- **Назначение**: Устанавливает специальный верхний padding для контента AppLayout
- **Зависимости**: [] (пустой массив - выполняется только при монтировании)
- **Логика**: 
  1. Находит элемент контента AppLayout через CSS селектор: `.rt-Box.AppLayout_content__XSUzC`
  2. Сохраняет оригинальное значение padding-top для восстановления
  3. Устанавливает padding-top: 8px для этой страницы
  4. Возвращает функцию очистки, которая восстанавливает оригинальный padding при размонтировании
- **Используется для**: 
  - Специального отступа для более компактного размещения элементов
  - Workflow страница требует меньший верхний отступ
- **Особенности**: 
  - Выполняется только при монтировании компонента
  - Восстанавливает оригинальный padding при размонтировании
  - Используется только на этой странице

### onSlotsClick
- **Назначение**: Обработчик клика на кнопку "слоты" в WorkflowHeader
- **Логика**: 
  - Инвертирует состояние slotsOpen: `setSlotsOpen(!slotsOpen)`
  - Открывает/закрывает панель слотов
- **Используется для**: 
  - Переключения видимости SlotsPanel
  - Передается в WorkflowHeader через пропс onSlotsClick

## Команды чата (WorkflowChat)

### #add
- **Назначение**: Добавление кандидата из файла резюме
- **Использование**: 
  - Прикрепить файл резюме (PDF, DOC, DOCX)
  - Отправить сообщение с тегом `#add`
- **Результат**: 
  - Система обрабатывает резюме (парсинг данных)
  - Добавляет кандидата в Huntflow
  - Показывает ответ системы с данными кандидата
- **Формат**: `#add` + прикрепленный файл

### #hr_screening
- **Назначение**: HR скрининг кандидата
- **Использование**: 
  - Отправить ссылку на кандидата в Huntflow
  - Добавить текст ответов на вопросы скрининга
- **Результат**: 
  - Система извлекает данные из ссылки Huntflow
  - Сохраняет ответы на вопросы
  - Передает данные в Huntflow
  - Показывает ответ системы с результатами
- **Формат**: `#hr_screening` + ссылка Huntflow + текст ответов

### #tech_screening
- **Назначение**: Технический скрининг
- **Использование**: 
  - Отправить ссылку на кандидата в Huntflow
  - Добавить дата-время интервью (формат: DD.MM.YYYY HH:MM)
- **Результат**: 
  - Система создает инвайт на технический скрининг
  - Добавляет событие в Google Calendar
  - Создает Google Meet ссылку (если онлайн)
  - Показывает ответ системы с деталями инвайта
- **Формат**: `#tech_screening` + ссылка Huntflow + дата-время

### #interview
- **Назначение**: Создание интервью
- **Использование**: 
  - Отправить ссылку на кандидата в Huntflow
  - Добавить дата-время интервью (формат: DD.MM.YYYY HH:MM)
- **Результат**: 
  - Система создает инвайт на интервью
  - Добавляет событие в Google Calendar
  - Создает Google Meet ссылку (если онлайн)
  - Создает scorecard в Google Drive
  - Показывает ответ системы с деталями инвайта и ссылкой на scorecard
- **Формат**: `#interview` + ссылка Huntflow + дата-время

### #delete
- **Назначение**: Отмена последнего действия
- **Использование**: 
  - Отправить сообщение с тегом `#delete`
- **Результат**: 
  - Система отменяет последнее действие
  - Удаляет кандидата, запись или событие
  - Показывает ответ системы с подтверждением отмены
- **Формат**: `#delete`

## Интеграции

### Huntflow
- **Синхронизация данных**: 
  - Кандидаты: добавление, обновление, получение данных
  - Вакансии: получение списка вакансий
  - Статусы: обновление статусов кандидатов
- **Ссылки**: 
  - Формат: `https://huntflow.ru/my/org#/vacancy/[id]/filter/[id]/id/[id]`
  - Используется для извлечения данных кандидата
- **Использование**: 
  - В командах `#hr_screening`, `#tech_screening`, `#interview`
  - Для получения данных о кандидате из ссылки

### Google Calendar
- **Создание событий**: 
  - При создании интервью через команды `#tech_screening` или `#interview`
  - Событие создается с датой-временем, участниками, описанием
- **Синхронизация слотов**: 
  - `SlotsPanel` получает свободные слоты из календарей интервьюеров
  - Отображает доступное время для планирования интервью
- **Использование**: 
  - Для планирования интервью
  - Для отображения свободных слотов

### Google Meet
- **Создание встреч**: 
  - Для онлайн интервью автоматически создается Google Meet ссылка
  - Ссылка добавляется в событие календаря
- **Использование**: 
  - В командах `#tech_screening` и `#interview` при формате "Онлайн"

### Google Drive
- **Scorecard**: 
  - Создание и хранение scorecard файлов
  - Создается при команде `#interview`
  - Ссылка на scorecard отображается в ответе системы
- **Использование**: 
  - Для хранения оценочных листов интервью

  - На экранах меньше 768px:
