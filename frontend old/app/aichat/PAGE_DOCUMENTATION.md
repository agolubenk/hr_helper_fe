# Документация страницы: ИИ Чат (aichat/page.tsx)

## Общее описание

Страница ИИ чата для помощи в HR задачах. Позволяет общаться с ИИ ассистентом, управлять историей чатов, отправлять сообщения и прикреплять файлы. Поддерживает форматирование текста в сообщениях от ИИ (markdown-подобный синтаксис).

**Путь**: `/aichat`

## Компоненты страницы

### 1. ChatHistory
- **Расположение**: `@/components/aichat/ChatHistory`
- **Назначение**: Боковая панель с историей чатов
- **Пропсы**:
  - `chats`: массив всех чатов пользователя (Chat[])
  - `selectedChatId`: ID выбранного чата (string)
  - `onChatSelect`: обработчик выбора чата (функция `(chatId: string) => void`)
  - `onNewChat`: обработчик создания нового чата (функция `() => void`)
  - `onChatDelete`: обработчик удаления чата из истории (функция `(chatId: string) => void`)
- **Логика**: 
  - Отображает список всех чатов пользователя
  - Показывает название и дату создания каждого чата
  - Подсвечивает выбранный чат
  - При наведении на чат показывает кнопку удаления
  - Кнопка "Новый чат" внизу списка
- **Связи**: 
  - Управляет переключением между чатами через `onChatSelect`
  - Создает новые чаты через `onNewChat`
  - Удаляет чаты через `onChatDelete`

### 2. ChatHeader
- **Расположение**: `@/components/aichat/ChatHeader`
- **Назначение**: Заголовок чата с управлением (переименование и удаление)
- **Пропсы**:
  - `title`: название чата (string)
  - `createdAt`: дата создания чата (string, опционально)
  - `onTitleChange`: обработчик изменения названия чата (функция `(newTitle: string) => void`)
  - `onDelete`: обработчик удаления чата (функция `() => void`)
- **Логика**: 
  - Отображает название чата (кликабельное для редактирования)
  - При наведении показывает иконку редактирования (карандаш)
  - При клике на название открывает режим редактирования (TextField)
  - В режиме редактирования: кнопки "Сохранить" (галочка) и "Отменить" (крестик)
  - Поддержка клавиатуры: Enter - сохранить, Escape - отменить
  - При наведении показывает кнопку "Удалить чат" справа
- **Связи**: 
  - Управляет названием текущего чата через `onTitleChange`
  - Удаляет текущий чат через `onDelete`

### 3. ChatMessages
- **Расположение**: `@/components/aichat/ChatMessages`
- **Назначение**: Отображение сообщений чата
- **Пропсы**:
  - `messages`: массив сообщений для отображения (Message[])
- **Логика**: 
  - Отображает сообщения пользователя и ИИ ассистента
  - Разные стили для сообщений пользователя (справа) и ИИ (слева)
  - Для сообщений ИИ использует компонент `FormattedText` для форматирования
  - Для сообщений пользователя отображает обычный текст
  - Показывает аватар для каждого сообщения (G для ИИ, PersonIcon для пользователя)
  - Отображает прикрепленные файлы (один файл - текст, несколько - tooltip со списком)
  - Показывает дату и время каждого сообщения
  - Пустое состояние: "Начните новый диалог, отправив сообщение"
- **Связи**: 
  - Использует `FormattedText` для форматирования сообщений от ИИ
  - Отображает сообщения из `messages[currentChatId]`

### 4. ChatInput
- **Расположение**: `@/components/aichat/ChatInput`
- **Назначение**: Поле ввода с отправкой сообщений и прикреплением файлов
- **Пропсы**:
  - `onSend`: обработчик отправки сообщения (функция `(message: string, modelId?: string, files?: File[]) => void`)
  - `onFileAttach`: обработчик прикрепления файла (функция `(file: File) => void`, опционально)
- **Логика**: 
  - Обертка над компонентом `AnimatedAIInput`
  - Передает все пропсы в `AnimatedAIInput`
- **Связи**: 
  - Использует `AnimatedAIInput` для ввода и отправки сообщений

### 5. AnimatedAIInput
- **Расположение**: `@/components/aichat/AnimatedAIInput`
- **Назначение**: Поле ввода с выбором модели ИИ и прикреплением файлов
- **Пропсы**:
  - `onSend`: обработчик отправки сообщения
  - `onFileAttach`: обработчик прикрепления файла (опционально)
  - `placeholder`: текст placeholder (по умолчанию "Введите ваше сообщение...")
  - `disabled`: флаг блокировки поля ввода
- **Логика**: 
  - TextArea с автоматическим изменением высоты (до 200px)
  - Выбор модели ИИ через Popover (o3-mini, Gemini 2.5 Flash, Claude 3.5 Sonnet, GPT-4-1 Mini, GPT-4-1)
  - Прикрепление файлов (до 10 файлов одновременно)
  - Отображение списка прикрепленных файлов с возможностью удаления
  - Кнопка отправки (активна только если есть текст или файлы)
  - Поддержка Enter для отправки, Shift+Enter для новой строки
- **Связи**: 
  - Используется в `ChatInput` компоненте

### 6. FormattedText
- **Расположение**: `@/components/aichat/FormattedText`
- **Назначение**: Компонент для отображения форматированного текста (markdown-подобный синтаксис)
- **Пропсы**:
  - `content`: текст для форматирования (string)
  - `disableHeadings`: отключить заголовки (boolean, опционально, по умолчанию false)
- **Поддерживаемые форматы**:
  - **Жирный текст**: `**текст**` или `__текст__`
  - **Курсив**: `*текст*` или `_текст_`
  - **Жирный и курсив**: `***текст***` или `___текст___`
  - **Подчеркнутый**: `<u>текст</u>`
  - **Зачеркнутый**: `~~текст~~`
  - **Inline код**: `` `код` ``
  - **Блок кода**: ` ```код``` ` (с кнопкой копирования)
  - **Заголовки**: `# Заголовок 1` до `###### Заголовок 6`
  - **Цитаты**: `> Текст цитаты`
  - **Ссылки**: `[текст](url)`
- **Логика**: 
  - Парсит текст построчно
  - Обрабатывает блоки кода (приоритет выше всего)
  - Обрабатывает заголовки и цитаты
  - Обрабатывает inline форматирование (рекурсивно для вложенного форматирования)
  - Кнопка копирования для блоков кода (показывает "Скопировано" на 2 секунды)
  - Поддержка превью ссылок (отображается под сообщением)
- **Связи**: 
  - Используется в `ChatMessages` для форматирования сообщений от ИИ

## Интерфейсы и типы

### Chat
```typescript
interface Chat {
  id: string                    // Уникальный идентификатор чата
  title: string                 // Название чата (можно редактировать)
  createdAt: string             // Дата создания в формате DD.MM.YYYY HH:MM
  lastMessage?: string          // Последнее сообщение (опционально, для отображения в истории)
}
```

### Message
```typescript
interface Message {
  id: string                    // Уникальный идентификатор сообщения
  role: 'user' | 'assistant'   // Роль отправителя
  content: string               // Текст сообщения
  timestamp: string             // Время отправки в формате DD.MM.YYYY HH:MM
  files?: File[]                // Прикрепленные файлы (опционально)
}
```

### AIModel
```typescript
interface AIModel {
  id: string                    // ID модели (например, 'o3-mini', 'gemini-2.5-flash')
  name: string                  // Название модели для отображения
  icon: React.ReactNode         // Иконка модели
  provider: 'openai' | 'gemini' | 'claude'  // Провайдер ИИ
}
```

## Состояние компонента

### State переменные:
- `chats` (Chat[]): массив всех чатов пользователя
  - Инициализация: моковые данные с 3 чатами
  - Обновление: при создании, удалении, переименовании чата
  - Использование: отображение в ChatHistory, выбор чата
- `selectedChatId` (string): ID выбранного чата
  - Инициализация: `'1'` (первый чат)
  - Обновление: при выборе чата из истории или создании нового
  - Использование: определение, какой чат отображается
- `messages` (Record<string, Message[]>): объект сообщений по чатам
  - Структура: ключ - ID чата, значение - массив сообщений
  - Инициализация: моковые данные с сообщениями для первого чата
  - Обновление: при отправке нового сообщения
  - Использование: отображение сообщений в ChatMessages
- `isAtBottom` (boolean): флаг нахождения внизу списка сообщений
  - Инициализация: `true`
  - Обновление: при скролле контейнера сообщений
  - Использование: определение, показывать ли кнопку прокрутки вниз

### Refs:
- `messagesScrollRef` (RefObject<HTMLDivElement>): ссылка на контейнер сообщений
  - Использование: прокрутка к последнему сообщению, определение позиции скролла

### Hooks:
- `useToast()`: хук для отображения уведомлений
  - Используется для подтверждения удаления чата

## Функции и обработчики

### handleNewChat()
- **Назначение**: Обработчик создания нового чата
- **Параметры**: Нет
- **Логика**: 
  1. Создает новый чат с автоматически сгенерированным названием
  2. Название: "Чат DD.MM.YYYY HH:MM" (текущая дата и время)
  3. ID: timestamp (Date.now().toString())
  4. Добавляет новый чат в начало списка чатов
  5. Автоматически выбирает новый чат (setSelectedChatId)
  6. Создает пустой массив сообщений для нового чата
- **Поведение**: 
  - Вызывается из ChatHistory при клике на кнопку "Новый чат"
  - Новый чат сразу становится активным
- **Связи**: 
  - ChatHistory: вызывает через `onNewChat` prop

### handleSendMessage(content, modelId?, files?)
- **Назначение**: Обработчик отправки сообщения
- **Параметры**: 
  - `content`: текст сообщения (string)
  - `modelId`: ID выбранной модели ИИ (string, опционально)
  - `files`: массив прикрепленных файлов (File[], опционально)
- **Логика**: 
  1. Создает новое сообщение пользователя с текущим timestamp
  2. Добавляет сообщение в текущий чат (оптимистичное обновление)
  3. Подготавливает данные для отправки на сервер
  4. В реальной реализации должен отправлять запрос на сервер и получать ответ от ИИ
- **Поведение**: 
  - Вызывается из ChatInput/AnimatedAIInput при отправке сообщения
  - Сразу добавляет сообщение пользователя в чат
  - TODO: Отправка на сервер и получение ответа от ИИ
- **Связи**: 
  - ChatInput: вызывает через `onSend` prop
  - API: должен отправлять сообщение на сервер и получать ответ от ИИ

### handleFileAttach(file)
- **Назначение**: Обработчик прикрепления файла
- **Параметры**: 
  - `file`: прикрепленный файл (File)
- **Логика**: 
  - В текущей реализации только логирует информацию о файле
  - Файл не отправляется сразу, а прикрепляется к следующему сообщению
- **Поведение**: 
  - Вызывается из ChatInput/AnimatedAIInput при выборе файла
  - Файл включается в handleSendMessage при отправке сообщения
- **Связи**: 
  - ChatInput: вызывает через `onFileAttach` prop
- **TODO**: Реализовать предпросмотр прикрепленных файлов

### handleTitleChange(newTitle)
- **Назначение**: Обработчик изменения названия чата
- **Параметры**: 
  - `newTitle`: новое название чата (string)
- **Логика**: 
  1. Обновляет название выбранного чата в массиве chats
  2. Остальные чаты остаются без изменений
- **Поведение**: 
  - Вызывается из ChatHeader при сохранении нового названия
  - Обновляет только выбранный чат
- **Связи**: 
  - ChatHeader: вызывает через `onTitleChange` prop

### performDeleteChat(chatId)
- **Назначение**: Выполнение удаления чата
- **Параметры**: 
  - `chatId`: ID чата для удаления (string)
- **Логика**: 
  1. Фильтрует чаты, удаляя чат с указанным ID
  2. Если после удаления не осталось чатов - создает новый пустой чат
  3. Если удаляется выбранный чат - выбирает первый оставшийся чат
  4. Удаляет сообщения удаленного чата из состояния messages
- **Поведение**: 
  - Вызывается после подтверждения удаления в showDeleteConfirm
  - Гарантирует, что всегда есть хотя бы один чат
- **Связи**: 
  - showDeleteConfirm: вызывает после подтверждения

### showDeleteConfirm(chatId)
- **Назначение**: Показ подтверждения удаления чата
- **Параметры**: 
  - `chatId`: ID чата для удаления (string)
- **Логика**: 
  1. Показывает предупреждающее toast-уведомление
  2. Предоставляет кнопки "Отмена" и "Удалить"
  3. При клике на "Удалить" вызывает performDeleteChat
- **Поведение**: 
  - Вызывается перед удалением чата
  - Toast показывается 12 секунд
  - При клике на "Удалить" удаляет чат
  - При клике на "Отмена" закрывает уведомление
- **Связи**: 
  - useToast: использует toast для отображения уведомления
  - performDeleteChat: вызывается при подтверждении удаления

### handleDeleteChat()
- **Назначение**: Обработчик удаления текущего чата
- **Параметры**: Нет
- **Логика**: 
  - Показывает подтверждение удаления для выбранного чата
  - Использует selectedChatId для определения чата
- **Поведение**: 
  - Вызывается из ChatHeader при клике на кнопку удаления
- **Связи**: 
  - ChatHeader: вызывает через `onDelete` prop

### handleDeleteChatFromHistory(chatId)
- **Назначение**: Обработчик удаления чата из истории
- **Параметры**: 
  - `chatId`: ID чата для удаления (string)
- **Логика**: 
  - Показывает подтверждение удаления для чата из истории
  - Может удалить любой чат, не только выбранный
- **Поведение**: 
  - Вызывается из ChatHistory при удалении чата из списка
- **Связи**: 
  - ChatHistory: вызывает через `onChatDelete` prop

### scrollToBottom()
- **Назначение**: Прокрутка контейнера сообщений вниз
- **Параметры**: Нет
- **Логика**: 
  1. Прокручивает контейнер сообщений до самого низа (scrollTop = scrollHeight)
  2. Устанавливает флаг isAtBottom в true
- **Поведение**: 
  - Вызывается при клике на кнопку "К последнему сообщению"
  - Используется для автоматической прокрутки при новых сообщениях
- **Связи**: 
  - messagesScrollRef: использует для прокрутки
  - isAtBottom: обновляет флаг

### handleMessagesScroll()
- **Назначение**: Обработчик скролла контейнера сообщений
- **Параметры**: Нет
- **Логика**: 
  1. Определяет, находится ли пользователь внизу списка сообщений
  2. Вычисляет расстояние от низа контейнера до текущей позиции скролла
  3. Если расстояние меньше threshold (80px) - считает, что пользователь внизу
  4. Обновляет флаг isAtBottom
- **Поведение**: 
  - Вызывается при каждом скролле контейнера сообщений
  - Используется для показа/скрытия кнопки "К последнему сообщению"
- **Связи**: 
  - messagesScrollRef: использует для определения позиции скролла
  - isAtBottom: обновляет флаг

## useEffect хуки

### useEffect - автоматическая прокрутка к последнему сообщению
- **Назначение**: Автоматическая прокрутка к последнему сообщению при изменении сообщений или чата
- **Зависимости**: `[currentMessages, selectedChatId]`
- **Логика**: 
  1. Прокручивает контейнер сообщений вниз сразу
  2. Прокручивает еще раз через requestAnimationFrame (для надежности после рендера)
  3. Устанавливает флаг isAtBottom в true
- **Поведение**: 
  - Выполняется при изменении currentMessages (новые сообщения)
  - Выполняется при изменении selectedChatId (переключение чата)
  - Двойная прокрутка гарантирует прокрутку после полного рендера DOM
- **Причина двойной прокрутки**: 
  - Первая прокрутка выполняется сразу
  - Вторая через requestAnimationFrame гарантирует прокрутку после полного рендера

    - **Box** (`styles.inputContainer`): контейнер поля ввода
      - **ChatInput**: компонент ввода сообщения
        - `onSend={handleSendMessage}` - обработчик отправки
        - `onFileAttach={handleFileAttach}` - обработчик прикрепления файла

## Форматирование текста

### Поддерживаемые форматы в FormattedText

#### Inline форматирование:
- **Жирный**: `**текст**` или `__текст__` → `<strong>текст</strong>`
- **Курсив**: `*текст*` или `_текст_` → `<em>текст</em>`
- **Жирный и курсив**: `***текст***` или `___текст___` → `<strong><em>текст</em></strong>`
- **Подчеркнутый**: `<u>текст</u>` → `<u>текст</u>`
- **Зачеркнутый**: `~~текст~~` → `<del>текст</del>`
- **Inline код**: `` `код` `` → `<code>код</code>`
- **Ссылки**: `[текст](url)` → `<a href="url">текст</a>`

#### Блочные элементы:
- **Заголовки**: `# Заголовок 1` до `###### Заголовок 6` → `<h1>` до `<h6>`
- **Блок кода**: ` ```код``` ` → блок с кнопкой копирования
- **Цитаты**: `> Текст цитаты` → блок цитаты с полупрозрачным фоном

#### Особенности:
- Поддержка вложенного форматирования (например, **жирный** с *курсивом*)
- Комбинации форматов (например, **жирный** с <u>подчеркиванием</u>)
- Кнопка копирования для блоков кода (показывает "Скопировано" на 2 секунды)
- Превью ссылок (отображается под сообщением с ссылкой)

## Прикрепление файлов

### Ограничения:
- Максимум 10 файлов одновременно
- Поддержка множественного выбора
- Отображение списка прикрепленных файлов с возможностью удаления

### Отображение:
- Один файл: показывает имя и размер файла
- Несколько файлов: показывает "Отправлено N файлов" с tooltip со списком всех файлов
- Формат размера: B, KB, MB

### Удаление:
- Кнопка удаления (крестик) для каждого файла
- Удаление из списка перед отправкой сообщения

## Стили

- **Файл**: `aichat.module.css`
- **Основные классы**:
  - `.aichatContainer` - основной контейнер страницы
  - `.mainContent` - контейнер для двух колонок
  - `.sidebarColumn` - левая колонка с историей чатов (300px)
  - `.chatColumn` - правая колонка с чатом (flex: 1)
  - `.chatCard` - карточка чата
  - `.messagesWrapper` - обертка для сообщений и кнопки прокрутки
  - `.messagesContainer` - контейнер сообщений (с прокруткой)
  - `.scrollToBottomBtn` - кнопка прокрутки вниз (абсолютное позиционирование)
  - `.inputContainer` - контейнер поля ввода
- **Адаптивность**: 
  - На мобильных (< 768px): вертикальная раскладка, sidebar на всю ширину
  - На планшетах (< 1024px): уменьшенная ширина sidebar (250px)
