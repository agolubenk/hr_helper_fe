# Документация страницы: Ответы кандидатам (candidate-responses/page.tsx)

## Общее описание

Страница управления шаблонами ответов кандидатам. Позволяет создавать и управлять шаблонами ответов для отказа кандидатам, привязанными к грейдам или общими, а также управлять настройками слотов для быстрого копирования свободного времени.

**Путь**: `/candidate-responses`

## Компоненты страницы

### 1. CandidateResponsesPage (основной компонент)
- **Расположение**: `@/app/candidate-responses/page.tsx`
- **Назначение**: Главный компонент страницы с оберткой Suspense
- **Функциональность**: 
  - Оборачивает CandidateResponsesPageContent в Suspense для корректной работы с useSearchParams()
  - Показывает fallback (загрузку) пока компонент загружается
- **Связи**: 
  - CandidateResponsesPageContent: основной компонент содержимого

### 2. CandidateResponsesPageContent (основной компонент содержимого)
- **Расположение**: `@/app/candidate-responses/page.tsx`
- **Назначение**: Основной компонент содержимого страницы
- **Функциональность**: 
  - Управление активной вкладкой
  - Синхронизация активной вкладки с URL параметром ?tab=
  - Отображение вкладок и их содержимого
- **Связи**: 
  - GeneralTemplatesTab: вкладка "Общие"
  - GradeTemplatesTab: вкладка "По грейдам"
  - SlotsTab: вкладка "Слоты"

### 3. GeneralTemplatesTab
- **Расположение**: `@/components/candidate-responses/GeneralTemplatesTab.tsx`
- **Назначение**: Управление общими шаблонами ответов (не привязанными к грейдам)
- **Пропсы**: Нет (использует внутреннее состояние)
- **Функциональность**: 
  - Отображение списка общих шаблонов, сгруппированных по типам отказа
  - Аккордеон для каждого типа отказа (раскрытие/сворачивание)
  - Создание новых шаблонов через форму
  - Редактирование существующих шаблонов
  - Удаление шаблонов с подтверждением через toast
  - Отображение статуса шаблона (активен/неактивен)
- **Типы отказа**:
  - `office_format`: Офисный формат
  - `finance`: Финансы
  - `finance_more`: Финансы - больше
  - `finance_less`: Финансы - меньше
  - `general`: Общий отказ
- **Связи**: 
  - RejectionTemplateForm: форма создания/редактирования шаблона
  - useToast: для отображения уведомлений при удалении
  - API: загрузка и сохранение шаблонов (TODO)

  - Удаление шаблонов с подтверждением через toast

- **Грейды**:
  - Junior

### 6. RejectionTemplateForm
- **Расположение**: `@/components/candidate-responses/RejectionTemplateForm.tsx`
- **Назначение**: Форма создания/редактирования шаблона отказа
- **Пропсы**: 
  - `template`: RejectionTemplate | null - шаблон для редактирования (null для создания)
  - `rejectionType`: string - тип отказа или 'grade' для шаблонов по грейдам
  - `gradeId`: number | null - ID грейда (для шаблонов по грейдам)
  - `onSave`: () => void - обработчик сохранения
  - `onCancel`: () => void - обработчик отмены
- **Функциональность**: 
  - Выбор типа отказа (для общих шаблонов)
  - Выбор грейда (для шаблонов по грейдам)
  - Ввод названия шаблона
  - Ввод текста ответа (TextArea)
  - Переключатель активности шаблона (Switch)
  - Валидация формы (название и текст обязательны)
  - Сохранение шаблона (TODO: через API)
- **Связи**: 
  - Используется в GeneralTemplatesTab и GradeTemplatesTab
  - API: сохранение шаблона (TODO)

## Интерфейсы и типы

### RejectionTemplate
```typescript
interface RejectionTemplate {
  id: number                                    // Уникальный идентификатор шаблона
  rejection_type: string                       // Тип отказа (office_format, finance, etc.)
  rejection_type_display: string               // Отображаемое название типа отказа
  grade_id: number | null                      // ID грейда (null для общих шаблонов)
  grade_name: string | null                    // Название грейда (null для общих шаблонов)
  title: string                                // Название шаблона
  message: string                              // Текст ответа для отказа
  is_active: boolean                           // Флаг активности шаблона
}
```

### Grade
```typescript
interface Grade {
  id: number                                   // Уникальный идентификатор грейда
  name: string                                 // Название грейда (Junior, Middle, Senior и т.д.)
}
```

### SlotsSettings
```typescript
interface SlotsSettings {
  textBeforeCurrentWeek: string                // Текст перед слотами текущей недели
  textBeforeNextWeek: string                   // Текст перед слотами следующей недели
  textBeforeAllSlots: string                   // Текст перед всеми слотами
  separatorBetweenWeeks: string               // Разделитель между неделями
}
```

### RejectionTemplateFormProps
```typescript
interface RejectionTemplateFormProps {
  template: RejectionTemplate | null           // Шаблон для редактирования (null для создания)
  rejectionType: string                        // Тип отказа или 'grade'
  gradeId?: number | null                     // ID грейда (для шаблонов по грейдам)
  onSave: () => void                          // Обработчик сохранения
  onCancel: () => void                        // Обработчик отмены
}
```

## Состояние компонента

### CandidateResponsesPageContent

#### State переменные:
- `activeTab` ('general' | 'grades' | 'slots'): текущая активная вкладка
  - Инициализация: определяется из URL параметра ?tab= или 'general' по умолчанию
  - Обновление: при переключении вкладки или изменении URL параметра
  - Использование: определение, какой контент отображать

#### Hooks:
- `useSearchParams()`: получение параметров из URL
  - Используется для: чтения параметра ?tab=
- `useRouter()`: программная навигация
  - Используется для: обновления URL при переключении вкладок
- `useEffect()`: синхронизация activeTab с URL параметром
  - Зависимости: [tabParam]
  - Логика: обновляет activeTab при изменении tabParam в URL

### GeneralTemplatesTab

#### State переменные:
- `templates` (Record<string, RejectionTemplate[]>): словарь шаблонов по типам отказа
  - Инициализация: mockTemplates
  - Обновление: при создании, редактировании или удалении шаблона
- `expandedSections` (Set<string>): множество раскрытых секций аккордеона
  - Инициализация: new Set(['office_format']) - по умолчанию раскрыта секция "Офисный формат"
  - Обновление: при клике на заголовок секции
- `showForm` (boolean): флаг отображения формы создания/редактирования
  - Инициализация: false
  - Обновление: при открытии/закрытии формы
- `editingTemplate` (RejectionTemplate | null): редактируемый шаблон
  - Инициализация: null
  - Обновление: при открытии формы редактирования
- `formType` (string): тип отказа для формы
  - Инициализация: ''
  - Обновление: при открытии формы создания

#### Hooks:
- `useToast()`: для отображения уведомлений при удалении

### GradeTemplatesTab

## Функции и обработчики

### CandidateResponsesPageContent

#### handleTabChange(value)
- **Назначение**: Обработчик переключения вкладки
- **Параметры**: 
  - `value`: string - значение новой вкладки ('general', 'grades', 'slots')
- **Логика**: 
  1. Обновляет activeTab
  2. Обновляет URL параметр ?tab= через router.push()
  3. Отключает прокрутку страницы (scroll: false)
- **Поведение**: 
  - Вызывается при клике на вкладку
  - Обновляет URL для возможности делиться ссылками на конкретные вкладки
- **Связи**: 
  - Используется в Tabs.Root onValueChange

#### useEffect для синхронизации с URL
- **Назначение**: Синхронизация активной вкладки с URL параметром
- **Зависимости**: [tabParam]
- **Логика**: 
  1. Проверяет наличие tabParam в URL
  2. Валидирует значение (должно быть 'general', 'grades' или 'slots')
  3. Обновляет activeTab если значение валидно
- **Поведение**: 
  - Выполняется при изменении tabParam
  - Позволяет открывать страницу с конкретной вкладкой через URL
  - Пример: /candidate-responses?tab=grades откроет вкладку "По грейдам"

### GeneralTemplatesTab

#### toggleSection(type)
- **Назначение**: Переключение раскрытия/сворачивания секции аккордеона
- **Параметры**: 
  - `type`: string - тип отказа (ключ секции)
- **Логика**: 
  1. Проверяет наличие type в expandedSections
  2. Если есть - удаляет, если нет - добавляет
  3. Обновляет expandedSections
- **Поведение**: 
  - Вызывается при клике на заголовок секции
  - Раскрывает или сворачивает содержимое секции
- **Связи**: 
  - Используется в onClick заголовка секции

#### handleCreate(type)
- **Назначение**: Обработчик создания нового шаблона
- **Параметры**: 
  - `type`: string - тип отказа для нового шаблона
- **Логика**: 
  1. Устанавливает formType в type
  2. Сбрасывает editingTemplate в null
  3. Открывает форму (setShowForm(true))
- **Поведение**: 
  - Вызывается при клике на кнопку "Создать шаблон"
  - Открывает форму создания шаблона
- **Связи**: 
  - Используется в кнопке "Создать шаблон"

#### handleEdit(template)
- **Назначение**: Обработчик редактирования шаблона
- **Параметры**: 
  - `template`: RejectionTemplate - шаблон для редактирования
- **Логика**: 
  1. Устанавливает editingTemplate в template
  2. Устанавливает formType в template.rejection_type
  3. Открывает форму (setShowForm(true))
- **Поведение**: 
  - Вызывается при клике на кнопку "Редактировать"
  - Открывает форму редактирования с заполненными данными
- **Связи**: 
  - Используется в кнопке "Редактировать"

#### handleDelete(id, rejectionType?)
- **Назначение**: Обработчик удаления шаблона
- **Параметры**: 
  - `id`: number - ID шаблона для удаления
  - `rejectionType?`: string - тип отказа (опционально)
- **Логика**: 
  1. Показывает toast с предупреждением и кнопками "Отмена" и "Удалить"
  2. При подтверждении удаляет шаблон из templates
  3. Если rejectionType указан - удаляет из конкретной секции
  4. Если нет - ищет и удаляет из всех секций
- **Поведение**: 
  - Вызывается при клике на кнопку "Удалить"
  - Показывает предупреждение перед удалением
  - Удаляет шаблон после подтверждения
- **Связи**: 
  - Использует useToast для отображения предупреждения
  - Используется в кнопке "Удалить"

#### handleFormClose()
- **Назначение**: Закрытие формы создания/редактирования
- **Параметры**: Нет
- **Логика**: 
  1. Закрывает форму (setShowForm(false))
  2. Сбрасывает editingTemplate в null
  3. Сбрасывает formType в ''
- **Поведение**: 
  - Вызывается при отмене или после сохранения
  - Возвращает к списку шаблонов
- **Связи**: 
  - Используется в RejectionTemplateForm onCancel

#### handleFormSave()
- **Назначение**: Обработчик сохранения шаблона
- **Параметры**: Нет
- **Логика**: 
  1. TODO: Сохранение через API
  2. Закрывает форму через handleFormClose()
- **Поведение**: 
  - Вызывается после успешного сохранения в форме
  - В текущей реализации только закрывает форму
- **Связи**: 
  - Используется в RejectionTemplateForm onSave
- **TODO**: Реализовать сохранение через API

## URL параметры

### ?tab=general | ?tab=grades | ?tab=slots
- **Назначение**: Открытие конкретной вкладки при загрузке страницы
- **Логика**: 
  - Читается через useSearchParams().get('tab')
  - Валидируется перед установкой (должно быть 'general', 'grades' или 'slots')
  - Используется для глубоких ссылок
- **Поведение**: 
  - При загрузке страницы с параметром ?tab= определяет активную вкладку
  - При переключении вкладки обновляет URL параметр
  - Позволяет делиться ссылками на конкретные вкладки
- **Пример**: 
  - `/candidate-responses?tab=grades` откроет вкладку "По грейдам"
  - `/candidate-responses?tab=slots` откроет вкладку "Слоты"
