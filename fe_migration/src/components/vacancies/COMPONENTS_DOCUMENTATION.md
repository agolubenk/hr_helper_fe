# Документация компонентов: vacancies

## Общее описание

Директория `components/vacancies` содержит компоненты для управления вакансиями в системе HR Helper. Компоненты обеспечивают отображение списка вакансий, создание новых вакансий, редактирование существующих, просмотр детальной информации и настройку различных параметров вакансий.

**Расположение**: `@/components/vacancies`

## Компоненты

### Основные компоненты списка

#### 1. VacanciesStats
- **Файл**: `VacanciesStats.tsx`
- **Назначение**: Компонент статистики по вакансиям
- **Пропсы**:
  - `total`: общее количество вакансий (number)
  - `active`: количество активных вакансий (number)
  - `inactive`: количество неактивных вакансий (number)
- **Функциональность**: 
  - Отображает 3 карточки со статистикой по вакансиям
  - Каждая карточка содержит число и иконку
  - Иконки: FileTextIcon (всего), CheckIcon (активные), StopwatchIcon (неактивные)
- **Используется в**: `/app/vacancies/page.tsx`

#### 2. VacanciesSearchFilters
- **Файл**: `VacanciesSearchFilters.tsx`
- **Назначение**: Компонент поиска и фильтров для вакансий
- **Пропсы**:
  - `searchQuery`: текущий поисковый запрос (string)
  - `onSearchChange`: обработчик изменения поискового запроса ((value: string) => void)
  - `selectedRecruiter`: выбранный рекрутер ('all' - все рекрутеры) (string)
  - `onRecruiterChange`: обработчик изменения выбранного рекрутера ((value: string) => void)
  - `selectedStatus`: выбранный статус ('all' | 'active' | 'inactive') (string)
  - `onStatusChange`: обработчик изменения выбранного статуса ((value: string) => void)
- **Функциональность**: 
  - Поле поиска: поиск по названию вакансии или ID
  - Фильтр по рекрутеру: выбор рекрутера из списка
  - Фильтр по статусу: выбор статуса (Все, Активные, Неактивные)
  - Кнопка поиска: выполнение поиска с применением фильтров
- **Используется в**: `/app/vacancies/page.tsx`

#### 3. VacancyCard
- **Файл**: `VacancyCard.tsx`
- **Назначение**: Компонент карточки вакансии (режим "Карточки")
- **Пропсы**:
  - `vacancy`: данные вакансии (Vacancy)
  - `onClick`: обработчик клика на карточку (опционально)
  - `onEditClick`: обработчик клика на кнопку редактирования (опционально)
  - `onStatusClick`: обработчик клика на статус (опционально)
- **Функциональность**: 
  - Отображает информацию о вакансии в виде карточки
  - Заголовок, ID, статус, рекрутер, локации, интервьюеры, предупреждения
  - Кнопки действий: редактирование и просмотр
  - Цветовая индикация статуса (зеленый для активных, серый для неактивных)
- **Используется в**: `/app/vacancies/page.tsx` (режим "Карточки")

#### 4. VacancyListItem
- **Файл**: `VacancyListItem.tsx`
- **Назначение**: Компонент элемента списка вакансии (режим "Список")
- **Пропсы**:
  - `vacancy`: данные вакансии (Vacancy)
  - `onClick`: обработчик клика на элемент списка (опционально)
  - `onEditClick`: обработчик клика на кнопку редактирования (опционально)
  - `onStatusClick`: обработчик клика на статус (опционально)
- **Функциональность**: 
  - Отображает информацию о вакансии в виде элемента списка
  - Компактное горизонтальное расположение информации
  - Заголовок, статус, ID, рекрутер, локации, интервьюеры, предупреждения в одной строке
  - Кнопки действий: редактирование и просмотр
- **Используется в**: `/app/vacancies/page.tsx` (режим "Список")

### Модальные окна

#### 5. AddVacancyModal
- **Файл**: `AddVacancyModal.tsx`
- **Назначение**: Модальное окно добавления новой вакансии
- **Пропсы**:
  - `isOpen`: флаг открытости модального окна (boolean)
  - `onClose`: обработчик закрытия модального окна (() => void)
  - `onSave`: обработчик сохранения вакансии ((data: AddVacancyFormData) => void)
- **Функциональность**: 
  - Создание новой вакансии с настройкой всех параметров
  - Многошаговая форма с 12 вкладками настроек
  - Боковая панель с навигацией по вкладкам
  - Адаптивный режим overlay для мобильных устройств (< 800px)
  - Валидация обязательных полей перед сохранением
  - Управление рекрутерами (выбор нескольких, назначение главного)
- **Вкладки**: Основная информация, Текст вакансии, Рекрутеры, Заказчики и интервьюеры, Вопросы и ссылки, Связи и интеграции, Статусы, Зарплатные вилки, Встречи и интервью, Scorecard, Обработка данных, История правок
- **Используется в**: `/app/vacancies/page.tsx`

#### 6. VacancyEditModal
- **Файл**: `VacancyEditModal.tsx`
- **Назначение**: Модальное окно редактирования/просмотра вакансии
- **Пропсы**:
  - `open`: флаг открытости модального окна (boolean)
  - `onOpenChange`: обработчик изменения открытости ((open: boolean) => void)
  - `vacancyId`: ID вакансии (number | null)
  - `vacancyTitle`: название вакансии (string, опционально)
  - `vacancy`: данные вакансии (VacancyViewItem | null, опционально)
  - `mode`: режим работы ('edit' | 'view', по умолчанию 'edit')
  - `vacancyStatus`: статус вакансии ('active' | 'inactive', опционально)
  - `onVacancyStatusChange`: обработчик изменения статуса (опционально)
  - `onSwitchToEdit`: обработчик переключения в режим редактирования (опционально)
  - `onSave`: обработчик сохранения (опционально)
- **Функциональность**: 
  - Редактирование существующей вакансии
  - Просмотр вакансии в режиме только чтения
  - 11 вкладок настроек (аналогично AddVacancyModal)
  - Управление всеми параметрами вакансии
  - История правок
  - **Текст вакансии**: вкладки с выбором страны, активность вакансии для каждой страны
  - **Вопросы и ссылки**: автоматическая активация/деактивация стран на основе активности вакансии
  - **Встречи и интервью**: связь с настройками этапов найма
    - Формат встречи автоматически устанавливается в "Онлайн" и становится disabled, если `showOffices = false` для выбранного этапа
    - При выборе этапа проверяется `showOffices`, и формат обновляется соответственно
- **Связь с настройками этапов**:
  - Использует `recruitmentStagesWithMeetings` для получения настроек этапов (`isMeeting`, `showOffices`, `showInterviewers`)
  - Формат встречи динамически управляется настройками выбранного этапа
- **Используется в**: `/app/vacancies/[id]/page.tsx`, `/app/vacancies/[id]/edit/page.tsx`

## Интерфейсы и типы

### Vacancy
```typescript
interface Vacancy {
  id: number                                    // Уникальный идентификатор вакансии
  title: string                                 // Название вакансии
  status: 'active' | 'inactive'                 // Статус вакансии
  recruiter: string                            // Имя рекрутера
  locations: string[]                           // Массив локаций вакансии
  interviewers: number                          // Количество интервьюеров
  date: string | null                           // Дата создания/обновления
  hasWarning: boolean                           // Флаг наличия предупреждения
  warningText?: string                          // Текст предупреждения
}
```

### AddVacancyFormData
```typescript
interface AddVacancyFormData {
  basicInfo: {
    title: string                               // Название вакансии
    status: 'active' | 'inactive'               // Статус вакансии
    recruiter: string                           // ID рекрутера
    technologies: string                         // Технологии (через запятую)
    huntflowId: string                          // ID в Huntflow
  }
  text: {
    departmentId: string                        // ID департамента
    header: string                              // Заголовок вакансии
  }
  recruiters: {
    selectedIds: string[]                       // Массив ID выбранных рекрутеров
    mainId: string | null                       // ID главного рекрутера
  }
}
```

## Функции и обработчики

### AddVacancyModal

#### handleBasicInfoChange
- **Назначение**: Обработчик изменения основной информации
- **Параметры**: `data: Partial<AddVacancyFormData['basicInfo']>`
- **Логика**: Обновляет данные основной информации в форме

#### handleRecruiterToggle
- **Назначение**: Обработчик переключения выбора рекрутера
- **Параметры**: `id: string` - ID рекрутера
- **Логика**: 
  - Если рекрутер уже выбран - удаляет его из списка
  - Если рекрутер не выбран - добавляет его в список
  - Если удаляется главный рекрутер - сбрасывает главного

#### handleMainRecruiterToggle
- **Назначение**: Обработчик переключения главного рекрутера
- **Параметры**: `id: string` - ID рекрутера
- **Логика**: 
  - Если рекрутер не выбран - не выполняет действий
  - Если рекрутер уже главный - снимает его с главного
  - Если рекрутер не главный - делает его главным

#### handleSave
- **Назначение**: Обработчик сохранения вакансии
- **Логика**: 
  - Валидирует обязательные поля (название, рекрутер)
  - При ошибке валидации показывает alert
  - При успехе вызывает onSave и закрывает модальное окно

#### useMatchMedia
- **Назначение**: Хук для определения соответствия медиа-запросу
- **Параметры**: `query: string` - медиа-запрос
- **Возвращает**: `boolean` - true если запрос соответствует
- **Используется для**: Определения overlay режима на мобильных устройствах

## Связи с другими компонентами

- Используется в `/app/vacancies/page.tsx` - страница управления вакансиями
- Используется в `/app/vacancies/[id]/page.tsx` - страница детального просмотра
- Используется в `/app/vacancies/[id]/edit/page.tsx` - страница редактирования
- Использует компоненты из `@radix-ui/themes` (Box, Flex, Text, Button, Dialog, TextField, Select и т.д.)
- Использует иконки из `@radix-ui/react-icons`
- Использует `useToast` из `@/components/Toast/ToastContext` для уведомлений
- Связан с Huntflow API для синхронизации данных

## Особенности реализации

1. **Два режима отображения**: Карточки (VacancyCard) и Список (VacancyListItem)
2. **Многошаговая форма**: AddVacancyModal и VacancyEditModal с 12 вкладками настроек
3. **Адаптивность**: Overlay режим для мобильных устройств (< 800px)
4. **Валидация**: Проверка обязательных полей перед сохранением
5. **Управление рекрутерами**: Выбор нескольких рекрутеров с назначением главного
6. **Цветовая индикация**: Статусы вакансий с цветовой индикацией
7. **Предупреждения**: Отображение предупреждений для вакансий
8. **Интеграция с Huntflow**: Связь с внешней системой через Huntflow ID
9. **История правок**: Отслеживание изменений вакансии
10. **Модульная структура**: Разделение на секции для просмотра и редактирования
