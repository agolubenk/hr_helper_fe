# Документация компонентов: GlobalSearch

## Общее описание

Директория `components/GlobalSearch` содержит компонент глобального поиска по приложению. Компонент обеспечивает поиск по вакансиям, кандидатам и компаниям с автодополнением, историей поиска, популярными запросами и предложениями изменения области поиска.

**Расположение**: `@/components/GlobalSearch`

## Компоненты

### 1. GlobalSearch
- **Файл**: `GlobalSearch.tsx`
- **Назначение**: Компонент глобального поиска по приложению
- **Пропсы**:
  - `placeholder`: текст плейсхолдера (string, по умолчанию 'Поиск...')
  - `shortcutHint`: подсказка для горячей клавиши (string, опционально, например '⌘K' или 'Ctrl+K')
  - `dark`: темная тема для правого слота (boolean, опционально, по умолчанию false)
  - `onSearch`: обработчик выполнения поиска ((query: string, scope: ScopeType | null) => void, опционально)
  - `onEntityClick`: обработчик клика по сущности ((entity: EntityData) => void, опционально)
- **Функциональность**: 
  - Поле ввода с автодополнением
  - Выпадающий список с предложениями:
    - **Запросы (query)**: история, популярные, автодополнение
    - **Сущности (entity)**: вакансии, кандидаты, компании
    - **Области поиска (scope)**: изменение области поиска
  - Поддержка горячих клавиш (Cmd+K / Ctrl+K) для фокуса
  - Выделение совпадающего префикса в предложениях
  - Навигация клавиатурой (стрелки, Enter, Escape)
  - Чип выбранной области поиска с возможностью сброса
- **Состояние**:
  - `value`: введенный текст поиска (string)
  - `scope`: выбранная область поиска (ScopeType | null, null означает "во всех данных")
  - `open`: флаг открытости выпадающего списка предложений (boolean)
  - `activeIndex`: индекс активного предложения для навигации клавиатурой (number)
- **Особенности**: 
  - Минимальная длина запроса для поиска сущностей: 2 символа
  - При пустом вводе и открытом списке показывает историю (первые 5 запросов)
  - Порядок предложений: сущности → запросы → области поиска
  - Автоматическая прокрутка активного элемента в видимую область
  - Закрытие списка при клике вне компонента
- **Используется в**: 
  - `/components/Header.tsx`: отображается в верхней панели
  - `/app/search/page.tsx`: может использоваться на странице поиска

## Интерфейсы и типы

### ScopeType
```typescript
export type ScopeType = 'all' | 'vacancies' | 'candidates' | 'companies'
```
- **Назначение**: Тип области поиска
- **Варианты**:
  - `'all'`: поиск во всех данных
  - `'vacancies'`: поиск только в вакансиях
  - `'candidates'`: поиск только в кандидатах
  - `'companies'`: поиск только в компаниях

### EntityData
```typescript
export interface EntityData {
  id: string                                    // Уникальный идентификатор сущности
  entityType: 'vacancy' | 'candidate' | 'company'  // Тип сущности
  title: string                                 // Заголовок сущности
  subtitle?: string                             // Подзаголовок (опционально)
  meta?: string                                 // Дополнительная информация (опционально)
}
```
- **Назначение**: Интерфейс данных сущности (вакансия, кандидат, компания)
- **Используется для**: Представления результатов поиска сущностей

### SuggestionType
```typescript
export type SuggestionType = 'query' | 'entity' | 'scope'
```
- **Назначение**: Тип предложения в автодополнении
- **Варианты**:
  - `'query'`: предложение запроса (из истории, популярных или автодополнения)
  - `'entity'`: предложение сущности (вакансия, кандидат, компания)
  - `'scope'`: предложение изменения области поиска

### Suggestion
```typescript
export interface Suggestion {
  type: SuggestionType                         // Тип предложения
  label: string                                // Отображаемый текст предложения
  query: string                                // Текст запроса для поиска
  scope?: ScopeType                            // Область поиска (опционально, только для type === 'scope')
  entity?: EntityData                          // Данные сущности (опционально, только для type === 'entity')
  matchPrefix?: string                         // Совпадающий префикс для выделения (опционально, только для type === 'query')
}
```
- **Назначение**: Интерфейс предложения в автодополнении
- **Используется для**: Представления всех типов предложений в выпадающем списке

### GlobalSearchProps
```typescript
export interface GlobalSearchProps {
  placeholder?: string                         // Плейсхолдер
  shortcutHint?: string                        // Подсказка для горячей клавиши, напр. "⌘K"
  dark?: boolean                               // Тема для правого слота (не передаём — не показываем)
  onSearch?: (query: string, scope: ScopeType | null) => void  // Выполнить поиск (запрос + опциональный скоуп)
  onEntityClick?: (entity: EntityData) => void  // Переход по сущности (вакансия/кандидат/компания)
}
```

## Функции и обработчики

### filterEntities
- **Назначение**: Фильтрация сущностей по поисковому запросу
- **Параметры**: 
  - `q`: поисковый запрос (string)
  - `limit`: максимальное количество результатов (number)
- **Возвращает**: массив отфильтрованных сущностей (EntityData[])
- **Логика**: 
  - Если запрос пустой или короче 2 символов - возвращает пустой массив
  - Фильтрует сущности по заголовку, подзаголовку и мета-информации (регистронезависимый поиск)
  - Ограничивает количество результатов до limit
- **Используется для**: Поиска сущностей при вводе текста

### getQueries
- **Назначение**: Получение предложений запросов для автодополнения
- **Параметры**: 
  - `value`: введенный текст (string)
  - `limit`: максимальное количество предложений (number)
- **Возвращает**: массив предложений запросов (Suggestion[])
- **Логика**: 
  1. Проверяет автодополнение по префиксу (если введенный текст начинается с ключа из MOCK_SUGGESTS)
  2. Добавляет популярные запросы, содержащие введенный текст
  3. Добавляет запросы из истории, содержащие введенный текст
  4. Удаляет дубликаты
  5. Ограничивает количество результатов до limit
  6. Вычисляет matchPrefix для выделения совпадающего префикса
- **Используется для**: Генерации предложений запросов в автодополнении

### getScopedSuggestions
- **Назначение**: Получение предложений изменения области поиска
- **Параметры**: 
  - `query`: поисковый запрос (string)
  - `currentScope`: текущая область поиска (ScopeType | null)
  - `limit`: максимальное количество предложений (number)
- **Возвращает**: массив предложений изменения области поиска (Suggestion[])
- **Логика**: 
  - Если запрос пустой - возвращает пустой массив
  - Пропускает текущую область поиска (не предлагает её)
  - Генерирует предложения для всех остальных областей
  - Формат предложения: "Найти «{query}» среди {scopeLabel}" или "Найти «{query}» во всех данных"
- **Используется для**: Предложения изменения области поиска в автодополнении

### handleKeyDown
- **Назначение**: Обработчик нажатий клавиш
- **Параметры**: `e: React.KeyboardEvent`
- **Логика**: 
  - **Escape**: закрывает список предложений
  - **Enter**: 
    - Если есть активное предложение - выбирает его (сущность, запрос или область)
    - Если нет активного - выполняет поиск с текущим текстом и областью
  - **ArrowDown**: переходит к следующему предложению (циклически)
  - **ArrowUp**: переходит к предыдущему предложению (циклически)
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `showDropdown`, `flat`, `activeIndex`, `value`, `scope`, `onSearch`, `onEntityClick`

### selectItem
- **Назначение**: Выбор предложения из списка
- **Параметры**: 
  - `s`: выбранное предложение (Suggestion)
  - `index`: индекс предложения в плоском списке (number)
- **Логика**: 
  - Устанавливает activeIndex
  - Для сущности: вызывает onEntityClick и закрывает список
  - Для области поиска: устанавливает область, выполняет поиск и закрывает список
  - Для запроса: устанавливает текст запроса, выполняет поиск и закрывает список
- **useCallback**: Мемоизирует функцию для оптимизации производительности
- **Зависимости**: `onSearch`, `onEntityClick`, `scope`

### handleFocus
- **Назначение**: Обработчик фокуса на поле ввода
- **Логика**: Открывает выпадающий список предложений

### handleChange
- **Назначение**: Обработчик изменения текста в поле ввода
- **Параметры**: `v: string` - новое значение поля ввода
- **Логика**: 
  - Обновляет value
  - Открывает список предложений для показа предложений

### clearScope
- **Назначение**: Сброс выбранной области поиска
- **Логика**: Сбрасывает область поиска в null (поиск "во всех данных")

## Хуки и эффекты

### useMemo: сбор всех подсказок для автодополнения
- **Назначение**: Генерация предложений сущностей, запросов и областей поиска
- **Возвращает**: объект с `entities`, `queries`, `scoped`, `flat`
- **Логика**: 
  - `entities`: сущности (только если введен текст, минимум 1 символ)
  - `queries`: запросы (из автодополнения, популярных, истории)
    - Если пустой ввод и открыт список - показывает историю (первые 5)
  - `scoped`: предложения изменения области поиска (только если введен текст)
  - `flat`: плоский список всех предложений (entities → queries → scoped)
- **Порядок в flat**: 
  1. Сначала сущности (entities)
  2. Затем запросы (queries)
  3. В конце области поиска (scoped)
- **Зависимости**: `value`, `scope`, `open`

### useEffect: обработка кликов вне компонента
- **Назначение**: Закрывает выпадающий список при клике вне компонента
- **Логика**: 
  - Выполняется только если список открыт (open === true)
  - Проверяет, был ли клик вне контейнера поиска
  - Если да - закрывает список
- **Зависимости**: `open`

### useEffect: сброс активного индекса
- **Назначение**: Сбрасывает activeIndex в 0 при изменении количества предложений
- **Логика**: Выполняется при изменении flat.length
- **Зависимости**: `flat.length`

### useEffect: прокрутка активного элемента
- **Назначение**: Прокручивает активное предложение в видимую область при навигации клавиатурой
- **Логика**: 
  - Выполняется при изменении activeIndex, showDropdown или flat.length
  - Находит элемент с data-active="true" и прокручивает его в видимую область
  - Использует smooth прокрутку для плавной анимации
- **Зависимости**: `activeIndex`, `showDropdown`, `flat.length`

## Вычисляемые значения

### showDropdown
- **Назначение**: Флаг отображения выпадающего списка предложений
- **Вычисление**: `open && (flat.length > 0 || value.length >= 1)`
- **Логика**: Показывается если список открыт (open === true) И (есть предложения ИЛИ введен текст)

### scopeChipLabel
- **Назначение**: Текст для отображения выбранной области поиска
- **Вычисление**: `scope ? SCOPE_LABELS[scope] : null`
- **Логика**: Возвращает название области или null если область не выбрана

## Использование

### Пример использования GlobalSearch:
```tsx
import GlobalSearch from '@/components/GlobalSearch/GlobalSearch'

function Header() {
  const router = useRouter()
  
  const handleSearch = (query: string, scope: ScopeType | null) => {
    // Выполнение поиска
    router.push(`/search?q=${encodeURIComponent(query)}${scope ? `&scope=${scope}` : ''}`)
  }
  
  const handleEntityClick = (entity: EntityData) => {
    // Переход к детальной странице сущности
    if (entity.entityType === 'vacancy') {
      router.push(`/vacancies/${entity.id}`)
    } else if (entity.entityType === 'candidate') {
      router.push(`/candidates/${entity.id}`)
    } else if (entity.entityType === 'company') {
      router.push(`/companies/${entity.id}`)
    }
  }
  
  return (
    <GlobalSearch 
      placeholder="Поиск вакансий, кандидатов, компаний..."
      shortcutHint="⌘K"
      onSearch={handleSearch}
      onEntityClick={handleEntityClick}
    />
  )
}
```
