# Документация компонентов: invites

## Общее описание

Директория `components/invites` содержит компоненты для управления инвайтами на интервью. Компоненты обеспечивают отображение статистики по инвайтам и создание новых инвайтов из комбинированных данных.

**Расположение**: `@/components/invites`

## Компоненты

### 1. InvitesStats
- **Файл**: `InvitesStats.tsx`
- **Назначение**: Компонент статистики по инвайтам на интервью
- **Пропсы**:
  - `total`: общее количество инвайтов (number)
  - `pending`: количество инвайтов со статусом "pending" (number)
  - `sent`: количество инвайтов со статусом "sent" (number)
  - `completed`: количество инвайтов со статусом "completed" (number)
- **Функциональность**: 
  - Отображение 4 карточек со статистикой
  - Каждая карточка имеет уникальную цветовую схему (границы сверху и снизу)
  - Иконки для визуального различия статусов
  - Крупный шрифт для чисел
- **Цветовые схемы карточек**:
  - "Всего инвайтов": синий сверху, зеленый снизу (CalendarIcon)
  - "Ожидают": оранжевый сверху и снизу (ClockIcon)
  - "Отправлены": синий сверху и снизу (PaperPlaneIcon)
  - "Завершены": зеленый сверху и снизу (CheckCircledIcon)
- **Особенности**: 
  - Карточки с цветными границами для визуального различия
  - Иконки для каждого типа статистики
  - Текст в верхнем регистре для заголовков
- **Используется в**: 
  - `/app/invites/page.tsx`: отображается на странице управления инвайтами

### 2. CreateInviteModal
- **Файл**: `CreateInviteModal.tsx`
- **Назначение**: Модальное окно создания инвайта на интервью
- **Пропсы**:
  - `isOpen`: флаг открытости модального окна (boolean)
  - `onClose`: обработчик закрытия модального окна (() => void)
  - `onSave`: обработчик успешного сохранения ((data: { combined_data: string }) => void, опционально)
- **Функциональность**: 
  - Поле ввода для комбинированных данных (ссылка на кандидата + дата-время интервью)
  - Поддержка различных форматов даты-времени
  - Поддержка кастомной длительности интервью (в скобках)
  - Информационная секция с примерами форматов
  - Валидация введенных данных
  - Отправка данных на сервер для создания инвайта
  - CSRF защита при отправке
- **Форматы данных**:
  - **Ссылка на кандидата**: 
    - Sandbox: `https://sandbox.huntflow.dev/my/org694#/vacancy/[id вакансии]/filter/[id статуса]/id/[id кандидата]`
    - Production: `https://huntflow.ru/my/[название_аккаунта]#/vacancy/[id вакансии]/filter/[id статуса]/id/[id кандидата]`
  - **Дата-время**: 
    - `YYYY-MM-DD HH:MM` (например, 2025-09-15 14:00)
    - `DD.MM.YYYY HH:MM` (например, 15.09.2025 14:00)
    - `DD/MM/YYYY HH:MM` (например, 15/09/2025 14:00)
  - **Длительность** (опционально, в скобках): 
    - `(1 час)`, `(30 минут)`, `(полчаса)`, `(2 ч)`, `(45 м)`
- **Особенности**: 
  - Monospace шрифт для поля ввода (лучшая читаемость ссылок и дат)
  - Информационная секция с примерами для каждого формата
  - Индикатор загрузки при отправке (isSubmitting)
  - Очистка формы при закрытии
  - TODO: Реализовать реальный API вызов
- **Используется в**: 
  - `/app/invites/page.tsx`: открывается при клике на "Создать инвайт"

## Функции и обработчики

### CreateInviteModal

#### getCsrfToken
- **Назначение**: Получение CSRF токена из cookies
- **Возвращает**: CSRF токен (string | null)
- **Логика**: 
  - Извлекает cookie с именем 'csrftoken'
  - Используется для защиты от CSRF атак
  - Возвращает null при SSR

#### handleSave
- **Назначение**: Обработка сохранения инвайта
- **Логика**: 
  1. Валидация: проверяет, что поле не пустое
  2. Устанавливает isSubmitting в true
  3. TODO: Отправка данных на сервер через POST `/google-oauth/invites/create/combined/`
  4. При успехе: показывает Toast, вызывает onSave, очищает форму, закрывает модальное окно
  5. При ошибке: показывает Toast с ошибкой
  6. В finally: устанавливает isSubmitting в false
- **API формат** (TODO):
  - Method: POST
  - URL: `/google-oauth/invites/create/combined/`
  - Headers: 
    - `Content-Type: application/x-www-form-urlencoded`
    - `X-CSRFToken: {csrfToken}` (если есть)
  - Body: `combined_data={combinedData}`
  - Credentials: include (для cookies)

#### handleClose
- **Назначение**: Обработка закрытия модального окна
- **Логика**: Очищает форму и закрывает модальное окно
