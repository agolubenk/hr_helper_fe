# Ревью стилей: Error Pages (fe_migration)

Дата: 2026-03-24
Область: `fe_migration/src/app/pages/styles/errors/*`

## Цель

Зафиксировать единые правила стилизации карточки ошибок (`401/402/403/404/500`) и избежать регрессий при работе с темами.

## Базовый источник стилей

- Общая карточка и фон для страниц ошибок берутся из:
  - `src/app/pages/styles/errors/error-pages-base.module.css`
- Специфичные стили страницы (иконка, цвет кода ошибки, локальная анимация) — в:
  - `error-401.module.css`
  - `error-402.module.css`
  - `error-403.module.css`
  - `error-404.module.css`
  - `error-500.module.css`

## Критичный момент (обязательная проверка)

### Dark theme selector mismatch

Проблема:
- Dark-стили карточки не применялись, потому что селектор в CSS не совпадал с фактическим DOM-контекстом темы.

Симптом:
- В dark-теме карточка ошибки выглядела светло-серой, как в light-варианте.

Что проверять:
- В `error-pages-base.module.css` должны быть селекторы, которые реально матчатся с текущим DOM:
  - `[data-theme="dark"] .content`
  - `.radix-theme[data-appearance="dark"] .content`
  - `.radix-themes[data-appearance="dark"] .content` (для совместимости)

Критерий готовности:
- На `/errors/401` и `/errors/402` в dark-теме карточка визуально соответствует ожидаемому dark-варианту (без светлого фона).

## Правила изменений

- Не дублировать базовые стили карточки по страницам, если нет необходимости.
- При изменении правил темы всегда проверять оба режима:
  - `light`
  - `dark`
- Любое изменение селекторов темы должно сопровождаться ручной проверкой:
  - `/errors/401`
  - `/errors/402`
  - `/errors/403`
  - `/errors/404`
  - `/errors/500`

## Чек-лист перед merge

- [ ] Карточка ошибки в dark-теме не светлее эталона.
- [ ] Нет расхождения между `data-theme` и `data-appearance` селекторами.
- [ ] Визуально одинаковая база карточки на всех страницах ошибок.
- [ ] Проверены все 5 маршрутов ошибок в light/dark.
